## 一句话总结

**梯度 $\nabla f$ = 把一个点上的所有偏导，按顺序排成的一个向量。它指向函数「上升最快」的方向；反过来，负梯度 $-\nabla f$ 就是「下降最快」的方向——训练时就朝它走。**

$$\nabla f=\left(\frac{\partial f}{\partial x_1},\ \frac{\partial f}{\partial x_2},\ \dots,\ \frac{\partial f}{\partial x_n}\right)$$

> ⚠️ 记号 $\nabla$ 读「nabla / del / grad」。输入一个**多元函数 + 一个点**，输出**一个向量**（和自变量一样长）。别和偏导（输出一个数）搞混：**偏导是分量，梯度是整根向量。**

对学 LLM 来说，最该记住的一句话是：

> **梯度 = 「所有参数各自该往哪调」的总清单。** 它是一个和参数一样长的向量，第 $i$ 个分量 $\partial L/\partial w_i$ 告诉你第 $i$ 个参数的调节方向和力度。训练一步就是：**每个参数都反着自己的梯度分量挪一小步**（2.5）。

---

## 1. 为什么要学梯度？（和训练的关系）

2.2 里我们学会了对**单个**参数求偏导 $\dfrac{\partial L}{\partial w_i}$——「只动这一个参数，损失变多少」。

但模型有**几百万到几千亿个参数**，训练时要**同时**决定「每个参数各自往哪调、调多少」。一个个偏导散着放太乱了，于是把它们**打包成一个向量**：

$$\nabla L=\left(\frac{\partial L}{\partial w_1},\ \frac{\partial L}{\partial w_2},\ \dots,\ \frac{\partial L}{\partial w_n}\right)$$

这就是**梯度**——一份「全体参数的调节清单」。有了它，训练一步就能写成一行向量式：

$$\mathbf{w}\leftarrow\mathbf{w}-\eta\,\nabla L$$

> 一句话：**偏导管一个旋钮，梯度是「把全部旋钮的调法一次性列出来的那张表」。**

---

## 2. 直觉：站在山坡上，梯度指向「最陡的上坡」

回忆 2.2：二元函数 $z=f(x,y)$ 是一张**曲面（山地）**，$x,y$ 是经纬度，$z$ 是海拔。

站在山上某一点，往不同方向走坡度不同。**梯度 $\nabla f$ 就是一个箭头，指向「此处上坡最陡的方向」**，它的**长度**就是那个最陡方向上的坡度大小。

```
        俯视图（等高线 = 地图上的一圈圈海拔线）
                              ∇f  ← 梯度：指向最陡上坡
              .-''''''-.      ↑        （垂直于等高线）
            .'          '.    │
           /   .-''-.    \    │
          |   / 高地 \    |   ●  ← 你站的位置
           \   '-..-'    /
            '.          .'
              '-......-'
```

两条关键性质（下面第 5、6 节会算给你看）：

1. **方向**：$\nabla f$ 指向**上升最快**的方向；$-\nabla f$ 指向**下降最快**的方向（下山最快就朝这走 → 梯度下降）。
2. **大小**：$\lVert\nabla f\rVert$ = 最陡方向上的坡度。梯度越长，此处越陡、损失面越「急」。

> 💡 训练在做的事：站在损失曲面的高处，想尽快走到谷底（损失最小），于是**每一步都朝 $-\nabla L$ 迈一小步**。这就是「梯度下降」的画面（2.5 讲透）。

## 3. 怎么算：把偏导排成一个向量

**算梯度没有新招——就是把 2.2 的偏导一个个求出来，按顺序摞成一列。**

例：$f(x,y)=x^2y+3y$（正是 2.2 那个例子）

- $\dfrac{\partial f}{\partial x}=2xy$（把 $y$ 当常数）
- $\dfrac{\partial f}{\partial y}=x^2+3$（把 $x$ 当常数）

所以

$$\nabla f(x,y)=\big(2xy,\ \ x^2+3\big)$$

在点 $(2,5)$：$\nabla f=(2\cdot2\cdot5,\ \ 2^2+3)=(20,\ 7)$。

> 📌 梯度是「随点变化」的：换个点，箭头的方向和长度都变。$\nabla f$ 本身是一个**向量值函数**——每个点吐出一个向量。

三元也一样，例 $g(x,y,z)=x^2+y^2+z^2$：

$$\nabla g=(2x,\ 2y,\ 2z)$$

---

## 4. 记号怎么读、怎么写

| 记号 | 读法 | 说明 |
|------|------|------|
| $\nabla f$ | nabla f / grad f | 最常见，「$f$ 的梯度」 |
| $\operatorname{grad} f$ | grad f | 同上，老教材写法 |
| $\nabla_{\mathbf w} L$ | | 强调「对参数 $\mathbf w$ 求梯度」，深度学习常见 |
| $\dfrac{\partial L}{\partial \mathbf w}$ | | 也表示梯度（对整个参数向量求导），和 $\nabla_{\mathbf w}L$ 同义 |
| $\nabla f(\mathbf x_0)$ | | 在具体点 $\mathbf x_0$ 求出的那个**数值向量** |

梯度的分量含义，和偏导（2.2）完全一致，只是现在**一次看一整排**：

| $\partial L/\partial w_i$（第 $i$ 个分量） | 含义 | 该参数怎么调 |
|:---:|------|------|
| $>0$ | 调大 $w_i$ → 损失变大 | 把 $w_i$ **调小** |
| $<0$ | 调大 $w_i$ → 损失变小 | 把 $w_i$ **调大** |
| $=0$ | 这个方向平 | 暂时不动它 |
| 绝对值大 | 这个参数敏感 | 影响大 |

> 所以「反着梯度走」= 每个分量都按上表调 = $\mathbf w\leftarrow\mathbf w-\eta\nabla L$。一根向量式，把 $n$ 个参数的调节一次说清。

---

## 5. 核心：方向导数 = 梯度·方向（为什么梯度是最陡方向）

这一节解释「梯度为什么恰好指向最陡上坡」——用到 [1.5 点积](../01-线性代数/1.5-点积.md)。

**方向导数**：沿任意**单位方向** $\mathbf u$（$\lVert\mathbf u\rVert=1$）走，坡度是多少？答案是梯度和方向的**点积**：

$$D_{\mathbf u}f=\nabla f\cdot\mathbf u$$

回忆 1.5 点积的几何式：$\nabla f\cdot\mathbf u=\lVert\nabla f\rVert\,\lVert\mathbf u\rVert\cos\theta=\lVert\nabla f\rVert\cos\theta$（因为 $\lVert\mathbf u\rVert=1$），$\theta$ 是「走的方向」和「梯度」的夹角。

要让坡度 $D_{\mathbf u}f$ **最大**，就让 $\cos\theta$ 最大 = 1，即 $\theta=0$：

> **朝着和梯度同向（$\mathbf u$ 与 $\nabla f$ 平行）走时，上升最快**，此时坡度 $=\lVert\nabla f\rVert$。
> 朝反方向（$\theta=180°$，$\cos\theta=-1$）走时**下降最快**，坡度 $=-\lVert\nabla f\rVert$ → 这就是**负梯度 = 最陡下降**。
> 和梯度**垂直**（$\theta=90°$，$\cos\theta=0$）时坡度为 0 → 沿等高线走，海拔不变（第 6 节）。

这就是梯度下降取 $-\nabla L$ 的严格理由：**在所有方向里，负梯度让损失降得最快。**

> 💡 也解释了「大小」的含义：最陡坡度恰好等于梯度的长度 $\lVert\nabla f\rVert$（1.9 范数）。梯度越长 → 损失面越陡 → 同样的学习率会迈出越大的实际步子。

---

## 6. 梯度垂直于等高线（俯视图直觉）

**等高线** = 函数值相同的点连成的线（地图上的等海拔圈、损失面的「等损失圈」）。

沿等高线走，函数值不变 → 那个方向的方向导数 = 0 → $\nabla f\cdot\mathbf u=0$ → **梯度和等高线方向垂直**。

> 所以俯视损失面时，梯度箭头总是**垂直穿过等高线**、指向数值增大的那侧。下山（减小损失）就沿箭头反方向、垂直往「圈内更低处」切。脚本会用 $f=x^2+y^2$（同心圆等高线）把这条验证出来。

---

## 7. 数值梯度：把 2.2 的数值偏导循环一遍

不会推公式也能算梯度——对**每个分量**做一次中心差分（2.2 学过），拼起来：

$$\big(\nabla f\big)_i\approx\frac{f(\dots,x_i+h,\dots)-f(\dots,x_i-h,\dots)}{2h}\quad(\text{只推第 }i\text{ 个分量，其余冻住})$$

- 有 $n$ 个变量就循环 $n$ 次，得到 $n$ 个偏导，摞成数值梯度向量。
- 深度学习框架做 **gradient check**（验证反向传播算得对不对）就是拿数值梯度和解析梯度比。
- 脚本会验证：数值梯度 ≈ 解析梯度。

> ⚠️ 同 2.1/2.2：$h$ 取 $10^{-5}$ 附近，太大不准、太小舍入误差反而变大。

## 8. 和 LLM 的联系（为什么⭐⭐⭐）

### 8.1 梯度 = 训练的「指南针」

模型 $n$ 个参数、损失 $L(\mathbf w)$ 是一个数。训练一步：

1. 反向传播（2.4）一次性算出所有偏导 → 拼成梯度 $\nabla L$
2. 每个参数反着自己的梯度分量挪一小步：$\mathbf w\leftarrow\mathbf w-\eta\,\nabla L$（2.5）
3. 损失下降一点点，重复亿万次

> 「几千亿参数的模型怎么训练」听起来玄，核心就这一句：**求梯度 → 反着梯度走一步 → 重复。** 梯度是那根始终指向「谷底方向」的指南针。

### 8.2 梯度的「形状」和参数一模一样

梯度是和参数**等长**的向量：某层权重是 $768\times768$ 的矩阵，它的梯度也是 $768\times768$；整个模型全部参数拉直成一根长度 $n$ 的向量 $\mathbf w$，梯度 $\nabla L$ 就是等长的另一根。**参数在哪、梯度就在哪，一一对应。** 优化器（Adam 等，3 章后面）就是拿这根梯度向量去更新参数。

### 8.3 一个最小例子：单神经元的梯度

承接 2.2 的神经元 $\hat y=wx+b$，损失 $L=(\hat y-t)^2$。它有两个参数 $w,b$，梯度是二维向量：

$$\nabla L=\left(\frac{\partial L}{\partial w},\ \frac{\partial L}{\partial b}\right)=\big(2(\hat y-t)x,\ \ 2(\hat y-t)\big)$$

- 反着它走一步：$w\leftarrow w-\eta\cdot2(\hat y-t)x$，$b\leftarrow b-\eta\cdot2(\hat y-t)$。
- 这就是一层网络训练的全部核心；几十亿参数只是把这根梯度向量拉得很长很长。脚本会把「预测→梯度→更新→损失下降」完整跑一遍。

> ⚠️ 梯度告诉你**方向**（往哪走）和**相对陡峭**（哪个参数敏感），但**不告诉你走多远**——步长由学习率 $\eta$ 控制（2.5/2.6）。梯度只是指南针，不是里程表。

### ✅ 参考答案

**A 组**

A1. $\nabla f=(3,4)$，是个**常向量**——不随点变化（因为 $f$ 是线性的，处处一样陡）。

A2. $\nabla f=(2x,2y)$；在 $(3,4)$：$\nabla f=(6,8)$，长度 $\lVert\nabla f\rVert=\sqrt{6^2+8^2}=\sqrt{100}=\mathbf{10}$。

A3. $\nabla f=(yz,\ xz,\ xy)$；在 $(1,2,3)$：$\nabla f=(2\cdot3,\ 1\cdot3,\ 1\cdot2)=(6,3,2)$。

A4. $\nabla f=(2xy,\ x^2+3)$；在 $(1,1)$：$(2\cdot1\cdot1,\ 1+3)=(2,4)$。

**B 组**

B1. $D_{\mathbf u}f=\nabla f\cdot\mathbf u=6\cdot\tfrac35+8\cdot\tfrac45=\tfrac{18}{5}+\tfrac{32}{5}=\tfrac{50}{5}=\mathbf{10}$。
恰好等于 $\lVert\nabla f\rVert=10$ → 因为 $\mathbf u=(6,8)/10$ 正是**梯度方向**，此方向上升最快、坡度最大。

B2. $D_{\mathbf u}f=6\cdot(-\tfrac45)+8\cdot\tfrac35=-\tfrac{24}{5}+\tfrac{24}{5}=\mathbf{0}$。
方向导数为 0 → 这个方向和梯度垂直，**沿等高线走，$f$ 值不变**。

B3. 下降最快 = 负梯度方向的单位向量：$-\nabla f=(-6,-8)$，长度 10，单位化得 $\mathbf u=(-\tfrac35,-\tfrac45)=\mathbf{(-0.6,-0.8)}$（朝原点方向，因为原点是碗底）。

**C 组**

C1. $\hat y=1.5\times2+0.5=\mathbf{3.5}$，$L=(3.5-3)^2=\mathbf{0.25}$。
$\nabla L=\big(2(\hat y-t)x,\ 2(\hat y-t)\big)=\big(2\times0.5\times2,\ 2\times0.5\big)=\mathbf{(2,\ 1)}$。
更新：$w\leftarrow1.5-0.1\times2=\mathbf{1.3}$，$b\leftarrow0.5-0.1\times1=\mathbf{0.4}$。
新 $\hat y=1.3\times2+0.4=3.0$，新 $L=(3.0-3)^2=\mathbf{0}$ ✅ 反着梯度走一步，损失降到 0。
