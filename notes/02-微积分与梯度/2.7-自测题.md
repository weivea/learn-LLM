## 🟢 A 组 · 概念（用自己的话说清楚）

### A1 · 计算图记录什么？

对下面的计算：

```python
product = w * x
prediction = product + b
error = prediction - target
loss = error ** 2
```

回答：

1. 计算图中的节点表示什么？
2. 边表示什么？
3. 哪些是叶子节点，哪些是中间节点？
4. 前向传播除了计算数值，还记录了什么？
5. 为什么计算图通常是有向无环图？

✍️ 你的答案：

1. 节点表示输入、参数、常量或某一步运算产生的值；有些计算图还会把乘法、加法等运算单独画成运算节点。
2. 边表示数据依赖和流动方向，例如 `prediction` 依赖 `product` 和 `b`；反向传播时梯度沿这些边的反方向传递。
3. $w,x,b,target$ 是直接创建、没有父节点的叶子节点；`product`、`prediction`、`error`、`loss` 是运算产生的中间节点。
4. 前向传播还会记录每一步使用的运算、父节点，以及反向计算本地导数所需的中间值。
5. 边从已经得到的输入指向依赖它的结果；一次具体前向执行中，一个结果不能在算出之前又成为自己的依赖，所以展开后的计算依赖通常构成有向无环图。

### A2 · 三种求导方式

分别用一句话解释：

1. 符号求导(symbolic differentiation)
2. 数值求导(numerical differentiation)
3. 自动求导(automatic differentiation)

并回答：

4\. 深度学习训练主要使用哪一种？
5\. gradient check 常使用哪一种？
6\. 自动求导是否等于“不需要知道任何导数规则”？为什么？

✍️ 你的答案：

1. **符号求导**：直接操作数学表达式，按求导规则生成新的解析导数表达式。
2. **数值求导**：给输入加入很小的扰动，用差商近似某一点处的导数。
3. **自动求导**：执行原程序时记录基础运算，再用各运算的本地导数和链式法则计算当前点的数值梯度。
4. 深度学习训练主要使用自动求导，因为它比逐参数数值差分高效，又能适应实际执行的程序和控制流。
5. gradient check 常使用数值求导，把有限差分结果与自动求导结果比较。
6. 不等于。自动求导仍需要为加法、乘法、矩阵乘法、激活函数等基础运算实现本地导数规则，只是规则实现一次后由框架自动组合。

### A3 · 四个动作不要混

分别说明：

1. `forward` 做什么？
2. `loss.backward()` 做什么？
3. `optimizer.step()` 做什么？
4. `optimizer.zero_grad()` 做什么？
5. `zero_grad()` 会不会把模型参数本身变成 0？

✍️ 你的答案：

1. `forward` 根据输入计算预测和损失，同时记录这次实际执行的运算、依赖关系和反向所需的中间值。
2. `loss.backward()` 从损失出发，按逆拓扑顺序应用链式法则，把梯度累积到需要梯度的节点或叶子参数的 `.grad` 中。
3. `optimizer.step()` 读取参数的 `.grad`，按照 SGD、AdamW 等优化规则真正修改参数值。
4. `optimizer.zero_grad()` 清除参数中上一轮遗留的梯度，避免无意地跨训练 step 累积。
5. 不会。它清零的是参数的 `.grad`，不是参数本身的数据。

### A4 · 反向传播的起点

1. 为什么标量损失调用 `backward()` 时，起始梯度是：

   $$
   \frac{\partial L}{\partial L}=1
   $$

2. 如果输出不是标量，而是向量，为什么通常还需要提供一个外部上游梯度？
3. “上游梯度 × 本地导数”与链式法则是什么关系？

✍️ 你的答案：

1. 反向传播需要一个种子梯度。对标量损失而言，它相对自身的变化率恒为 $1$，所以从 $\partial L/\partial L=1$ 开始，后续梯度不会被额外缩放。
2. 向量输出对输入的导数一般是雅可比矩阵，单独调用“反向”并没有唯一的标量起点；外部上游梯度指定了各输出分量怎样加权，使框架计算相应的向量－雅可比积。
3. 若节点 $z=f(u)$ 已收到上游梯度 $\partial L/\partial z$，传给 $u$ 的梯度就是：

    $$
    \frac{\partial L}{\partial u}
    =
    \frac{\partial L}{\partial z}
    \frac{\partial z}{\partial u}
    $$

    前一项是上游梯度，后一项是当前运算的本地导数，这正是链式法则在一个节点上的执行方式。

### A5 · 为什么使用反向模式？

神经网络可能有数十亿参数，但训练目标通常是一个标量损失。

1. 为什么反向模式自动求导适合这种“很多输入、一个输出”的结构？
2. 反向传播与反向模式自动求导是什么关系？
3. 框架是否必须显式创建完整雅可比矩阵？通常怎样避免？

✍️ 你的答案：

1. 反向模式从一个标量损失出发反向遍历一次，就能同时得到损失对所有可训练参数的梯度；正向模式通常要为许多不同的参数方向分别传播，因此不适合参数极多而输出很少的训练结构。
2. 反向传播就是反向模式自动求导在神经网络上的应用：它沿计算图从损失向输入方向传播上游梯度。
3. 不需要。框架通常让每个节点直接计算“上游梯度与本地雅可比的乘积”，即向量－雅可比积(VJP)，从而避免构造和保存巨大的完整雅可比矩阵。

### B2 · 多路径梯度累加

给定：

$$
y=x^2+x=x\cdot x+x
$$

在 $x=3$ 时：

1. 画出或用文字描述 $x$ 影响 $y$ 的三条输入路径。
2. 分别求三条路径对梯度的贡献。
3. 求最终 $\dfrac{dy}{dx}$。
4. 为什么自动求导实现必须使用：

   ```python
   x.grad += contribution
   ```

   而不能总是：

   ```python
   x.grad = contribution
   ```

5. 参数共享为什么会产生同类现象？

✍️ 你的答案：

1. $x$ 有三条输入路径：作为 $x\cdot x$ 的左输入、作为 $x\cdot x$ 的右输入，以及直接作为加法 $+x$ 的输入。
2. 在 $x=3$ 时，三条路径的贡献分别为：

    $$
    \underbrace{x}_{\text{乘法左输入}}=3,
    \qquad
    \underbrace{x}_{\text{乘法右输入}}=3,
    \qquad
    \underbrace{1}_{\text{直接相加}}=1
    $$

3. 最终梯度是所有路径贡献之和：

    $$
    \frac{dy}{dx}=x+x+1=2x+1=7
    $$

4. `+=` 会保留并累加不同路径已经传回的贡献；若使用 `=`，后到的贡献会覆盖先到的贡献，结果可能只剩 $3$ 或 $1$，而不是完整的 $7$。
5. 参数共享意味着同一个参数节点会在多个运算、token 位置或样本中被重复使用，于是损失也会通过多条路径依赖该参数，其最终梯度同样必须汇总所有路径的贡献。

### B3 · 逆拓扑顺序

已知依赖：

```text
w, x → product
product, b → prediction
prediction, target → error
error → loss
```

1. 写出一种合法的前向拓扑顺序。
2. 写出对应的反向执行顺序。
3. 为什么不能在 `prediction` 尚未收齐来自所有下游路径的梯度时，就提前向 `product` 和 `b` 传播？
4. 如果一个节点只被访问一次，反向遍历为什么仍需要 `visited` 集合？

✍️ 你的答案：

1. 一种合法的前向拓扑顺序是：

    ```text
    w, x, b, target, product, prediction, error, loss
    ```

    叶子节点之间的先后可以调整，只要每个中间结果都排在其依赖之后即可。

2. 对应的反向执行顺序是：

    ```text
    loss, error, prediction, product, target, b, x, w
    ```

3. 如果 `prediction` 被多条下游路径使用，提前传播只会把当时收到的部分梯度传给 `product` 和 `b`；后来到达的贡献将无法经过已经执行过的本地反向规则继续传递，导致更上游梯度缺失。逆拓扑顺序保证节点先收齐所有下游贡献，再向父节点传播。
4. 构建拓扑顺序的 DFS 可能沿多条路径多次遇到同一个共享节点。`visited` 集合保证该节点的父节点只需展开一次、该节点也只加入拓扑序一次，从而避免重复执行反向规则和重复累加梯度。

### B4 · 矩阵乘法的反向规则

给定：

$$
Y=XW
$$

形状为：

$$
X:(B,D_\text{in}),\qquad
W:(D_\text{in},D_\text{out}),\qquad
Y:(B,D_\text{out})
$$

从后面收到：

$$
G=\frac{\partial L}{\partial Y}
$$

形状为 $(B,D_\text{out})$。

1. 写出 $\dfrac{\partial L}{\partial X}$。
2. 写出 $\dfrac{\partial L}{\partial W}$。
3. 检查两个结果的形状是否分别与 $X,W$ 一致。
4. 为什么说张量自动求导和标量自动求导的核心原理相同？

✍️ 你的答案：

1. 对 $X$ 的梯度为：

    $$
    \frac{\partial L}{\partial X}=GW^{\mathsf T}
    $$

2. 对 $W$ 的梯度为：

    $$
    \frac{\partial L}{\partial W}=X^{\mathsf T}G
    $$

3. 形状检查：

    $$
    (B,D_\text{out})(D_\text{out},D_\text{in})
    =
    (B,D_\text{in})
    $$

    所以 $GW^{\mathsf T}$ 与 $X$ 形状一致；同时：

    $$
    (D_\text{in},B)(B,D_\text{out})
    =
    (D_\text{in},D_\text{out})
    $$

    所以 $X^{\mathsf T}G$ 与 $W$ 形状一致。

4. 两者都在前向记录依赖和必要的中间值，反向按逆拓扑顺序执行“上游梯度与本地导数的组合”，并累加多路径贡献。区别只是标量的本地乘法规则变成了矩阵或张量运算。

### B5 · 广播的反向传播

给定：

$$
Y=X+b
$$

其中：

$$
X:(B,D),\qquad b:(D,),\qquad Y:(B,D)
$$

1. 前向时 $b$ 发生了什么？
2. 若上游梯度为 $G=\partial L/\partial Y$，$\partial L/\partial X$ 是什么？
3. $\partial L/\partial b$ 为什么不能直接保留 $(B,D)$ 形状？
4. 应沿哪个维度求和？
5. 这与“同一个参数被 batch 中多个样本共享”有什么关系？

✍️ 你的答案：

1. 前向时，同一个 $b$ 在 batch 维被逻辑地扩展并加到 $X$ 的每一行上，得到形状为 $(B,D)$ 的 $Y$。
2. 加法对 $X$ 的本地导数为 $1$，且形状无需还原，因此：

    $$
    \frac{\partial L}{\partial X}=G
    $$

3. $b$ 的实际形状是 $(D,)$，而 $(B,D)$ 的梯度分别来自 $B$ 个样本对同一个 $b$ 的使用。若直接保留该形状，既与 $b$ 不匹配，也没有汇总共享参数的全部贡献。
4. 应沿被广播的 batch 维，即第 $0$ 维求和：

    $$
    \frac{\partial L}{\partial b}
    =
    \sum_{i=1}^{B}G_{i,:}
    $$

    结果形状为 $(D,)$。
5. batch 中每个样本都使用同一个偏置 $b$，因此每个样本都会产生一份梯度贡献；沿 batch 维求和就是把这些共享使用产生的贡献合并到同一个参数梯度中。

## 🔴 C 组 · 代码与 LLM 工程理解

### C1 · 阅读最小自动求导代码

观察乘法节点的反向规则：

```python
def backward():
    left.grad += right.data * out.grad
    right.grad += left.data * out.grad
```

回答：

1. `out.grad` 表示什么？
2. `right.data` 为什么是输出对 `left` 的本地导数？
3. `left.data` 为什么是输出对 `right` 的本地导数？
4. 为什么把本地导数再乘 `out.grad`？
5. 为什么使用 `+=`？

✍️ 你的答案：

1. `out.grad` 表示最终损失对乘法节点输出 `out` 的梯度，即上游传来的 $\partial L/\partial out$。
2. 若 $out=left\times right$，把 `right` 视为常量，则 $\partial out/\partial left=right$，所以其数值就是 `right.data`。
3. 同理，把 `left` 视为常量可得 $\partial out/\partial right=left$，其数值就是 `left.data`。
4. 本地导数只描述 `left` 或 `right` 怎样影响 `out`；再乘 `out.grad` 才能通过链式法则得到它们怎样影响最终损失：

    $$
    \frac{\partial L}{\partial left}
    =
    \frac{\partial L}{\partial out}
    \frac{\partial out}{\partial left}
    $$

5. 同一个节点可能通过多条下游路径影响损失，`+=` 能汇总所有路径的贡献；使用 `=` 会覆盖之前已经收到的贡献。

### C2 · 找出训练循环中的错误

下面的训练循环有一个会影响训练结果的问题：

```python
for batch in data_loader:
    logits = model(batch.tokens)
    loss = cross_entropy(logits, batch.targets)
    loss.backward()
    optimizer.step()
```

1. 缺少哪一步？
2. 不修复会发生什么？
3. 给出一种正确顺序。
4. 如果故意做 gradient accumulation，应该怎样安排清零与更新？
5. gradient accumulation 与忘记清零的本质区别是什么？

✍️ 你的答案：

1. 缺少 `optimizer.zero_grad()`，也就是在不需要跨 step 累积时清除旧梯度。
2. 参数的 `.grad` 默认累加；若不清零，当前 batch 的梯度会叠加以前 step 的旧梯度，`optimizer.step()` 将使用非预期的总和更新参数，导致更新尺度和方向失控。
3. 一种正确顺序是：

   ```python
   for batch in data_loader:
       optimizer.zero_grad()
       logits = model(batch.tokens)
       loss = cross_entropy(logits, batch.targets)
       loss.backward()
       optimizer.step()
   ```

4. 若每 $K$ 个 micro-batch 更新一次，可在累积窗口开始前清零，每个 micro-batch 都前向并反向，通常将 loss 除以 $K$ 以保持平均梯度尺度；收齐 $K$ 份梯度后再 `step()`，然后开始下一窗口：

   ```python
   optimizer.zero_grad()
   for step, batch in enumerate(data_loader, start=1):
       logits = model(batch.tokens)
       loss = cross_entropy(logits, batch.targets) / accumulation_steps
       loss.backward()

       if step % accumulation_steps == 0:
           optimizer.step()
           optimizer.zero_grad()
   ```

   实际代码还要处理最后不足 $K$ 个 micro-batch 的尾部窗口。

5. gradient accumulation 是有计划地规定累积窗口、调整 loss 尺度并只在窗口边界更新；忘记清零则没有明确边界，旧梯度会意外跨越本应独立的优化 step。

### C3 · 动态图与图释放

1. PyTorch 的动态计算图是什么时候建立的？
2. 为什么下一次训练 step 通常需要重新前向建图？

5. `retain_graph=True` 应该无条件使用吗？为什么？

✍️ 你的答案：

1. PyTorch 在程序实际执行每一次前向运算时动态建立计算图，因此不同输入触发的 Python 分支可以产生不同的图。
2. `backward()` 通常会释放图中供反向使用的中间状态，而参数也已被优化器更新；下一 step 必须用新参数和新 batch 重新前向，得到新的损失和与本次执行对应的新图。
3. 默认释放可以及时回收中间激活等内存；普通训练只需要沿一张图反向一次，继续保留没有收益。
4. 长期保留图会让中间激活及其引用无法释放，导致内存或显存持续增长，最终可能 OOM；还可能意外把多个 step 的历史串成更大的图。
5. 不应该。只有确实需要对同一张图执行多次反向等特殊场景才使用 `retain_graph=True`；普通训练使用它只会增加内存占用，并可能掩盖计算图生命周期设计上的问题。

### C4 · `requires_grad`、`no_grad` 与 `detach`

分别说明：

1. 参数设置 `requires_grad=True` 的含义。
2. 推理为什么通常使用 `no_grad()`？
3. `detach()` 会切断什么？
4. 误用 `detach()` 为什么可能使一部分模型学不到？
5. 为什么反向可能依赖前向时保存的旧值，从而使某些原地操作不安全？

✍️ 你的答案：

1. `requires_grad=True` 表示框架应追踪与该张量相关、需要求导的运算，使反向传播能够计算损失对它的梯度；叶子参数的结果通常存入 `.grad`。
2. 推理不需要反向传播，使用 `no_grad()` 可避免建立梯度历史和保存反向所需的中间值，从而减少显存、内存和额外开销。
3. `detach()` 返回与原张量共享数据但不再连接原梯度历史的张量，切断的是后续计算返回原上游计算图的梯度路径。
4. 若在本应端到端训练的路径中误用 `detach()`，梯度会在该处停止，更前面的模块或参数收不到这条损失路径的梯度，因而无法从它学习。
5. 乘法、激活、归一化等反向规则可能需要前向时的输入、输出或统计量。若原地操作在反向前改写这些值，本地导数会基于错误的新值；框架因此会用版本检查报错或限制这类操作。

### C5 · LLM 的一条梯度路径

用下面的模块描述一次 LLM 训练中的前向和反向顺序：

```text
embedding
Transformer block 1
...
Transformer block N
output projection
cross entropy loss
```

回答：

1. 前向顺序是什么？
2. 反向顺序是什么？
3. 调用一次 `loss.backward()` 最终会得到哪些量？
4. 为什么一个参数的最终梯度会包含多个 token、多个样本的贡献？
5. 残差连接中的多条路径在反向时怎样合并？

✍️ 你的答案：

1. 前向顺序为：token IDs 经 `embedding` 得到表示，依次通过 Transformer block 1 到 block N，再经 `output projection` 得到 logits，最后与标签计算 `cross entropy loss`。
2. 反向顺序与依赖方向相反：从交叉熵开始，经过输出投影、block N 一直反传到 block 1 和 embedding。
3. 一次 `loss.backward()` 会计算损失对图中所有相关中间节点的梯度，并把所有需要梯度的叶子参数的结果累积到 `.grad`；它不会自行更新参数。
4. 同一组权重会被 batch 中多个样本、序列中多个 token 位置重复使用，而最终损失通常又聚合了这些位置的损失，所以参数梯度包含所有对应使用路径的贡献之和；若 loss 取均值，整体还会带相应的平均缩放。
5. 残差连接让同一上游激活通过恒等分支和子层分支共同影响损失；反向时两条路径分别产生梯度，并在共享激活处相加后继续向前传播。

### C6 · 自动求导与显存

1. 为什么训练时需要保存中间激活？
2. 为什么训练显存通常明显高于纯推理？
3. gradient checkpointing 保存了什么、丢弃了什么？
4. 反向时如何补回被丢弃的中间结果？
5. 它用什么代价换取显存下降？

✍️ 你的答案：

1. 许多本地反向规则需要前向时的输入、输出、掩码或统计量，例如乘法需要另一侧输入，ReLU 需要激活符号；保存中间激活可在反向时直接使用这些值。
2. 纯推理只需完成前向，很多临时激活用完即可释放；训练还需保存反向所需的激活、参数梯度和优化器状态，因此显存通常明显更高。
3. gradient checkpointing 只保存选定的检查点激活和必要边界状态，丢弃检查点之间原本要为反向保留的部分中间激活。
4. 反向到某个区段时，从最近的检查点重新执行该区段的前向运算，重建所需中间结果，然后再计算该区段的梯度。
5. 它用额外的前向重计算和更长的训练时间换取更低的激活显存占用。

### C7 · 综合解释

请用不超过 8 句话向另一名程序员解释：

1. 框架为什么能自动算梯度？
2. 计算图在前向时记录什么？
3. 反向时为什么要逆拓扑遍历？
4. 链式法则怎样体现在每个节点？
5. 多路径梯度怎样处理？
6. `backward()`、`step()`、`zero_grad()` 怎样分工？
7. 为什么反向模式适合训练神经网络？
8. 自动求导为什么会消耗额外显存？

✍️ 你的答案：

1. 框架能自动算梯度，是因为每种基础运算都预先实现了本地反向规则，复杂模型只是这些运算的组合。
2. 前向执行时，计算图会记录运算结果、父节点、依赖关系以及反向所需的中间值。
3. 反向必须按逆拓扑顺序执行，才能保证一个节点先收齐所有下游梯度，再向它的父节点传播。
4. 每个节点都把收到的上游梯度乘以自己的本地导数，这就是链式法则的一步。
5. 同一节点通过多条路径影响损失时，各路径的梯度贡献必须相加而不能互相覆盖。
6. `backward()` 计算并累积梯度，`step()` 根据梯度更新参数，`zero_grad()` 清除不再需要的旧梯度。
7. 神经网络有大量参数但通常只有一个标量损失，反向模式一次反传就能得到所有参数的梯度，因此特别合适。
8. 自动求导需要保存供反向使用的计算图和中间激活，所以训练会消耗额外显存。
