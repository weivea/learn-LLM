## 🟢 A 组 · 概念（用自己的话说清楚）

### A1 · 四个量不要混

梯度下降的一步为：

$$
\Delta\boldsymbol{\theta}
=
-\eta\nabla L
$$

分别解释：

1. 参数误差 $e_k=\theta_k-\theta^*$ 是什么？
2. 梯度 $\nabla L$ 是什么？
3. 学习率 $\eta$ 是什么？
4. 更新量 $\Delta\boldsymbol{\theta}$ 是什么？
5. 为什么“学习率为 $0.1$”不等于“参数每步移动 $0.1$”？

✍️ 你的答案：

1. 参数误差 $e_k=\theta_k-\theta^*$ 是什么？\
   $e_k$ 是第 $k$ 步参数相对于最优参数的**带符号误差**；符号表示参数在最优点的哪一侧，绝对值表示离最优点多远。
2. 梯度 $\nabla L$ 是什么？\
   $\nabla L$ 是损失对所有参数的偏导数组成的向量，指向损失上升最快的方向；每个分量表示损失对对应参数的局部敏感程度。
3. 学习率 $\eta$ 是什么？\
   $\eta$ 是用于缩放更新方向的正超参数，控制一次参数更新的整体尺度。
4. 更新量 $\Delta\boldsymbol{\theta}$ 是什么？\
   它是当前这一步所有参数的实际变化量：

   $$
   \Delta\boldsymbol{\theta}
   =
   \boldsymbol{\theta}_{k+1}-\boldsymbol{\theta}_k
   =
   -\eta\nabla L
   $$

5. 为什么“学习率为 $0.1$”不等于“参数每步移动 $0.1$”？\
   因为实际更新量还取决于梯度：

   $$
   \lVert\Delta\boldsymbol{\theta}\rVert
   =
   \eta\lVert\nabla L\rVert
   $$

> ✅ **订正：**原答案的核心理解正确。需要说得更准确：梯度不是“参数的变化方向和快慢”，而是损失上升最快的方向及其局部变化率；真正的参数更新方向是**负梯度方向**。

### A2 · 太小、合适、太大

分别说明下面三种学习率在训练损失曲线上可能出现的现象：

1. 学习率太小
2. 学习率合适
3. 学习率太大

为什么不能简单地说“学习率越大，模型学得越快”？

✍️ 你的答案：
1. 学习率太小\
   参数每步更新很小，loss 通常稳定但下降缓慢，可能在给定训练预算内来不及学好。
2. 学习率合适
   loss 整体较快下降，允许 mini-batch 带来的小幅波动，并能达到较低损失。
3. 学习率太大
   参数可能反复越过较优区域，loss 剧烈振荡、持续上升，甚至出现 `inf` / `NaN`。

为什么不能简单地说“学习率越大，模型学得越快”？\
因为学习率只在稳定范围内增大时才可能加快训练；学习率太大会使单步参数调整过大，越过稳定区域，出现振荡甚至发散。

> ✅ **订正：**判断正确。还要区分“偏大但仍稳定”和“超过稳定上限”：前者可能振荡收敛，后者才会发散。

### A3 · 越过最低点不等于发散

对 $L(w)=(w-3)^2$：

$$
e_{k+1}=(1-2\eta)e_k
$$

当 $\eta=0.9$ 时：

$$
e_{k+1}=-0.8e_k
$$

1. 负号表示什么？
2. $0.8$ 表示什么？
3. 为什么参数会来回越过最低点，却仍然能够收敛？

✍️ 你的答案：
1. 负号表示什么？\
   表示新旧误差符号相反，即参数每一步都会从最低点的一侧跳到另一侧，不是固定只从右边跳到左边。
2. $0.8$ 表示什么？\
   表示新误差的绝对值是旧误差的 $80\%$，即到最低点的距离每一步缩小 $20\%$。
3. 为什么参数会来回越过最低点，却仍然能够收敛？\
   因为误差符号每步改变，但绝对值持续按 $0.8$ 倍缩小：

   $$
   |e_{k+1}|=0.8|e_k|
   $$

   所以参数会来回越过最低点，同时越来越靠近最低点，这叫振荡收敛。

> ⚠️ **订正：**原方向正确，但 `$0.8$` 必须明确为“保留旧距离的 $80\%$”，而不是只说“距离缩短了”。

### A4 · 没有万能学习率

至少写出三个原因，解释为什么同一个学习率不能保证适合所有模型、损失函数和训练任务。

✍️ 你的答案：\
1. 损失曲面的曲率不同：曲率越大，稳定学习率的上限通常越小。
2. batch size 不同：梯度噪声和一次更新包含的信息量会改变。
3. 训练阶段不同：训练初期、中期和后期对稳定性与更新精度的需求不同。

> ✅ **订正：**三个原因都正确。除此之外，梯度尺度、优化器、模型参数化方式和数值精度也会影响合适的学习率。

### B4 · 从曲率求稳定范围

给定：

$$
L(w)=5(w-4)^2
$$

1. 把它写成：

   $$
   L(w)=\frac a2(w-w^*)^2
   $$

   求 $a$ 和 $w^*$。

2. 推导误差更新式：

   $$
   e_{k+1}=(1-a\eta)e_k
   $$

3. 求学习率的稳定范围。
4. 判断 $\eta=0.15$ 时是单调收敛、振荡收敛还是发散。

✍️ 你的答案：
1. $a=10$ ；  $w^*=4$。

2. 推导误差更新式
$$
\left.\frac{dL}{dw}\right|_{w=w_k}=a(w_k-w^*) \\

\begin{aligned}
w_{k+1}
&=w_k-\eta a(w_k-w^*)\\
e_{k+1}
&=w_{k+1}-w^*\\
&=w_k-\eta a(w_k-w^*)-w^*\\
&=(1-a\eta)(w_k-w^*)\\
&=(1-a\eta)e_k
\end{aligned}
$$

3. 学习率的稳定范围

$$
|1-a\eta|<1 \\
0<\eta<\frac{2}{a} \\
0<\eta<0.2
$$

4. 判断 $\eta=0.15$ 时是单调收敛、振荡收敛还是发散。
误差系数为：

$$
1-a\eta=1-10\times0.15=-0.5
$$

其绝对值小于 $1$，所以 $\eta=0.15$ 时是振荡收敛。

> ⚠️ **订正：**$a=10$、$w^*=4$、稳定范围和最终判断都正确。原推导中把减号误写成了等号：应为 $w_k-w^*$，不是 $w_k=w^*$。

### B5 · 多维中的最大曲率

给定：

$$
L(x,y)=\frac12(x^2+20y^2)
$$

两个方向的曲率分别是 $1$ 和 $20$。

1. 根据 $\eta<2/\lambda_{\max}$ 求整体稳定学习率的上限。
2. 判断 $\eta=0.08$ 是否在稳定范围。
3. 判断 $\eta=0.12$ 是否在稳定范围。
4. 为什么不能只看较平缓的 $x$ 方向来选择学习率？

✍️ 你的答案：\
整体稳定学习率必须满足：

$$
0<\eta<\frac{2}{20}=0.1
$$

因此 $\eta=0.08$ 在稳定范围内，$\eta=0.12$ 不在稳定范围内。

不能只看较平缓的 $x$ 方向，因为同一个学习率也用于曲率更大的 $y$ 方向。若学习率超过 $y$ 方向的稳定上限，该方向的误差会被逐步放大，最终使整体损失发散。

> ✅ **订正：**数值判断全部正确。更准确的说法是“最大曲率方向”，而不是“陡峭梯度的维度”。

## 🔴 C 组 · LLM 训练与工程理解

### C1 · Warmup 与 decay

一个 LLM 训练配置使用：

```text
前 2,000 step：学习率从 0 逐渐升到峰值
之后：学习率按 cosine 曲线逐渐下降
```

1. 前 2,000 step 属于什么阶段？
2. 为什么训练刚开始不直接使用峰值学习率？
3. 后期为什么要逐渐降低学习率？
4. 用一句话概括 warmup 和 decay 各自解决的问题。

✍️ 你的答案：

前 2,000 step属于 Warmup阶段，逐渐提升学习率
为什么训练刚开始不直接使用峰值学习率?\
训练初期 AdamW 的一阶、二阶统计量仍在建立，模型激活和梯度也可能不稳定；直接使用峰值学习率容易造成过大的参数更新甚至发散。\
后期为什么要逐渐降低学习率？\
训练后期通常已经接近较优区域，逐渐降低学习率可以减少震荡，用更小的步子精细收敛。

用一句话概括 warmup 和 decay 各自解决的问题。\
Warmup 通过在训练初期逐渐增大学习率来避免参数更新不稳定甚至发散，decay 则通过在训练后期逐渐减小学习率来减少最优点附近的震荡并实现更精细的收敛。

> ⚠️ **订正：**整体理解正确，但“训练后期”不等于“微调阶段”。这里的 decay 可以发生在一次预训练或微调任务内部的后半段。

### C3 · backward、optimizer 与 scheduler

观察伪代码：

```python
optimizer.zero_grad()
logits = model(tokens)
loss = cross_entropy(logits, targets)
loss.backward()
optimizer.step()
scheduler.step()
```

分别说明：

1. `loss.backward()` 做什么？
2. `optimizer.step()` 做什么？
3. `scheduler.step()` 做什么？
4. `scheduler.step()` 会重新计算梯度吗？
5. 如果使用 AdamW，是否就不需要设置学习率？为什么？

✍️ 你的答案：
1. `loss.backward()` 做什么？
   - 通过反向传播计算损失对各参数的梯度，并保存到参数的 `.grad` 中。
2. `optimizer.step()` 做什么？
   - 读取当前梯度，按照 AdamW 等优化规则和当前学习率真正更新模型参数。
3. `scheduler.step()` 做什么？
   - 按预定调度规则修改之后训练 step 使用的学习率。
4. `scheduler.step()` 会重新计算梯度吗？
   - 不会。梯度由 `loss.backward()` 计算。
5. 如果使用 AdamW，是否就不需要设置学习率？为什么？
   - 仍然需要。AdamW 会根据梯度的一阶、二阶统计量调整不同参数的相对更新尺度，但仍需基础学习率控制整体更新尺度。

> ❌ **重点订正：**原答案把 `optimizer.step()` 和 `scheduler.step()` 的职责写反了。务必记住：
>
> - `backward()`：**算梯度**
> - `optimizer.step()`：**改参数**
> - `scheduler.step()`：**改后续学习率**

### C4 · batch size 改变后

训练从：

```text
每卡 batch=4，GPU 数=8，梯度累积=2
```

改为：

```text
每卡 batch=8，GPU 数=8，梯度累积=4
```

1. 修改前的 global batch size 是多少？
2. 修改后的 global batch size 是多少？
3. global batch size 放大了多少倍？
4. 为什么此时应该重新检查学习率和 warmup，而不是无条件沿用旧配置？
5. “batch 变大，学习率就一定按同样倍数放大”为什么不是严格定律？

✍️ 你的答案：
1. 修改前的 global batch size 是：$4\times 8\times 2=64$。
2. 修改后的 global batch size 是：$8\times 8\times 4=256$。
3. global batch size 放大了 $256\div 64=4$ 倍。
4. global batch size 变大后，梯度噪声和每次参数更新所包含的信息量都会改变，而且相同的 warmup step 会对应更多训练样本，因此原来的学习率和 warmup 配置不一定仍然合适，需要重新验证。
5. 学习率与 batch size 的线性缩放只是一种经验规则，合适的学习率还受模型规模、损失曲率、优化器、数据和训练阶段等因素影响，所以 batch 变大后，学习率不一定能按相同倍数增大。

### C5 · 综合解释

请用不超过 5 句话向另一名程序员解释：

1. 梯度负责什么？
2. 学习率负责什么？
3. 为什么太小和太大都不好？
4. 为什么 LLM 常用 warmup + decay？
5. AdamW 与学习率是什么关系？

✍️ 你的答案：
1. 梯度描述损失对各参数的变化有多敏感，并指出损失上升最快的方向，优化器据此选择更新方向。
2. 学习率控制每次参数更新的整体步幅，即把优化器算出的更新方向缩放到合适大小。
3. 学习率太小会使训练收敛缓慢，太大则可能越过较优位置并引起震荡甚至发散。
4. LLM 常用 warmup 在训练初期逐渐增大学习率以稳定更新，再用 decay 在后期减小步幅以实现更精细的收敛。
5. AdamW 会根据梯度统计调整各参数的相对更新幅度并解耦权重衰减，但仍需学习率控制整体更新尺度。

> ✅ **批改：**五句话准确、完整，已经能把学习率与 LLM 训练联系起来。
