## 一句话总结

**转置 = 把矩阵「沿主对角线翻一下」，行变列、列变行。**
形状从 `(m, n)` 变成 `(n, m)`，元素位置 `A[i][j]` 跑到 `Aᵀ[j][i]`。

$$(A^\top)_{ij} = A_{ji}$$

> ⚠️ 转置**只搬动位置，不改变任何数值**，也不做加减乘。它就是「换个方向摆」。

---

## 1. 直观：沿对角线翻折

$$A=\begin{bmatrix}1&2&3\\4&5&6\end{bmatrix}_{2\times3}
\qquad\Longrightarrow\qquad
A^\top=\begin{bmatrix}1&4\\2&5\\3&6\end{bmatrix}_{3\times2}$$

- 原来的**第 1 行** `[1,2,3]` → 变成 `Aᵀ` 的**第 1 列**
- 原来的**第 1 列** `[1,4]` → 变成 `Aᵀ` 的**第 1 行**
- 形状 `(2,3)` → `(3,2)`，行列数对调

**主对角线**（左上到右下 `A[0][0], A[1][1], …`）上的元素**待在原地不动**，其余元素以它为轴镜像交换。

---

## 2. 三步手算法

以 `A[i][j] → Aᵀ[j][i]` 为唯一规则：

1. 写出新形状：原来 `(m, n)`，新的就是 `(n, m)`。
2. 把原矩阵的**每一行**，竖着抄下来当**新矩阵的每一列**。
3. 检查：原来的 `A[i][j]`，应该出现在新矩阵的第 `j` 行第 `i` 列。

$$\begin{bmatrix}a&b\\c&d\\e&f\end{bmatrix}^\top=\begin{bmatrix}a&c&e\\b&d&f\end{bmatrix}$$

---

## 3. 四条必记性质

| 性质 | 公式 | 说明 |
|------|------|------|
| 转两次回到原样 | $(A^\top)^\top = A$ | 翻一下再翻回来 |
| 加法可拆 | $(A+B)^\top = A^\top + B^\top$ | 分别转置再相加 |
| **乘法要换序**（最易错） | $(AB)^\top = B^\top A^\top$ | 顺序**反过来**，不是 $A^\top B^\top$ |
| 数乘不受影响 | $(cA)^\top = cA^\top$ | 标量提到外面 |

> 🔑 第 3 条 **$(AB)^\top = B^\top A^\top$** 记不住就想形状：
> `A(m,n)·B(n,p)` 结果 `(m,p)`，转置后要 `(p,m)`；
> 只有 `Bᵀ(p,n)·Aᵀ(n,m)` 才拼得出 `(p,m)`，`AᵀBᵀ` 的形状根本对不上。

**对称矩阵**：若 $A^\top = A$（转置等于自己），称为对称矩阵，比如协方差矩阵、注意力分数矩阵在某些情况下。

---

## 4. 为什么 LLM 里到处是转置（为什么⭐⭐⭐）

### 4.1 注意力：`scores = Q @ K.T` 的那个 `.T`

这是你在 Transformer 代码里见到转置最多的地方：

$$\text{scores} = \frac{Q\,K^\top}{\sqrt{d_k}}$$

- `Q` 形状 `(seq, d_k)`：每行是一个 query 向量
- `K` 形状 `(seq, d_k)`：每行是一个 key 向量
- 想让**每个 query 和每个 key 两两做点积**，需要 `(seq, d_k) @ (d_k, seq)`
- `K` 是 `(seq, d_k)`，乘不了；`K.T` 变成 `(d_k, seq)`，**内层维度对上了**，`Q @ K.T` 结果 `(seq, seq)` 正是注意力分数矩阵

> 一句话：**`.T` 是为了让矩阵乘法的维度对齐**（回忆 1.4：内层维度必须相等）。它把 K 的每一行「立起来」变成列，好让 Q 的行去和它做点积（1.5）。

### 4.2 权重矩阵：`y = x @ W.T`

PyTorch 的 `nn.Linear` 内部权重 `W` 存成 `(out, in)`，前向传播是 `y = x @ W.T + b`，转置也是为了对齐维度。

### 4.3 反向传播里的梯度

链式法则求梯度时，经常出现 `dL/dx = dL/dy @ W`、`dL/dW = x.T @ dL/dy` 这类式子，转置是把维度重新对齐好做乘法。现在混个脸熟即可。

## 5. NumPy / PyTorch 里的坑

| 写法 | 作用 | 备注 |
|------|------|------|
| `A.T` | 二维转置，最常用 | 一维向量用 `.T` **没效果**（见下） |
| `np.transpose(A)` | 同 `A.T` | 高维可指定 `axes` |
| `A.transpose(0, 2, 1)` | 高维张量按指定轴序重排 | batch 场景常用 |
| `A.swapaxes(-1, -2)` | 只交换最后两个轴 | batch 转置的安全写法 |
| `A.mT`（新版本） | 只转最后两维 | 专为批量矩阵设计 |

三个高频坑：

1. **一维向量 `.T` 无效**：`np.array([1,2,3]).T` 还是 `[1,2,3]`，shape 仍是 `(3,)`。
   一维没有「行列」概念。想变列向量要先升维：`a.reshape(-1, 1)` 得到 `(3,1)`。

2. **带 batch 千万别用 `.T`**：一个 `(batch, seq, d)` 张量，`.T` 会把所有轴倒过来变成 `(d, seq, batch)`，全乱套。
   正确做法是只翻最后两维：`x.transpose(0, 2, 1)` 或 `x.swapaxes(-1, -2)`，所以注意力代码里常写 `K.transpose(-2, -1)`。

3. **转置是「视图(view)」不是拷贝**：`A.T` 通常和 `A` 共享同一块内存，只是换了读法（改变 stride）。改 `A.T` 会影响 `A`。需要独立副本用 `A.T.copy()`。

## 6. 动手验证

```bash
.venv/bin/python examples/1_6_transpose.py
```

重点看：

- `(2,3)` 转置后变 `(3,2)`，行列对调
- `(Aᵀ)ᵀ == A`，`(AB)ᵀ == BᵀAᵀ` 而 `AᵀBᵀ` 形状对不上
- 一维向量 `.T` 无效，`(batch,seq,d)` 用 `.T` 会翻乱、用 `swapaxes` 才对
- 注意力里 `Q @ K.T` 如何靠转置把维度对齐成 `(seq, seq)`
