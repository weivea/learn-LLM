# 1.3 张量 / 维度(dim, axis)

> 板块：线性代数 · 重要度 ⭐⭐⭐
> 目标：能解释 `(batch, seq_len, d_model)` 每一维
> 配套脚本：[`examples/1_3_tensor.py`](../../examples/1_3_tensor.py)
> 配套资料：[NumPy 基础](https://numpy.org/doc/stable/user/absolute_beginners.html)

---

## 一句话总结

**张量 = 数排成 N 维的数组，是标量/向量/矩阵的统称。** `ndim` 是维数，`shape` 里数字的个数就是 `ndim`，每个数字对应一个 **axis（轴 / dim）**。

---

## 1. 张量就是「矩阵再往上叠」

维度一层层往上加，都是同一个东西的不同 `ndim`：

| 对象 | ndim | shape 例子 | 直觉 |
|------|:----:|-----------|------|
| 标量 scalar | 0 | `()` | 一个数 |
| 向量 vector | 1 | `(3,)` | 一排数 |
| 矩阵 matrix | 2 | `(2, 3)` | 一网格数 |
| 张量 tensor | 3 | `(2, 3, 4)` | 一摞矩阵 |
| 张量 tensor | 4 | `(2, 3, 4, 5)` | 一摞「一摞矩阵」 |

> **ndim = 定位一个元素需要几个下标。** 3 维张量取值要写 `t[i, j, k]`，三个下标。
> `ndim = len(shape)`，`size = shape 各数字相乘`（元素总数）。

---

## 2. axis / dim 到底指哪个方向（核心）

**`shape` 从左到右，就是 axis 0, 1, 2, …**

```
t.shape = (2, 3, 4)
           ↑  ↑  ↑
        axis0 axis1 axis2   (也可写成 axis=-3, -2, -1)
```

- `axis=0` 是**最外层**，最先被「剥开」：`t[0]` 就是剥掉 axis0 后剩下的 `(3, 4)`。
- `axis=-1` 是**最内层**，就是最里面那一排数。
- PyTorch 里叫 `dim`，NumPy 里叫 `axis`，**是同一个东西**。

---

## 3. 沿某个 axis 做运算：那一维会被「压掉」

这是最容易绕晕、也最常用的点。规则一句话：

> **`sum(axis=k)` = 把第 k 维求和吃掉，`shape` 里去掉第 k 个数。**

```
M.shape = (2, 3)
M.sum(axis=0) -> (3,)   # 压掉 axis0（2 那一维），沿"行"方向纵向相加
M.sum(axis=1) -> (2,)   # 压掉 axis1（3 那一维），沿"列"方向横向相加
```

- 想保留那一维为长度 1，用 `keepdims=True`：`(2, 3).sum(axis=1, keepdims=True) -> (2, 1)`，方便后续**广播**（见 1.7）。
- 同理 `mean / max / softmax` 都要指定 `axis`，比如 softmax 通常沿 `axis=-1`（对每个 token 的词表分布归一化）。

### 3 维及以上：规则完全一样

把 `(2, 3, 4)` 想成 **「2 张 3×4 的矩阵叠起来」**，`sum(axis=k)` 依旧是「删掉 shape 第 k 个数」：

| 操作 | 结果 shape | 直觉 | 手验 |
|------|:---------:|------|------|
| `sum(axis=0)` | `(3, 4)` | 2 张矩阵**对应位置相加**，叠成 1 张 | `t[0,0,0] + t[1,0,0]` |
| `sum(axis=1)` | `(2, 4)` | 每张矩阵内部**把 3 行相加**（纵向压行） | `t[0,0,0]+t[0,1,0]+t[0,2,0]` |
| `sum(axis=2)` | `(2, 3)` | 每行的 **4 个数横向相加**（压列） | `t[0,0,0]+…+t[0,0,3]` |
| `sum(axis=(1,2))` | `(2,)` | 一次压两维，只剩 batch，每张矩阵总和 | — |

> **通用心法（N 维通用）**：固定住**其它所有下标**，只让第 k 个下标从头扫到尾，把扫过的元素加起来。
> 结果元素 `out[i, j] = t[i, 0, j] + t[i, 1, j] + …`（`axis=1` 时只有中间下标在动）。

---

## 4. 增 / 减 / 换维度（读代码高频操作）

| 操作 | 作用 | 例子 |
|------|------|------|
| `reshape(...)` | 只改「怎么看」，不改数据，元素总数不变 | `(12,) -> (3, 4)` |
| `x[:, None]` / `np.newaxis` | 加一个长度 1 的新维 | `(3,) -> (3, 1)` |
| `squeeze()` | 去掉所有长度为 1 的维 | `(3, 1) -> (3,)` |
| `transpose(...)` / `permute` | 调换维度**顺序** | `(2,3,4) -> (2,4,3)` |

> ⚠️ `(3,)`（1 维向量）≠ `(3, 1)`（2 维列）≠ `(1, 3)`（2 维行）。值可能一样，`ndim` 不同，写代码常踩坑。

---

## 5. 和 LLM 的联系（为什么⭐⭐⭐）

Transformer 里流动的数据，几乎都是这个 3 维张量：

$$\texttt{hidden.shape} = (\underbrace{\text{batch}}_{axis0},\ \underbrace{\text{seq\_len}}_{axis1},\ \underbrace{\text{d\_model}}_{axis2})$$

| axis | 名字 | 含义 | 例值 |
|:----:|------|------|:----:|
| 0 | batch | 一次并行喂进几句话 | 8 |
| 1 | seq_len | 每句话几个 token | 128 |
| 2 | d_model | 每个 token 用几维向量表示 | 768 |

- `hidden[0]` → `(128, 768)`：第 0 句话，退回成一个矩阵（1.2 学的）。
- `hidden[0, 5]` → `(768,)`：第 0 句第 5 个 token，退回成一个向量（1.1 学的）。

**一句话 = 矩阵；一批句子 = 3 维张量。** 看懂这一维一维，注意力机制的 shape 变换才不发懵。
