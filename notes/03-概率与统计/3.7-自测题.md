## 1 · 概率加权平均（20 分）

某个非均匀骰子的分布为：

| 点数 $x$ | 1 | 2 | 3 | 4 | 5 | 6 |
|---:|---:|---:|---:|---:|---:|---:|
| $P(X=x)$ | 0.05 | 0.10 | 0.15 | 0.20 | 0.20 | 0.30 |

1. 验证它是合法的概率分布。
2. 计算 $\mathbb E[X]$。
3. 为什么不能直接使用 $(1+2+3+4+5+6)/6$？
4. 骰子能否掷出期望值？这是否影响期望的合法性？
5. 用一句话解释期望的直觉含义。

✍️ 你的答案：\
1.  每一个概率都`0<p<=1`, 概率之合为1，是合法的概率分布
2.  $\mathbb E[X] = 1*0.05 + 2*0.10+3*0.15+4*0.20+5*0.20+6*0.30 = 4.3$
3.  因为它不能够反映数值的概率分布，它只能表示平均概率分数的数值期望
4.  按照当前的概率分布，骰子无法掷出，期望是一个计算值，并不要求它必须等于某一个可能的值，是合法的
5.  期望表示大量重复试验时结果的长期平均

> ⚠️ **批改第 1 题（19 / 20）**：期望计算、长期平均直觉和“期望不必是可能取值”均正确。两处表述需要收紧：
>
> 1. 合法概率的一般范围是 $0\le p\le1$，概率可以等于 $0$；本题所有概率恰好都大于 $0$，所以不影响合法性判断。
> 2. 不能使用普通平均的直接原因是六个点数**不等概率**。$(1+\cdots+6)/6$ 等价于错误地给每个点数都分配 $1/6$ 的权重，而本题必须使用实际概率加权。

## 2 · 函数期望与线性性质（20 分）

仍使用第 1 题的骰子。

1. 计算 $\mathbb E[X^2]$。
2. 计算 $(\mathbb E[X])^2$。
3. 本题是否满足 $\mathbb E[X^2]=(\mathbb E[X])^2$？一般能否把函数直接移到期望外？
4. 不重新逐项展开，使用线性性质计算 $\mathbb E[2X+3]$。
5. $\mathbb E[X+Y]=\mathbb E[X]+\mathbb E[Y]$ 是否要求 $X,Y$ 独立？$\mathbb E[XY]=\mathbb E[X]\mathbb E[Y]$ 是否总成立？

✍️ 你的答案：
1. $\mathbb E[X^2]=20.8$
2. $(\mathbb E[X])^2=18.49$
3. 不满足$\mathbb E[X^2]=(\mathbb E[X])^2$，一般不能直接展开
4. $\mathbb E[2X+3]=11.55$
5. $\mathbb E[X+Y]=\mathbb E[X]+\mathbb E[Y]$ 不要求 $X,Y$ 独立？$\mathbb E[XY]=\mathbb E[X]\mathbb E[Y]$ 不是总成立？

> ⚠️ **批改第 2 题（16 / 20）**：$\mathbb E[X^2]=20.8$、$(\mathbb E[X])^2=18.49$ 以及二者通常不相等均正确。需要纠正和补全：
>
> 1. 第 4 问有一处算术错误：
>
>    $$
>    \mathbb E[2X+3]
>    =
>    2\mathbb E[X]+3
>    =
>    2(4.3)+3
>    =
>    11.6
>    $$
>
>    不是 $11.55$。
> 2. “一般不能直接展开”应准确表述为：一般不能把**非线性函数移到期望外**，即 $\mathbb E[g(X)]\ne g(\mathbb E[X])$。
> 3. 你的第 5 问结论正确：期望的加法性质不要求独立；乘积期望不能总拆开。独立是使 $\mathbb E[XY]=\mathbb E[X]\mathbb E[Y]$ 成立的常见充分条件，但不是必要条件。

## 3 · 样本均值与 Monte Carlo（20 分）

从第 1 题的骰子独立采样 10 次，得到：

```text
[6, 3, 5, 4, 6, 2, 5, 6, 4, 6]
```

1. 计算样本均值。
2. 它与理论期望 $\mathbb E[X]$ 是否必须完全相等？
3. 如果改用 100,000 次独立采样，样本均值通常会发生什么变化？
4. 写出用 $n$ 个样本估计期望的公式。
5. 解释为什么换一个随机种子后，估计结果可能不同。

✍️ 你的答案：

1. 样本均值 = 4.7
2. 不必，根据大数定理，样本越多越接近
3. 样本均值会更接近期望
4.  n个样本估计期望的公式
    $$
    \overline X_n
    =
    \frac1n\sum_{i=1}^{n}X_i
    $$
5. 因为采样觉有随机性，不可能每次采样完全一致

> ✅ **批改第 3 题（20 / 20）**：样本均值 $4.7$、大数定律直觉、估计公式和随机种子对有限样本的影响均正确。更完整地说，不同随机种子会产生不同的伪随机样本序列，所以有限样本均值会波动；它不要求单调接近期望，但样本增多时通常更稳定。

## 4 · LLM 的期望损失（20 分）

数据中有两类 token 预测位置：

| 类型 | 在数据中的概率 | 模型给真实 token 的概率 |
|---|---:|---:|
| A | 0.75 | 0.8 |
| B | 0.25 | 0.2 |

已知：

$$
-\ln0.8\approx0.223,
\qquad
-\ln0.2\approx1.609
$$

1. 分别写出两类位置的 NLL。
2. 计算数据分布上的期望 NLL。
3. 为什么不能简单地计算 $(0.223+1.609)/2$？
4. 类型 B 只占 $25\%$，为什么仍会明显拉高期望损失？
5. 随机 mini-batch 的平均 loss 与理论期望 loss 是什么关系？

✍️ 你的答案：

1. 
$$
NLL_A = 0.223 \\
NLL_B = 1.609
$$

2. 期望 $E[NLL]=0.75*NLL_A + 0.25*NLL_B = 0.570$

3. 不能直接算术平均，因为 A、B 在数据中的概率不同。期望损失必须按出现概率加权；直接除以 2 相当于假设两类各占 50%。

4. 印着类型B的争取率很小只有0.2,其损失就很大，为1.609，虽然它只站25%，但还是会对期望影响很大
5. 单个 mini-batch 会因抽样波动而偏高或偏低；batch 越大或统计的 batch 越多，平均 loss 通常越接近理论期望。

> ✅ **批改第 4 题（20 / 20）**：两类 NLL、概率加权原因和 mini-batch 的样本估计含义全部正确。精确使用题目给出的近似值时：
>
> $$
> \mathbb E[\mathrm{NLL}]
> =
> 0.75(0.223)+0.25(1.609)
> =
> 0.5695
> \approx
> 0.570
> $$
>
> 第 4 问中的“争取率”应为“真实 token 概率”，“只站 25%”应为“只占 25%”；数学解释不受影响。

## 5 · RLHF 的期望奖励与 NumPy（20 分）

固定一个提示词，策略模型可能生成：

| 回答 | 策略概率 | 奖励 |
|---|---:|---:|
| 准确简洁 | 0.60 | 1.0 |
| 正确但冗长 | 0.30 | 0.2 |
| 包含错误 | 0.10 | -1.0 |

1. 计算当前策略的期望奖励。
2. 新策略概率为 `[0.70, 0.25, 0.05]`，奖励不变。计算新期望奖励并判断是否改善。
3. “期望奖励为某个数”是否代表每次回答都会获得这个奖励？
4. 补全下面两个函数，要求检查输入并沿最后一维计算离散期望：

   ```python
   import numpy as np


   def expected_value(values, probabilities):
       pass


   def batched_expected_value(values, probabilities):
       """
       values.shape == probabilities.shape
       最后一维枚举随机结果，返回 shape == values.shape[:-1]
       """
       pass
   ```

5. 用下面的数据验证：

   ```python
   rewards = np.array([1.0, 0.2, -1.0])
   policies = np.array(
       [
           [0.60, 0.30, 0.10],
           [0.70, 0.25, 0.05],
       ]
   )
   ```

✍️ 你的答案：
1. 期望奖励 $=0.60 \times 1+0.30 \times 0.2+0.10 \times (-1)=0.56$。
2. 新期望奖励 $=0.70 \times 1+0.25 \times 0.2+0.05 \times (-1)=0.70$。因为 $0.70>0.56$，所以新策略有所改善。
3. 不代表。期望奖励描述大量重复生成时奖励的长期平均值，单次回答只能获得表中某个具体奖励，即 $1.0$、$0.2$ 或 $-1.0$。
4. 实现如下：

   ```python
   import numpy as np


   def validate_inputs(values, probabilities):
       outcomes = np.asarray(values, dtype=np.float64)
       weights = np.asarray(probabilities, dtype=np.float64)

       if outcomes.ndim == 0 or outcomes.size == 0:
           raise ValueError("values 必须是非空数组")
       if outcomes.shape != weights.shape:
           raise ValueError("values 与 probabilities 形状必须相同")
       if not np.all(np.isfinite(outcomes)):
           raise ValueError("values 不能包含 NaN 或 Inf")
       if not np.all(np.isfinite(weights)):
           raise ValueError("probabilities 不能包含 NaN 或 Inf")
       if np.any(weights < 0):
           raise ValueError("概率不能为负数")
       if not np.allclose(weights.sum(axis=-1), 1.0):
           raise ValueError("最后一维的概率和必须为 1")

       return outcomes, weights


   def expected_value(values, probabilities):
       outcomes, weights = validate_inputs(values, probabilities)
       if outcomes.ndim != 1:
           raise ValueError("expected_value 只接受一维输入")
       return float(np.sum(outcomes * weights))


   def batched_expected_value(values, probabilities):
       """
       values.shape == probabilities.shape
       最后一维枚举随机结果，返回 shape == values.shape[:-1]
       """
       outcomes, weights = validate_inputs(values, probabilities)
       return np.sum(outcomes * weights, axis=-1)
   ```

5. 验证结果如下：

   ```python
   rewards = np.array([1.0, 0.2, -1.0])
   policies = np.array(
       [
           [0.60, 0.30, 0.10],
           [0.70, 0.25, 0.05],
       ]
   )

   assert np.isclose(expected_value(rewards, policies[0]), 0.56)

   reward_matrix = np.broadcast_to(rewards, policies.shape)
   result = batched_expected_value(reward_matrix, policies)

   assert np.allclose(result, [0.56, 0.70])
   print(result)  # [0.56 0.7 ]
   ```

## 第 1 题

所有概率非负，并且：

$$
0.05+0.10+0.15+0.20+0.20+0.30=1
$$

所以分布合法。

$$
\begin{aligned}
\mathbb E[X]
&=
1(0.05)+2(0.10)+3(0.15)\\
&\quad+4(0.20)+5(0.20)+6(0.30)\\
&=
4.3
\end{aligned}
$$

不能直接除以 $6$，因为六个点数并非等概率。期望 $4.3$ 不是骰子可能掷出的单次点数，但期望不要求属于可能取值集合；它表示大量重复投掷后的长期平均点数。

## 第 2 题

$$
\begin{aligned}
\mathbb E[X^2]
&=
1^2(0.05)+2^2(0.10)+3^2(0.15)\\
&\quad+4^2(0.20)+5^2(0.20)+6^2(0.30)\\
&=
20.8
\end{aligned}
$$

$$
(\mathbb E[X])^2
=
4.3^2
=
18.49
$$

所以：

$$
\mathbb E[X^2]\ne(\mathbb E[X])^2
$$

一般不能把非线性函数直接移到期望外。

使用线性性质：

$$
\mathbb E[2X+3]
=
2\mathbb E[X]+3
=
11.6
$$

$\mathbb E[X+Y]=\mathbb E[X]+\mathbb E[Y]$ 不要求独立。$\mathbb E[XY]=\mathbb E[X]\mathbb E[Y]$ 并非总成立；独立是使其成立的常见充分条件。

## 第 3 题

样本和为：

$$
6+3+5+4+6+2+5+6+4+6=47
$$

样本均值：

$$
\overline X
=
\frac{47}{10}
=
4.7
$$

它不必与理论期望 $4.3$ 完全相等。有限样本存在随机波动；在独立同分布且期望存在等常见条件下，样本数增加到 100,000 时，样本均值通常会更接近 $4.3$。

估计公式：

$$
\widehat{\mathbb E[X]}
=
\overline X_n
=
\frac1n\sum_{i=1}^{n}X_i
$$

随机种子改变了伪随机采样序列，因此有限样本的均值可能改变。

## 第 4 题

两类位置的 NLL 分别为：

$$
\mathcal L_A=-\ln0.8\approx0.223
$$

$$
\mathcal L_B=-\ln0.2\approx1.609
$$

期望 NLL：

$$
\begin{aligned}
\mathbb E[\mathcal L]
&=
0.75(0.223)+0.25(1.609)\\
&=
0.16725+0.40225\\
&=
0.5695
\end{aligned}
$$

不能对两个 NLL 直接取普通平均，因为两类位置不是等概率。类型 B 虽然只占 $25\%$，但单次 NLL 很大，因此加权后仍贡献 $0.40225$，超过类型 A 的贡献。

随机 mini-batch 的平均 loss 是数据分布上理论期望 loss 的样本估计；不同 batch 会波动，随机采样合理且 batch 增大时通常更稳定。

## 第 5 题

当前策略：

$$
\begin{aligned}
\mathbb E[R]
&=
0.60(1.0)+0.30(0.2)+0.10(-1.0)\\
&=
0.56
\end{aligned}
$$

新策略：

$$
\begin{aligned}
\mathbb E[R]_{\text{new}}
&=
0.70(1.0)+0.25(0.2)+0.05(-1.0)\\

\end{aligned}
$$

新策略的期望奖励更高。期望描述长期平均，不代表某次回答一定得到 $0.56$ 或 $0.70$。

```python
import numpy as np


def validate_inputs(values, probabilities):
    outcomes = np.asarray(values, dtype=np.float64)
    weights = np.asarray(probabilities, dtype=np.float64)

    if outcomes.ndim == 0 or outcomes.size == 0:
        raise ValueError("values 必须是非空数组")
    if outcomes.shape != weights.shape:
        raise ValueError("values 与 probabilities 形状必须相同")
    if not np.all(np.isfinite(outcomes)):
        raise ValueError("values 不能包含 NaN 或 Inf")
    if not np.all(np.isfinite(weights)):
        raise ValueError("probabilities 不能包含 NaN 或 Inf")
    if np.any(weights < 0.0):
        raise ValueError("概率不能为负数")
    if not np.allclose(weights.sum(axis=-1), 1.0):
        raise ValueError("最后一维的概率和必须为 1")
    return outcomes, weights


def expected_value(values, probabilities):
    outcomes, weights = validate_inputs(values, probabilities)
    if outcomes.ndim != 1:
        raise ValueError("expected_value 只接受一维输入")
    return float(np.sum(outcomes * weights))


def batched_expected_value(values, probabilities):
    outcomes, weights = validate_inputs(values, probabilities)
    return np.sum(outcomes * weights, axis=-1)
```
