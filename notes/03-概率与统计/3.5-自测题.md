## 1 · 从概率手算 PPL（20 分）

三个位置的真实 token 概率为：

$$
[0.5,\ 0.25,\ 0.125]
$$

已知：

$$
\ln0.5\approx-0.693,\quad
\ln0.25\approx-1.386,\quad
\ln0.125\approx-2.079
$$

计算：

1. 逐 token NLL
2. 平均 NLL
3. PPL
4. 三个概率的几何平均，并说明它与 PPL 的关系

✍️ 你的答案：
1. 逐 token NLL \
NLLs = [0.693, 1.386, 2.079] \
2. $\text {NLL}_{mean}=1.386$ \
3. PPL = exp($\text {NLL}_{mean}$) = 4
4. 三个概率的几何平均为0.25， 是PPL的倒数

> ✅ **更新批改第 1 题（20 / 20）**：逐 token NLL、平均 NLL、PPL、几何平均及其倒数关系全部正确。

3. 只对 `mask == True` 的位置求平均 NLL

✍️ 你的答案：
```python
import numpy as np


def log_softmax(logits, axis=-1):
    values = np.asarray(logits, dtype=np.float64)
    if values.ndim == 0 or values.size == 0:
        raise ValueError("logits 至少需要一维非空输入")
    if not np.all(np.isfinite(values)):
        raise ValueError("logits 不能包含 NaN 或 Inf")
    maximum =  np.max(values, axis=axis, keepdims=True);
    shifted = values - maximum
    return shifted - np.log(np.exp(shifted).sum(axis=axis,keepdims=True))


def perplexity_from_logits(logits, targets, mask):
    values = np.asarray(logits, dtype=np.float64)
    target_ids = np.asarray(targets)

    if values.ndim < 2:
        raise ValueError("logits shape 至少应为 (..., vocab_size)")
    if values.shape[-1] == 0:
        raise ValueError("vocab_size 不能为 0")
    if target_ids.shape != values.shape[:-1]:
        raise ValueError(
            "targets shape 应等于 logits.shape[:-1]："
            f"{values.shape[:-1]}"
        )
    if not np.issubdtype(target_ids.dtype, np.integer):
        raise TypeError("targets 必须是整数 token ID")
    if np.any(target_ids < 0) or np.any(
        target_ids >= values.shape[-1]
    ):
        raise ValueError("targets 包含超出词表范围的 token ID")

    log_probabilities = log_softmax(values, axis=-1)
    target_log_probabilities = np.take_along_axis(
        log_probabilities,
        target_ids[..., np.newaxis],
        axis=-1,
    )[..., 0]
    token_nll = -target_log_probabilities
    mean_nll = float(token_nll[mask].mean())
    perplexity = float(np.exp(mean_nll))
    return perplexity
```

为什么不能先把所有真实 token 概率相乘\
因为 每个概率都小于1，多个概率相乘，可能导致 结果小到超出 浮点数表示的范围。

## 第 1 题

逐 token NLL：

$$
[0.693,\ 1.386,\ 2.079]
$$

平均 NLL：

$$
\frac{0.693+1.386+2.079}{3}
=
1.386
$$

PPL：

$$
e^{1.386}\approx4
$$

几何平均：

$$
\sqrt[3]{0.5\times0.25\times0.125}
=
\sqrt[3]{\frac1{64}}
=
\frac14
=
0.25
$$

PPL 是几何平均概率的倒数：

$$
\operatorname{PPL}=\frac1{0.25}=4
$$

## 第 2 题

真实 token 概率的几何平均为：

$$
\frac1{\operatorname{PPL}}
=
\frac18
=
0.125
$$

“8 个等可能候选”只是**等效直觉**，不能当作真实预测过程。语言模型通常面对整个词表，概率不均匀，而且每个位置的分布不同。

PPL 的理论最小值为：

$$
1
$$

当每个被评分位置都给真实 token 概率 $1$ 时达到。PPL 不是正确率；它由真实 token 概率的几何平均得到，会考虑概率置信程度。

## 第 3 题

只有前两个位置有效：

$$
\overline{\mathcal L}
=
\frac{0.2+0.4}{2}
=
0.3
$$

$$
\operatorname{PPL}
=
e^{0.3}
\approx1.350
$$

`9.9` 对应无效位置，可能是 padding 或不参与本轮评分的上下文。如果纳入平均，会人为抬高损失和 PPL。

## 第 5 题

```python
import numpy as np


def log_softmax(logits, axis=-1):
    maximum = np.max(logits, axis=axis, keepdims=True)
    shifted = logits - maximum
    return shifted - np.log(
        np.exp(shifted).sum(axis=axis, keepdims=True)
    )


def perplexity_from_logits(logits, targets, mask):
    log_probabilities = log_softmax(logits, axis=-1)
    target_log_probabilities = np.take_along_axis(
        log_probabilities,
        targets[..., np.newaxis],
        axis=-1,
    )[..., 0]

    mean_nll = -target_log_probabilities[mask].mean()
    return np.exp(mean_nll)
```

很多小概率相乘可能发生浮点下溢，错误地变成 `0.0`，随后无法得到有限的 log-probability。稳定做法是直接对真实 token 的 log-probabilities 求平均，最后只执行一次 `exp`。
