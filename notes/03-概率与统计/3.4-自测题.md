## 1 · 定义、逆运算与定义域（20 分）

1. 把下面的指数形式改写成对数形式：

   $$
   2^3=8,\qquad10^{-2}=0.01
   $$

2. 计算：

   $$
   \ln1,\qquad\ln e,\qquad\ln(e^{-3})
   $$

3. 在实数范围内，说明 $\ln0$、$\ln(-2)$ 为什么不是正常的有限值。

✍️ 你的答案：
1. $log_2 8=3$; $log_{10}0.1=-2$
2. $ln 1 = 0; ln e=1; ln(e^{-3})=-3$
3. 因为在实数范围内，$\ln x$ 的定义域是 $x>0$。当 $x\to0^+$ 时，$\ln x\to-\infty$，所以 $\ln0$ 不是有限值；而 $-2<0$，因此 $\ln(-2)$ 在实数范围内没有定义。

> ⚠️ **批改第 1 题（18 / 20）**：第 2、3 问正确。第 1 问的第二个真数抄错了，题目是 $10^{-2}=0.01$，因此应写：
>
> $$
> \log_{10}0.01=-2
> $$
>
> 原答案 $\log_{10}0.1=-2$ 不成立，因为 $\log_{10}0.1=-1$。另外，规范数学写法使用 `\log` 和 `\ln`。

## 2 · 运算律与常见陷阱（20 分）

已知：

$$
\ln2\approx0.693,\qquad
\ln3\approx1.099,\qquad
\ln5\approx1.609
$$

1. 不直接使用计算器，计算 $\ln6$。
2. 把 $\ln(a^3/c)$ 展开，其中 $a>0,c>0$。
3. 判断并解释：$\ln(2+3)=\ln2+\ln3$ 是否正确。

✍️ 你的答案：\
1. $\ln 6 = \ln (2*3) = \ln 2 + \ln 3 = 1.792$
2. $\ln(a^3/c) = 3\ln a-\ln c$
3. $\ln(2+3)=\ln2+\ln3$ 不正确， 对数加法不能拆分

> ✅ **批改第 2 题（20 / 20）**：计算与展开全部正确。第 3 问可以表述得更精确：**和的对数不能拆成对数之和；乘积的对数才能拆**。即 $\ln(2+3)=\ln5$，而 $\ln2+\ln3=\ln6$。

## 3 · 概率、log-probability 与 NLL（20 分）

正确 token 的两个候选概率为：

$$
p_A=0.1,\qquad p_B=0.01
$$

已知：

$$
\ln0.1\approx-2.303,\qquad
\ln0.01\approx-4.605
$$

1. 分别计算两个概率的 log-probability 和 NLL。
2. 哪个概率对应更大损失？为什么？
3. 概率从 $0.01$ 提高到 $0.1$，NLL 减少多少？这和“概率提高 10 倍”有什么关系？

✍️ 你的答案：\
1.
$$
log\text-probability(p_A) = -2.303 \\
log\text-probability(p_B) = -4.605 \\
NLL(p_A)=2.303 \\
NLL(p_B)=4.605
$$
2. $p_B$的损失更大, 因为对数的特性
3. 概率从 0.01 提高到 0.1，NLL 减少2.303， 关系是$ln10 \approx 2.303$

> ✅ **批改第 3 题（19 / 20）**：数值和 10 倍关系正确。第 2 问的理由需要说完整：
>
> ```text
> p_B 更小
> → ln(p_B) 更负
> → -ln(p_B) 更大
> → NLL 更大
> ```
>
> 第 3 问按题目给定近似值计算为 $4.605-2.303=2.302$；写成约 $2.303=\ln10$ 也正确。

## 4 · 序列概率与数值稳定性（20 分）

三个位置的正确 token 概率为：

$$
[0.8,\ 0.5,\ 0.25]
$$

已知：

$$
\ln0.8\approx-0.223,\quad
\ln0.5\approx-0.693,\quad
\ln0.25\approx-1.386
$$

1. 计算序列概率乘积、序列 log-probability、总 NLL 和平均 token NLL。
2. 如果序列很长，为什么不应先把所有概率相乘再取 log？
3. 写出稳定计算总 NLL 的 NumPy 表达式。

✍️ 你的答案：\
1. 计算序列概率乘积为 0.1， \
   log-probability = [0.223, 0.693, 1.386];\
   $NLL_sum = 2.302$ \
   $NLL_mean = 0.767$
2. 这样可能导致概率乘积太小，超出浮点数的表示范围
3. `total_nll = -np.sum(np.log(np.array([0.8, 0.5, 0.25])))`

> ⚠️ **批改第 4 题（16 / 20）**：序列概率、总 NLL、平均 NLL、下溢直觉和 NumPy 表达式正确，但 **log-probability 的符号和对象混淆了**。
>
> 原答案中的：
>
> $$
> [0.223,\ 0.693,\ 1.386]
> $$
>
> 是逐 token 的 **NLL**，即 $-\ln p_t$，不是 log-probability。正确结果为：
>
> $$
> \begin{aligned}
> \text{逐 token log-probability}
> &=[-0.223,-0.693,-1.386]\\
> \text{序列 log-probability}
> &=\sum_t\ln p_t\approx-2.302\\
> \text{总 NLL}
> &=-\sum_t\ln p_t\approx2.302
> \end{aligned}
> $$
>
> 第 2 问更准确地说：很多小概率连乘可能发生**下溢**，浮点结果变成 `0.0`，随后 `log(0.0)` 得到 `-inf`。

再回答：

1. 为什么不能直接计算 `np.log(logits)`？
2. 若 `logits.shape == (2, 4, 10_000)`，沿词表维计算后，`log_softmax` 的输出 shape 是什么？
3. 如何用代码验证输出确实是归一化概率的对数？

✍️ 你的答案：\
NumPy 实现：
```python
import numpy as np

def logsumexp(values, axis=-1, keepdims=False):
    maximum = np.max(values, axis=axis, keepdims=True)
    shifted = values - maximum
    result = maximum + np.log(
        np.exp(shifted).sum(axis=axis, keepdims=True)
    )
    if keepdims:
        return result
    return np.squeeze(result, axis=axis)

def log_softmax(logits, axis=-1):
    return logits - logsumexp(
        logits,
        axis=axis,
        keepdims=True,
    )
```
1. logits不是概率，它的值有正有负，不符合要求
2. log_softmax 的输出 shape是 (2, 4, 10_000)
3. 在对其求e的指数，相加后的总和为1

## 第 1 题

指数形式与对数形式互换：

$$
2^3=8
\quad\Longleftrightarrow\quad
\log_2 8=3
$$

$$
10^{-2}=0.01
\quad\Longleftrightarrow\quad
\log_{10}0.01=-2
$$

利用指数与自然对数互为逆运算：

$$
\ln1=0,\qquad
\ln e=1,\qquad
\ln(e^{-3})=-3
$$

对实数 $y$，$e^y$ 永远大于 $0$，不能等于 $0$ 或负数。因此 $\ln0$ 没有有限值，$\ln(-2)$ 在实数范围内没有定义。

## 第 2 题

乘积变加法：

$$
\ln6
=
\ln(2\times3)
=
\ln2+\ln3
\approx
0.693+1.099
=
1.792
$$

幂次和除法规则：

$$
\ln\left(\frac{a^3}{c}\right)
=
\ln(a^3)-\ln c
=
3\ln a-\ln c
$$

等式：

$$
\ln(2+3)=\ln2+\ln3
$$

错误。左边是 $\ln5\approx1.609$；右边是 $\ln6\approx1.792$。对数只能把乘积拆成和，不能把加法拆开。

序列 log-probability：

$$
\ln P
=
-0.223-0.693-1.386

## 第 5 题

```python
import numpy as np


def logsumexp(values, axis=-1, keepdims=False):
    maximum = np.max(values, axis=axis, keepdims=True)
    shifted = values - maximum
    result = maximum + np.log(
        np.exp(shifted).sum(axis=axis, keepdims=True)
    )
    if keepdims:
        return result
    return np.squeeze(result, axis=axis)


def log_softmax(logits, axis=-1):
```
