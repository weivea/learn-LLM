# 3.9 采样（sampling）

> 重要度：⭐⭐（LLM 把下一个 token 的概率分布变成实际输出的关键）
> 目标：理解“按概率抽取一个 token”；能手算一次 categorical sampling；能解释并实现 temperature、top-k、top-p；能说清它们在自回归生成中的顺序

## 一句话总结

**采样就是把模型预测的“下一个 token 概率分布”变成一个实际 token；temperature 负责调整分布的尖锐程度，top-k 和 top-p 负责裁掉不太可能的候选，再从剩余概率中随机抽取。**

```text
前文 tokens
    ↓ 模型
下一个 token 的 logits
    ↓ temperature
调整概率分布的尖锐程度
    ↓ top-k / top-p
限制候选集合
    ↓ 重新归一化并随机抽取
得到一个 token
    ↓ 追加到前文，继续下一轮
```

最核心的区别：

```text
模型学习：应该给每个 token 多大概率
采样策略：怎样把这组概率变成一个实际输出
```

采样策略不会让模型获得原本没有的知识。它只会改变模型已有概率分布被转成文本的方式。

## 1. 从概率分布到一个实际 token

假设模型根据前文预测出：

| token | `数学` | `代码` | `模型` | `天气` |
|---|---:|---:|---:|---:|
| 概率 | 0.50 | 0.30 | 0.15 | 0.05 |

这表示：

$$
P(\text{下一个 token}=\text{数学}\mid\text{前文})=0.50
$$

但模型最终必须输出一个具体 token，而不能只停留在概率表上。

一种做法是永远选择概率最大的 `数学`，这称为 **greedy decoding（贪心解码）**。另一种做法是：

```text
数学：50% 的机会
代码：30% 的机会
模型：15% 的机会
天气： 5% 的机会
```

然后随机抽取一次。这就是从**类别分布**（categorical distribution）中采样。

> 概率最高的 token 最容易被抽中，但不保证每次都被抽中；概率较低的 token 也不是绝对不可能出现。

### 1.1 采样不是“随便随机”

“随机”不等于“每个 token 等概率”。

若完全均匀随机，四个 token 的概率都会是 $0.25$；但按模型分布采样时，必须尊重模型给出的：

$$
[0.50,\ 0.30,\ 0.15,\ 0.05]
$$

因此：

```text
均匀随机：忽略模型判断
按分布采样：高概率 token 更常出现，低概率 token 偶尔出现
```

## 2. 手算一次 categorical sampling

计算累积概率：

| token | 概率 | 累积区间 |
|---|---:|---|
| `数学` | 0.50 | $[0.00,0.50)$ |
| `代码` | 0.30 | $[0.50,0.80)$ |
| `模型` | 0.15 | $[0.80,0.95)$ |
| `天气` | 0.05 | $[0.95,1.00)$ |

从 $[0,1)$ 均匀抽一个随机数 $U$：

$$
U\sim\operatorname{Uniform}(0,1)
$$

然后看 $U$ 落在哪个累积区间：

```text
U = 0.12 → 数学
U = 0.63 → 代码
U = 0.91 → 模型
U = 0.99 → 天气
```

用累积分布函数表示，令：

$$
C_j=\sum_{i=1}^{j}p_i
$$

就选择第一个满足下式的类别 $j$：

$$
\boxed{
C_j>U
}
$$

这称为**逆变换采样**的离散版本。

### 2.1 为什么必须重新归一化

如果某种策略过滤后只保留：

```text
数学：0.50
代码：0.30
```

剩余概率之和是 $0.80$，不能直接作为完整概率分布。需要除以总和：

$$
\begin{aligned}
P'(\text{数学})
&=
\frac{0.50}{0.80}
=
0.625\\
P'(\text{代码})
&=
\frac{0.30}{0.80}
=
0.375
\end{aligned}
$$

过滤后的标准步骤是：

```text
不保留的 token 概率设为 0
            ↓
剩余概率除以它们的总和
            ↓
得到概率和重新为 1 的分布
```

## 3. 一次采样与大量采样

一次采样只会得到一个结果，不能要求它“符合长期比例”。

例如 `天气` 的概率只有 $0.05$，但第一次仍可能抽到它；这不说明概率表错误。

如果独立采样 $N$ 次，token $i$ 的出现次数记为 $C_i$，则：

$$
C_i\sim\operatorname{Binomial}(N,p_i)
$$

其期望和方差为：

$$
\boxed{
\mathbb E[C_i]=Np_i
}
$$

$$
\boxed{
\operatorname{Var}(C_i)=Np_i(1-p_i)
}
$$

这个方差公式可以从 Bernoulli 随机变量推导出来。令：

$$
I_n=
\begin{cases}
1, & \text{第 }n\text{ 次采样得到 token }i\\
0, & \text{第 }n\text{ 次采样没有得到 token }i
\end{cases}
$$

因为每次采到 token $i$ 的概率都是 $p_i$，所以：

$$
I_n\sim\operatorname{Bernoulli}(p_i)
$$

对于只取 $0$ 或 $1$ 的指示变量，有 $I_n^2=I_n$，因此：

$$
\begin{aligned}
\operatorname{Var}(I_n)
&=\mathbb E[I_n^2]-\mathbb E[I_n]^2\\
&=p_i-p_i^2\\
&=p_i(1-p_i)
\end{aligned}
$$

$N$ 次采样中 token $i$ 的总出现次数就是：

$$
C_i=I_1+I_2+\cdots+I_N
$$

当各次采样相互独立时，和的方差等于各项方差之和：

$$
\begin{aligned}
\operatorname{Var}(C_i)
&=\sum_{n=1}^{N}\operatorname{Var}(I_n)\\
&=\sum_{n=1}^{N}p_i(1-p_i)\\
&=Np_i(1-p_i)
\end{aligned}
$$

直观上，每次采样都贡献一份 $p_i(1-p_i)$ 的不确定性，独立重复 $N$ 次后，这些方差累加起来。多个 token 的计数整体服从多项分布，而只看某一个 token 是否出现时，它的计数边缘分布就是二项分布。

> 这个结论要求每次都从同一个固定分布中独立采样。LLM 自回归生成时，上一个 token 会改变下一步的概率分布，因此一段生成中某个 token 的总出现次数通常不能直接套用这个二项分布方差公式。

例如独立采样 $1000$ 次，`数学` 的概率是 $0.5$：

$$
\mathbb E[C_{\text{数学}}]
=
1000(0.5)
=
500
$$

$$
\operatorname{Std}(C_{\text{数学}})
=
\sqrt{1000(0.5)(0.5)}
\approx
15.811
$$

这不是说它一定出现恰好 $500$ 次，而是反复进行“采样 1000 次”的实验时，计数以 $500$ 为中心波动。

经验频率为：

$$
\widehat p_i=\frac{C_i}{N}
$$

随着 $N$ 增大，经验频率通常越来越接近理论概率 $p_i$。这把 3.7 的期望和 3.8 的方差连接到了采样。

### 3.1 不要计算 token ID 的“语义方差”

token ID 只是词表索引。例如：

```text
数学 → 1523
代码 → 87
```

这些数字的大小没有语义距离，因此直接计算 token ID 的均值或方差通常没有语义解释。可以研究：

- 某个 token 的出现次数和频率
- 整个概率分布有多集中
- 生成文本的重复率、正确率或其他任务指标

但不要因为两个 token ID 数值接近，就认为它们语义接近。

## 4. LLM 为什么需要逐步采样

自回归语言模型建模的是：

$$
P(x_t\mid x_1,x_2,\ldots,x_{t-1})
$$

即“给定此前全部 token，下一个 token 的条件概率”。

完整序列的联合概率可按链式法则分解：

$$
\boxed{
P(x_1,\ldots,x_T)
=
\prod_{t=1}^{T}
P(x_t\mid x_{<t})
}
$$

其中：

$$
x_{<t}=(x_1,\ldots,x_{t-1})
$$

生成过程：

```text
第 1 步：根据提示词得到 P(x1 | prompt)，采样 x1
第 2 步：把 x1 追加到前文，得到 P(x2 | prompt, x1)，采样 x2
第 3 步：继续追加，得到 P(x3 | prompt, x1, x2)，采样 x3
...
直到采到结束 token 或达到长度上限
```

重要结论：

> 每生成一个 token，条件概率分布都会改变。LLM 不是先生成一张固定概率表，再从中连续抽很多次。

一个早期采样结果不同，后续上下文就不同，后续所有概率也可能随之改变。因此很小的随机差异也可能让完整回答走向不同方向。

## 5. Greedy decoding 与随机采样

### 5.1 Greedy decoding

每一步都选择概率最大的 token：

$$
\boxed{
x_t
=
\underset{i}{\arg\max}\;
P(x_t=i\mid x_{<t})
}
$$

特点：

- 同样输入和模型通常得到相同结果
- 简单、稳定
- 只考虑当前一步，不保证整段序列的联合概率最大
- 容易反复走向相似表达，缺少多样性

严格来说，greedy 是确定性选择，不是随机采样；但讨论生成策略时通常会与 sampling 放在一起比较。

### 5.2 随机采样

按完整概率分布抽取：

$$
x_t
\sim
P(\cdot\mid x_{<t})
$$

特点：

- 能生成多个合理但不同的回答
- 保留低概率候选的机会
- 也可能抽到不连贯或错误的 token

### 5.3 二者不是“正确”和“错误”的区别

```text
greedy：更确定，但可能僵硬、重复或局部最优
sampling：更多样，但随机性过强时可能不稳定
```

temperature、top-k、top-p 的目的，就是控制“确定性与多样性”的平衡。

## 6. Temperature：改变分布尖锐程度

模型先输出 logits：

$$
z_1,z_2,\ldots,z_V
$$

温度为 $T>0$ 时：

$$
\boxed{
P_T(i)
=
\frac{\exp(z_i/T)}
{\sum_j\exp(z_j/T)}
}
$$

即先把 logits 除以 $T$，再做 Softmax。

### 6.1 三种情况

#### $0<T<1$：分布更尖锐

除以小于 $1$ 的数会放大 logits 之间的差距：

```text
高概率 token 更高
低概率 token 更低
输出更保守、更集中
```

#### $T=1$：保持原分布

$$
P_1(i)=\operatorname{softmax}(z)_i
$$

#### $T>1$：分布更平坦

除以大于 $1$ 的数会缩小 logits 之间的差距：

```text
高概率 token 优势减弱
低概率 token 获得更多机会
输出更多样，也更容易出现不稳定内容
```

### 6.2 数值例子

原概率为：

$$
[0.50,\ 0.30,\ 0.15,\ 0.05]
$$

如果这些概率来自某组 logits，则也可以用等价形式计算：

$$
\boxed{
P_T(i)
=
\frac{P(i)^{1/T}}
{\sum_jP(j)^{1/T}}
}
$$

得到：

| temperature | 调整后的概率 |
|---:|---|
| $T=0.5$ | $[0.6849,\ 0.2466,\ 0.0616,\ 0.0068]$ |
| $T=1$ | $[0.5000,\ 0.3000,\ 0.1500,\ 0.0500]$ |
| $T=2$ | $[0.3790,\ 0.2936,\ 0.2076,\ 0.1198]$ |

注意最高概率 token 一直是第一个，因为正温度只改变差距，不改变 logits 的大小顺序。

### 6.3 `temperature=0` 是特殊约定

公式要求：

$$
T>0
$$

因为除以 $0$ 没有定义。许多生成 API 把 `temperature=0` 约定为 greedy decoding，但这属于 API 的特殊分支，不是把 $T=0$ 直接代入公式。

理论上：

```text
T → 0+：概率越来越集中到最大 logit
T → +∞：有限 logits 的概率越来越接近均匀
```

### 6.4 为什么应修改 logits

标准实现是：

```python
probabilities = softmax(logits / temperature)
```

不是：

```python
probabilities = softmax(logits) / temperature
```

后者只是把所有概率同时除以一个数，既不能正确改变相对比例，概率和也不再是 $1$。

## 7. Top-k：只保留概率最高的 k 个 token

仍使用：

$$
[0.50,\ 0.30,\ 0.15,\ 0.05]
$$

若：

$$
k=2
$$

只保留概率最高的两个 token：

```text
过滤前：[0.50, 0.30, 0.15, 0.05]
保留后：[0.50, 0.30, 0.00, 0.00]
```

重新归一化：

$$
\boxed{
[0.625,\ 0.375,\ 0,\ 0]
}
$$

### 7.1 边界

```text
k = 1：等价于在当前分布中只保留最大概率 token
k = V：保留整个词表，不做 top-k 过滤
```

其中 $V$ 是词表大小。

### 7.2 优点与局限

优点：

- 简单
- 能排除概率极低的长尾 token
- 候选数量有固定上限

局限：

- 无论当前分布很确定还是很不确定，都固定保留 $k$ 个
- 当分布很尖锐时，后几个候选可能质量很低
- 当分布很平坦时，固定的 $k$ 又可能删掉很多合理候选

---

## 8. Top-p：保留累积概率达到 p 的最小候选集合

Top-p 也叫 **nucleus sampling（核采样）**。

步骤：

1. 按概率从大到小排序
2. 从最大概率开始累加
3. 保留使累积概率首次达到或超过阈值 $p$ 的最小集合
4. 将其余概率设为 $0$
5. 对保留概率重新归一化

仍使用：

$$
[0.50,\ 0.30,\ 0.15,\ 0.05]
$$

累积概率：

| 排名 | 概率 | 累积概率 |
|---:|---:|---:|
| 1 | 0.50 | 0.50 |
| 2 | 0.30 | 0.80 |
| 3 | 0.15 | 0.95 |
| 4 | 0.05 | 1.00 |

若：

$$
p=0.80
$$

最小集合是前两个 token，重新归一化后：

$$
[0.625,\ 0.375,\ 0,\ 0]
$$

若：

$$
p=0.90
$$

前两个 token 的累积概率只有 $0.80$，必须包含第三个，使累积概率达到 $0.95$：

$$
\left[
\frac{0.50}{0.95},
\frac{0.30}{0.95},
\frac{0.15}{0.95},
0
\right]
$$

约等于：

$$
[0.5263,\ 0.3158,\ 0.1579,\ 0]
$$

### 8.1 为什么要保留“跨过阈值”的 token

若只保留累积概率仍小于等于阈值的 token，可能永远达不到目标概率质量。

正确规则是：

> 保留第一个使累积概率达到或超过 $p$ 的 token。

### 8.2 Top-p 的候选数量是动态的

```text
分布很尖锐：
    少数 token 就能达到 p
    → 候选集合较小

分布很平坦：
    需要更多 token 才能达到 p
    → 候选集合较大
```

这是 top-p 相比固定 top-k 的核心优势。

### 8.3 边界

```text
p = 1：保留完整分布
p 很小：仍至少保留最高概率 token
0 < p <= 1
```

## 9. Temperature、top-k、top-p 的区别

| 方法 | 做了什么 | 是否删除候选 | 控制对象 |
|---|---|:---:|---|
| temperature | 调整所有 logits 的相对差距 | 否 | 分布尖锐或平坦 |
| top-k | 只保留排名最高的固定 $k$ 个 | 是 | 候选数量 |
| top-p | 保留累计概率达到 $p$ 的最小集合 | 是 | 候选概率质量 |

记忆：

```text
temperature：改形状
top-k：按名次截断
top-p：按累计概率截断
```

高 temperature 不能恢复已经被 top-k 或 top-p 删除的 token。过滤后的 token 概率已经是 $0$。

## 10. 组合时的完整顺序

一个清晰的实现顺序是：

```text
1. 模型输出 logits
2. logits / temperature
3. Softmax 得到温度调整后的概率
4. top-k 过滤（如果启用）
5. top-p 过滤（如果启用）
6. 重新归一化
7. 从 categorical distribution 采样
```

工程实现中常直接在 logits 上把被过滤项设为 $-\infty$，最后统一 Softmax：

```text
logits
  ↓ 除以 temperature
temperature-adjusted logits
  ↓ top-k / top-p mask
不保留的位置设为 -∞
  ↓ softmax
被屏蔽位置概率变为 0，剩余位置自动归一化
```

不同框架可能以 logits processor / warper 的形式组合策略；阅读 API 时应确认具体顺序。

### 10.1 为什么 temperature 通常在 top-p 前

temperature 会改变各 token 的概率和累积概率，因此可能改变 top-p 最终保留多少 token。

正 temperature 不改变 token 排名，所以它不会改变 top-k 选中了哪些 token；但会改变这些 token 内部的相对概率。

### 10.2 同时使用 top-k 与 top-p

同时启用时，最终候选集是连续过滤后留下的集合：

```text
先 top-k：最多保留 k 个
再 top-p：在这些候选中继续按累计概率缩小
```

这不是必须的。实际调参时可以：

- 只用 temperature
- temperature + top-p
- temperature + top-k
- 三者组合

关键是理解每个参数在改变什么，而不是机械记住一组“万能参数”。

## 11. 参数如何影响生成

下面只是一般趋势，不是质量保证：

| 设置 | 常见趋势 |
|---|---|
| 较低 temperature | 更集中、更稳定、更保守 |
| 较高 temperature | 更多样、更意外、错误风险也更高 |
| 较小 top-k | 候选少、输出更受约束 |
| 较大 top-k | 保留更多长尾候选 |
| 较小 top-p | 只保留较高概率质量 |
| 较大 top-p | 允许更多低概率候选 |

典型任务直觉：

```text
事实问答、代码生成：
    通常更重视稳定与准确
    → 随机性不宜过强

头脑风暴、创意写作：
    通常希望多种表达
    → 可以允许更多样性
```

但没有适合所有模型和任务的固定数值。模型本身、提示词、词表、上下文和生成 API 都会影响结果。

### 11.1 采样不能修复模型知识

如果正确答案在模型分布中的概率很低：

- 降低 temperature 可能让当前最高概率错误更加稳定
- 提高 temperature 可能偶然抽到正确答案，也会增加其他错误
- top-k / top-p 甚至可能直接过滤掉正确 token

因此采样策略是**解码控制**，不是知识校正或事实验证机制。

## 12. 随机种子与可复现性

伪随机数生成器在相同初始状态下会产生相同随机序列：

```python
rng_a = np.random.default_rng(42)
rng_b = np.random.default_rng(42)
```

在模型、输入、参数、硬件算法和执行顺序都相同的简化条件下，相同 seed 有助于复现采样结果。

但要区分：

```text
固定 seed：让伪随机过程更容易复现
greedy decoding：过程本身不需要随机抽取
```

真实深度学习系统还可能受到并行执行、浮点精度和非确定性算子的影响，所以“相同 seed”不是跨所有环境绝对逐位一致的保证。

## 13. NumPy 核心实现

### 13.1 稳定 Softmax 与 temperature

```python
# 导入 NumPy，并使用惯用别名 np
import numpy as np


def softmax(logits):
    # 将输入转换为 float64 数组，保证后续支持稳定的浮点运算
    logits = np.asarray(logits, dtype=np.float64)
    # 所有 logit 同减最大值，避免计算指数时发生数值溢出
    shifted = logits - np.max(logits)
    # 对平移后的每个 logit 计算指数
    exp_values = np.exp(shifted)
    # 每个指数值除以总和，将结果归一化为概率分布
    return exp_values / exp_values.sum()


def temperature_distribution(logits, temperature=1.0):
    # temperature 必须为正数，否则无法执行 logits / temperature
    if temperature <= 0:
        raise ValueError("temperature 必须大于 0")
    # 先用温度缩放 logits，再通过 Softmax 转换为概率分布
    return softmax(logits / temperature)
```

减去最大 logit 不改变 Softmax 结果，但能降低指数溢出的风险。

### 13.2 Top-k

```python
def top_k_distribution(probabilities, k):
    # 将输入统一转换为 float64 数组
    probabilities = np.asarray(
        probabilities,
        dtype=np.float64,
    )
    # argsort 返回从小到大的索引，取最后 k 个即概率最高的 k 项
    top_indices = np.argsort(probabilities)[-k:]
    # 创建与原概率数组形状相同的全零数组
    filtered = np.zeros_like(probabilities)
    # 只复制 top-k 候选的原概率，其余位置继续保持为 0
    filtered[top_indices] = probabilities[top_indices]
    # 除以保留概率之和，使过滤后的概率总和重新等于 1
    return filtered / filtered.sum()
```

### 13.3 Top-p

```python
def top_p_distribution(probabilities, top_p):
    # 将输入统一转换为 float64 数组
    probabilities = np.asarray(
        probabilities,
        dtype=np.float64,
    )
    # argsort 默认升序排列索引，[::-1] 将其反转为概率降序
    order = np.argsort(probabilities)[::-1]
    # 按降序索引重排概率，便于从最高概率开始累加
    sorted_probabilities = probabilities[order]
    # 计算降序概率的累积和
    cumulative = np.cumsum(sorted_probabilities)

    # 找到累积概率首次达到或超过 top_p 的位置，并计入该 token
    keep_count = (
        np.searchsorted(cumulative, top_p, side="left")
        + 1
    )
    # 取出需要保留的 token 在原概率数组中的索引
    keep_indices = order[:keep_count]

    # 创建全零数组，并只恢复需要保留的候选概率
    filtered = np.zeros_like(probabilities)
    filtered[keep_indices] = probabilities[keep_indices]
    # 重新归一化，使过滤后的概率总和等于 1
    return filtered / filtered.sum()
```

`searchsorted(...)+1` 保证包含第一个让累积概率达到或超过阈值的 token。

### 13.4 抽取 token

```python
def sample_token(tokens, probabilities, rng):
    # 按 probabilities 指定的类别概率，从 token 索引中随机抽取一个
    index = rng.choice(
        len(tokens),
        p=probabilities,
    )
    # 将抽中的索引映射回对应 token
    return tokens[index]
```

## 14. 常见误区

| 错误说法 | 正确理解 |
|---|---|
| 采样就是从词表均匀随机选 token | 采样必须按模型给出的概率分布抽取 |
| 概率最高的 token 每次都会被抽中 | 它最容易被抽中，但随机采样不保证必选 |
| 低概率 token 一次出现就说明分布错误 | 概率描述长期频率，不禁止单次小概率事件 |
| 过滤后不需要重新归一化 | categorical distribution 的概率和必须为 $1$ |
| temperature 是直接除概率 | 应先用 `logits / T`，再做 Softmax |
| `temperature=0` 可直接代入公式 | 公式要求 $T>0$；`0` 通常是 API 的 greedy 特殊约定 |
| temperature 会改变 token 排名 | 正温度改变差距，但不改变 logits 排名 |
| top-k 保留累计概率最高的候选 | top-k 固定保留概率排名最高的 $k$ 个 |
| top-p 固定保留 $p$ 个 token | $p$ 是累计概率阈值，候选数量是动态的 |
| top-p 不用排序 | 必须先按概率从大到小排序再累计 |
| top-p 应删除跨过阈值的 token | 必须保留第一个使累计概率达到或超过阈值的 token |
| greedy 一定得到整句概率最大的序列 | greedy 只保证每一步局部概率最大 |
| 采样参数能修复模型知识错误 | 它们只控制如何从已有分布选择输出 |
| token ID 方差能表示文本语义多样性 | token ID 是任意索引，其数值距离通常没有语义 |

## 核心记忆卡

```text
基础采样：
    x ~ Categorical(p)
    高概率更常出现，但单次结果不确定

累积区间：
    抽 U ~ Uniform(0,1)
    选择第一个 cumulative_probability > U 的 token

自回归：
    P(x1,...,xT) = Π P(xt | x<t)
    每采一个 token，都要重新计算下一步分布

temperature：
    p_i(T) = softmax(z_i / T)
    0<T<1：更尖锐
    T=1：不变
    T>1：更平坦
    T=0：不能代入公式，API 常特殊处理为 greedy

top-k：
    保留概率排名最高的固定 k 个 token

top-p：
    保留累积概率首次达到 p 的最小 token 集合
    候选数量会随分布变化

组合顺序：
    logits
      → temperature
      → top-k / top-p
      → 重新归一化
      → categorical sampling

频数：
    E[count_i] = N p_i
    Var(count_i) = N p_i(1-p_i)
```

掌握标准只保留四项：

1. 能用累积概率区间手算一次 categorical sampling
2. 能从公式和直觉解释 temperature 的作用及 $T=0$ 边界
3. 能手算 top-k、top-p 过滤和重新归一化
4. 能写出并解释 LLM 自回归采样的完整流程
