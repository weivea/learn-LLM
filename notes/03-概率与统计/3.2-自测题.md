## 🟢 A 组 · 核心概念

### A1 · 一句话定义

用自己的话解释：

1. 什么是 logits？
2. Softmax 做了什么？
3. 为什么 LLM 输出层需要 Softmax？

✍️ 你的答案：

1. logits是一组实属分数，用来体现token的注意力，或者token输出时的可能性高低
2. Softmax 将logits转换为总和为1的概率分布
3. 因为只看logits不是合法的离散概率分布，只看logits没法采样。 Softmax做的事情就是将logits 转换成合法的离散分布

> ⚠️ **纠正 A1.1**：整体直觉正确，但 logits 的定义应该更通用：**logits 是模型为一组互相竞争的候选项输出的、未经归一化的任意实数分数**。输出层的是“词表 logits”；注意力中通常称为“attention scores / 注意力分数”，不宜笼统说 logits 用来“体现 token 的注意力”。


### A2 · 公式与符号

写出：

$$
p_i=\operatorname{softmax}(\boldsymbol z)_i
$$

的完整公式，并解释：

1. $z_i$
2. $e^{z_i}$
3. 分母 $\sum_j e^{z_j}$
4. $p_i$

✍️ 你的答案：\
完整公式\
$$
\boxed{
p_i
=
\operatorname{softmax}(\boldsymbol z)_i
=
\frac{e^{z_i}}
{\sum_{j=1}^{K}e^{z_j}}
}
$$

1. $z_i$表示第i个类别的logit实数分数
2. $e^{z_i}$表示对第i个的logit值求指数
3. 分母 $\sum_j e^{z_j}$为所有logits的指数之和
4. $p_i$为第个类别的离散分布的出现概率

> ✅ **批改 A2**：公式与前三个解释正确。第 4 项更准确的表述是：$p_i$ 是**第 $i$ 个类别的概率**，整个向量 $\boldsymbol p$ 才是分类概率分布。


### A3 · 为什么是合法概率

不用只说“因为公式规定”，解释：

1. 为什么每个 softmax 输出都大于 $0$？
2. 为什么所有输出之和等于 $1$？
3. Softmax 输出可以恰好等于 $0$ 吗？数学上的答案和浮点计算中的现象可能有什么区别？

✍️ 你的答案：

1. 对任意有限实数 $z_i$，都有 $e^{z_i}>0$。分母 $\sum_j e^{z_j}$ 是若干正数之和，也大于 $0$，因此：

    $$
    p_i=\frac{e^{z_i}}{\sum_j e^{z_j}}>0
    $$

2. 所有 Softmax 输出之和为：

    $$
    \sum_i p_i
    =\sum_i\frac{e^{z_i}}{\sum_j e^{z_j}}
    =\frac{\sum_i e^{z_i}}{\sum_j e^{z_j}}
    =1
    $$

    因为所有分子相加后恰好等于共同的分母。

3. 数学上，只要所有 logits 都是有限实数，Softmax 输出就不可能恰好等于 $0$，只能无限接近 $0$。但在浮点计算中，如果某个指数值极小，例如 $e^{-1000}$，就可能发生下溢并被表示为 `0.0`，使对应的 Softmax 输出看起来恰好为 `0`。这是有限精度导致的现象，不是数学公式真的得到了 $0$。

> ✅ **批改 A3**：三问全部正确，且准确区分了数学结果与浮点下溢。

## 🟠 C 组 · NumPy 与张量

### C1 · 补全稳定实现

补全代码：

```python
def softmax(logits, axis=-1):
    shifted = logits - ______________________________
    exp_values = __________________
    return exp_values / ______________________________
```

解释两个 `keepdims=True` 分别为什么重要。

✍️ 你的答案：
```python
def softmax(logits, axis=-1):
    values = np.asarray(logits, dtype=np.float64)
    shifted = logits - np.max(values, axis=axis, keepdims=True)
    exp_values = np.exp(shifted)
    return exp_values / exp_values.sum(axis=axis, keepdims=True)
```

> ⚠️ **纠正 C1**：实现主体正确，但应使用转换后的 `values` 继续计算，并且漏答了 `keepdims=True` 的作用：
>
> ```python
> def softmax(logits, axis=-1):
>     values = np.asarray(logits, dtype=np.float64)
>     shifted = values - np.max(values, axis=axis, keepdims=True)
>     exp_values = np.exp(shifted)
>     return exp_values / exp_values.sum(axis=axis, keepdims=True)
> ```
>
> - 第一个 `keepdims=True` 保留被归一化的轴，使每组 logits 能减去自己的最大值。
> - 第二个 `keepdims=True` 让每组指数值能除以自己的总和。
> - 例如输入 shape 为 `(2, 3, 4)`，两个统计结果的 shape 都保持为 `(2, 3, 1)`，可沿最后一维广播。

### C6 · 输入检查

一个可靠的教学版 `softmax` 至少应该拒绝哪些输入？

1. 标量输入
2. 空数组
3. 包含 `NaN` 或 `Inf` 的 logits
4. `temperature=0`
5. `temperature=-1`

分别解释原因。

✍️ 你的答案：

题目列出的五种输入都应该拒绝：

1. **标量输入**：Softmax 需要一组相互竞争的 logits，并沿某个轴归一化。标量没有类别轴，无法表示分类概率分布。
2. **空数组**：空数组没有任何候选类别，既无法求最大值，也无法形成总和为 $1$ 的非空概率分布。
3. **包含 `NaN` 或 `Inf` 的 logits**：`NaN` 会传播到最大值、指数和最终概率中；`Inf` 在稳定实现中还可能导致 `Inf - Inf = NaN`，使计算结果失去明确意义。
4. **`temperature=0`**：带温度的 Softmax 需要计算 $z_i/T$，当 $T=0$ 时会发生除零，公式没有定义。
5. **`temperature=-1`**：负温度会反转 logits 的大小顺序，不符合温度只调节分布尖锐程度并保留排序的通常定义。

因此，教学版 Softmax 应要求 logits 是至少一维、非空且全部为有限值的数组，并要求温度满足：

$$
T>0
$$

> ✅ **批改 C6**：五种非法输入及原因全部解释正确。

### D1 · 一次 token 生成

按正确顺序排列：

1. 从概率分布中选择或采样一个 token

### D2 · Temperature 与生成

同一个模型、同一个前文、同一组 logits：

1. 降低温度通常如何影响生成？
2. 提高温度通常如何影响生成？
3. 温度是否会改变模型参数或给模型增加知识？
4. 温度很低时是否在数学上等于 `argmax`？
5. 为什么实际生成中还可能同时使用 top-k 或 top-p？

✍️ 你的答案：

1. 降低温度会放大 logits 之间的相对差距，使概率分布更集中。生成通常更保守、稳定，也更倾向于选择高概率 token，但这不等于一定提升准确性。
2. 提高温度会缩小 logits 之间的相对差距，使概率分布更平坦。低概率 token 更容易被采样，因此生成通常更多样、更随机，但这不等于提升模型的泛化能力。
3. 不会。温度只在生成时缩放 logits，不会修改模型参数，也不会给模型增加训练时没有学到的知识。
4. 不等于。只要温度是有限的正数，其他 token 通常仍有非零概率；只有当 $T\to0^+$ 时，Softmax 才趋近于把概率集中到最大 logit 上。`argmax` 则是直接、确定性地选择最大项。
5. 温度会调整整个概率分布的尖锐程度，但不会直接排除低概率 token。top-k 只保留概率最高的 $k$ 个候选，top-p 只保留累计概率达到阈值 $p$ 的最小候选集合，从而减少采到极低概率 token 的风险；它们可以和温度配合使用。

> ✅ **批改 D2**：五问全部正确，尤其准确区分了“有限低温”与 $T\to0^+$ 的极限。


### D3 · 输出层与注意力

补全并解释：

2. K，V匹配度候选项
3. 一个是输出token的概率分布，一个是当前位置对其他可见位置的相对关注权重

>
> 1. 输出层在整个词表的候选 token 之间分配概率。

## C 组答案

### C1

```python
def softmax(logits, axis=-1):
    shifted = logits - np.max(logits, axis=axis, keepdims=True)
    exp_values = np.exp(shifted)
    return exp_values / exp_values.sum(axis=axis, keepdims=True)
```

第一个 `keepdims=True` 保留归一化轴，使最大值可以按原位置广播回 logits；第二个保留同一轴，使每组指数值能除以自己的分母。

### C2

1. 小 logits 通常可以正常工作。
2. `np.exp` 可能溢出为 `inf`，`inf / inf` 产生 `nan`。
3. 先让 logits 减去沿目标轴的最大值，再取指数。
4. Softmax 对统一平移不变，因此结果在数学上相同。

### C3

1. `axis=-1`，即 `vocab_size` 维。
2. $4\times128=512$ 个分布。
3. 每个分布有 `50_000` 个候选概率。
4. shape 为 `(4, 128)`。
5. 每个值都应接近 `1`。

### C4

1. 沿 `batch_size` 维归一化。
2. `probabilities.sum(axis=0)` 会得到 `1`。
3. 它让不同样本相互竞争，没有让同一位置的 `50_000` 个 token 形成分布。
4. 合法的归一化方向必须与“哪些候选项互相竞争”的语义一致。

### C5

1. `(2, 3, 1)`。
2. 最后一维的 `1` 可以广播为 `4`，让每个 `(batch, position)` 减去自己的最大值。
3. 不保留维度时为 `(2, 3)`。
4. 不能按预期广播，因为从右向左比较时 `3` 与 `4` 不兼容。

### C6

1. 标量没有“一组互相竞争的类别”轴。
2. 空数组没有最大值，也无法形成分布。
3. `NaN` 或 `Inf` 会让最大值、指数或除法结果失去明确语义。
