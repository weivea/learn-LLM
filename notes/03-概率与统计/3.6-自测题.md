## 1 · 从语料计数理解条件概率（20 分）

$$

```python
import numpy as np

def conditional_distribution(counts):
    values = np.asarray(counts, dtype=np.float64)
    if values.ndim == 0 or values.size == 0:
        raise ValueError("counts 至少需要一个元素")
    if not np.all(np.isfinite(values)):
        raise ValueError("counts 不能包含 NaN 或 Inf")
    if np.any(values < 0.0):
        raise ValueError("counts 不能为负数")

    totals = values.sum(axis=-1, keepdims=True)
    if np.any(totals <= 0.0):
        raise ValueError("每个前文至少要有一次后续 token 计数")
    return values / totals

def sequence_metrics(target_conditional_probabilities):
    probabilities = np.asarray(
        target_conditional_probabilities,
        dtype=np.float64,
    )
    if probabilities.ndim != 1 or probabilities.size == 0:
        raise ValueError("条件概率必须是一维非空数组")
    if not np.all(np.isfinite(probabilities)):
        raise ValueError("条件概率不能包含 NaN 或 Inf")
    if np.any(probabilities <= 0.0) or np.any(
        probabilities > 1.0
    ):
        raise ValueError("条件概率必须满足 0 < p <= 1")

    log_probability = float(np.log(probabilities).sum())
    sequence_probability = float(np.exp(log_probability))
    total_nll = -log_probability
    mean_nll = total_nll / probabilities.size
    perplexity = float(np.exp(mean_nll))
    return sequence_probability, total_nll, mean_nll, perplexity
```

## 第 1 题

条件概率只在前文“我 喜欢”出现的 $50$ 次中计算：

$$
P(\text{学习}\mid\text{我 喜欢})
=
\frac{25}{50}
=
0.5
$$

$$
P(\text{编程}\mid\text{我 喜欢})
=
\frac{15}{50}
=
0.3
$$

$$
P(\text{休息}\mid\text{我 喜欢})
=
\frac{10}{50}
=
0.2
$$

概率和：

$$
0.5+0.3+0.2=1
$$

边缘概率使用全部 $200$ 个位置：

$$
P(\text{学习})
=
\frac{60}{200}
=
0.3
$$

加入“我 喜欢”这个条件后，样本范围改变，因此：

$$
P(\text{学习}\mid\text{我 喜欢})=0.5
\ne
P(\text{学习})=0.3
$$

一般定义：

$$
P(A\mid B)
=
\frac{P(A\cap B)}{P(B)},
\qquad P(B)>0
$$

分母 $P(B)$ 表示满足条件 $B$ 的样本范围。

## 第 2 题

概率链式分解：

$$
\begin{aligned}
P(\text{整段序列})
&=
P(\text{我}\mid\text{<BOS>})\\
&\quad\times
P(\text{喜欢}\mid\text{<BOS>, 我})\\
&\quad\times
P(\text{AI}\mid\text{<BOS>, 我, 喜欢})\\
&\quad\times
P(\text{<EOS>}\mid\text{<BOS>, 我, 喜欢, AI})\\
&=
0.5\times0.4\times0.25\times0.8\\
&=
0.04
\end{aligned}
$$

总 log-probability：

$$
\ln P(\text{整段序列})
\approx
-0.693-0.916-1.386-0.223
=
-3.218
$$

总 NLL：

$$
\mathcal L_{\text{total}}
\approx
3.218
$$

平均 NLL：

$$
\overline{\mathcal L}
\approx
\frac{3.218}{4}
\approx
0.805
$$

PPL：

$$
\operatorname{PPL}
=
e^{0.805}
\approx
2.237
$$

某一步条件概率非常小时，连乘得到的序列概率会显著减小；其负对数很大，因此总 NLL 会明显增加。

概率链式法则用于分解联合概率；求导链式法则用于复合函数求导。二者不是同一条公式。

## 第 3 题

概念上的 shift：

```text
模型输入：[<BOS>, 我,   喜欢, AI]
预测目标：[我,    喜欢, AI,   <EOS>]
```

目标为 `AI` 时，模型只允许看到：

```text
[<BOS>, 我, 喜欢]
```

如果它看到 `AI` 或 `<EOS>`，就使用了目标本身或未来信息，产生答案泄漏，不再是在估计合法的：

$$
P(\text{AI}\mid\text{<BOS>, 我, 喜欢})
$$

causal mask 为每个位置屏蔽右侧未来 token，所以所有位置可以在同一次矩阵运算中并行计算，同时保持各自的因果约束。

`(2, 4, 10000)` 表示：

```text
2     = batch_size
4     = seq_len / 需要预测的位置数
10000 = vocab_size
```

Softmax 沿最后的 `vocab_size` 维执行，使每个 batch、每个位置的 $10000$ 个候选概率之和为 $1$。

## 第 5 题

```python
import numpy as np


def conditional_distribution(counts):
    values = np.asarray(counts, dtype=np.float64)
    if values.ndim == 0 or values.size == 0:
        raise ValueError("counts 至少需要一个元素")
    if not np.all(np.isfinite(values)):
        raise ValueError("counts 不能包含 NaN 或 Inf")
    if np.any(values < 0.0):
        raise ValueError("counts 不能为负数")

    totals = values.sum(axis=-1, keepdims=True)
    if np.any(totals <= 0.0):
        raise ValueError("每个前文至少要有一次计数")
    return values / totals


def sequence_metrics(target_conditional_probabilities):
    probabilities = np.asarray(
        target_conditional_probabilities,
        dtype=np.float64,
    )
    if probabilities.ndim != 1 or probabilities.size == 0:
        raise ValueError("条件概率必须是一维非空数组")
    if not np.all(np.isfinite(probabilities)):
        raise ValueError("条件概率不能包含 NaN 或 Inf")
    if np.any(probabilities <= 0.0) or np.any(
        probabilities > 1.0
    ):
        raise ValueError("条件概率必须满足 0 < p <= 1")

    log_probability = np.log(probabilities).sum()
    sequence_probability = np.exp(log_probability)
    total_nll = -log_probability
    mean_nll = total_nll / probabilities.size
    perplexity = np.exp(mean_nll)
    return (
        sequence_probability,
        total_nll,
        mean_nll,
        perplexity,
    )
```

验证：

```python
counts = np.array([[6, 3, 1], [1, 2, 7]])
distribution = conditional_distribution(counts)

assert np.allclose(
    distribution,
    [[0.6, 0.3, 0.1], [0.1, 0.2, 0.7]],
)

metrics = sequence_metrics([0.5, 0.4, 0.25, 0.8])
assert np.isclose(metrics[0], 0.04)
```

长序列中很多小于 $1$ 的概率直接连乘，可能发生浮点下溢并错误变成 `0.0`。稳定做法是先对每个条件概率取 log，再求和；只在确实需要普通尺度的序列概率时执行一次 `exp`。
