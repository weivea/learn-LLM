# 3.6 条件概率：P(下一个 token | 前文)

> 重要度：⭐⭐⭐（自回归语言模型真正建模的目标）
> 目标：理解条件概率的含义；会用概率链式法则分解序列概率；能解释 LLM 的训练错位与自回归生成

## 一句话总结

**条件概率表示“已知前文后，下一个 token 出现的概率”；LLM 每一步都根据当前前文输出一个新的词表概率分布。**

$$
\boxed{
p_\theta(x_t\mid x_{<t})
}
$$

读作：

> 在前文 $x_{<t}$ 已知的条件下，下一个 token 是 $x_t$ 的概率。

其中：

- $x_t$：第 $t$ 个实际出现的 token
- $x_{<t}$：第 $t$ 个 token 之前的所有 token
- $\theta$：模型学到的全部参数
- 竖线 $\mid$：读作“在……条件下”

```text
当前前文
    ↓ LLM
整个词表的下一个 token 概率分布
    ↓ 选择或采样一个 token
把新 token 追加到前文，再预测下一步
```

---

## 1. 为什么需要“条件”

一个词没有脱离上下文的固定“下一个词概率”。

```text
前文 A：我 喜欢 吃
候选：苹果 0.6，香蕉 0.4，代码 0.0

前文 B：我 喜欢 写
候选：苹果 0.0，文章 0.3，代码 0.7
```

同一个候选 token“苹果”，因为前文不同，概率也不同：

$$
P(\text{苹果}\mid\text{我 喜欢 吃})
\ne
P(\text{苹果}\mid\text{我 喜欢 写})
$$

所以语言模型不是在记一个固定的：

$$
P(\text{苹果})
$$

而是在每个位置估计：

$$
P(\text{苹果}\mid\text{当前前文})
$$

这就是上下文会改变预测的数学表达。

---

## 2. 条件概率的基础定义

对事件 $A$ 和 $B$，当 $P(B)>0$ 时：

$$
\boxed{
P(A\mid B)
=
\frac{P(A\cap B)}{P(B)}
}
$$

含义是：已知 $B$ 已经发生后，只在满足 $B$ 的情况中，计算 $A$ 所占的比例。

### 2.1 用语料计数理解

假设语料中：

- 前文“我 喜欢 吃”共出现 $10$ 次
- 其中后面接“苹果”共出现 $6$ 次

令：

```text
B = 前文是“我 喜欢 吃”
A = 下一个 token 是“苹果”
```

那么：

$$
P(A\mid B)
=
\frac{6}{10}
=
0.6
$$

这里的分母不再是全部语料，而是被条件 $B$ 筛选后的 $10$ 次。

```text
全部样本
   ↓ 加入条件 B
只保留“我 喜欢 吃”出现的样本
   ↓ 在这些样本中统计 A
60% 的下一词是“苹果”
```

### 2.2 乘法形式

把定义移项可得：

$$
\boxed{
P(A\cap B)
=
P(B)P(A\mid B)
}
$$

也就是：

> “$B$ 发生，并且随后 $A$ 也发生”的联合概率
> = “$B$ 的概率” × “已知 $B$ 后 $A$ 的概率”。

这会直接导出语言模型的序列概率。

---

## 3. 每个前文对应一个词表概率分布

设词表为 $\mathcal V$。给定同一个前文 $x_{<t}$，模型要为词表中的每个候选 token $v$ 给出概率：

$$
P(X_t=v\mid X_{<t}=x_{<t})
$$

所有候选概率必须满足：

$$
\boxed{
P(X_t=v\mid X_{<t}=x_{<t})\ge0
}
$$

$$
\boxed{
\sum_{v\in\mathcal V}
P(X_t=v\mid X_{<t}=x_{<t})
=
1
}
$$

例如：

| 候选 token | $P(\text{候选}\mid\text{我 喜欢 吃})$ |
|---|---:|
| 苹果 | 0.60 |
| 香蕉 | 0.30 |
| 面条 | 0.10 |
| **合计** | **1.00** |

注意求和的方向：

> 固定一个前文，对**整个词表维度**的概率求和等于 $1$。

在代码中：

```text
logits.shape        = (batch, seq_len, vocab_size)
probabilities.shape = (batch, seq_len, vocab_size)
```

Softmax 沿最后的 `vocab_size` 维执行。于是每个 batch、每个位置都有一套独立的下一个 token 条件分布。

## 4. 概率链式法则：整段文本如何拆成逐 token 预测

对序列：

$$
x_1,x_2,\ldots,x_T
$$

联合概率可以逐步分解：

$$
\boxed{
P(x_1,x_2,\ldots,x_T)
=
\prod_{t=1}^{T}
P(x_t\mid x_{<t})
}
$$

展开就是：

$$
\begin{aligned}
P(x_1,x_2,\ldots,x_T)
&=
P(x_1)\\
&\quad\times P(x_2\mid x_1)\\
&\quad\times P(x_3\mid x_1,x_2)\\
&\quad\times\cdots\\
&\quad\times P(x_T\mid x_1,\ldots,x_{T-1})
\end{aligned}
$$

第一个 token 没有普通前文，可以把它写成以 `<BOS>`（序列开始标记）为条件。

### 4.1 手算例子

假设模型给出：

$$
\begin{aligned}
P(\text{我}\mid\text{<BOS>})&=0.5\\
P(\text{喜欢}\mid\text{<BOS>, 我})&=0.4\\
P(\text{AI}\mid\text{<BOS>, 我, 喜欢})&=0.25\\
P(\text{<EOS>}\mid\text{<BOS>, 我, 喜欢, AI})&=0.8
\end{aligned}
$$

完整序列“我 喜欢 AI `<EOS>`”的概率为：

$$
\begin{aligned}
P(\text{整段序列})
&=
0.5\times0.4\times0.25\times0.8\\
&=
0.04
\end{aligned}
$$

序列越长，很多不大于 $1$ 的条件概率连乘后通常越小。因此实际计算会进入 log 空间。

> 这里是**概率的链式法则**；它与 2.4 中求导用的链式法则不是同一条公式。

## 5. LLM 训练：输入与目标错开一个 token

训练自回归语言模型时，概念上把同一个序列错开一位：

```text
原序列：  [<BOS>, 我,   喜欢, AI,    <EOS>]
模型输入：[<BOS>, 我,   喜欢, AI]
预测目标：[我,    喜欢, AI,   <EOS>]
```

每个位置都在做下一个 token 预测：

| 模型可见前文 | 该位置目标 |
|---|---|
| `<BOS>` | 我 |
| `<BOS>, 我` | 喜欢 |
| `<BOS>, 我, 喜欢` | AI |
| `<BOS>, 我, 喜欢, AI` | `<EOS>` |

于是该序列的训练损失为：

$$
\boxed{
\mathcal L_{\text{total}}
=
-\sum_{t=1}^{T}
\ln p_\theta(x_t\mid x_{<t})
}
$$

这就是之前学过的逐 token NLL 之和，只是现在明确了概率 $p_t$ 的完整含义：

$$
p_t
=
p_\theta(x_t\mid x_{<t})
$$

### 5.1 为什么一次前向传播能同时训练所有位置

公式看起来是从左到右逐步预测，但 Transformer 训练时可以并行计算各位置。关键是使用 **causal mask（因果遮罩）**：

```text
位置 t 可以看：x_1 ... x_t
位置 t 不可看：x_(t+1) ... x_T
```

这样每个位置虽然并行计算，却仍然不能偷看未来答案。

如果预测 $x_t$ 时看到了 $x_t$ 或更后面的 token，就发生了未来信息泄漏，计算的不再是合法的：

$$
P(x_t\mid x_{<t})
$$

不同框架可能由调用者手动错开输入和标签，也可能在损失函数内部完成 shift；数学目标不变。

## 6. LLM 生成：不断把输出追加到条件中

生成时没有现成的正确下一个 token，模型要循环执行：

```text
第 1 步：P(· | 用户前文)
        → 选出 token x_1

第 2 步：P(· | 用户前文, x_1)
        → 选出 token x_2

第 3 步：P(· | 用户前文, x_1, x_2)
        → 选出 token x_3
```

伪代码：

```python
tokens = prompt_tokens.copy()

while not finished:
    logits = model(tokens)
    next_distribution = softmax(logits[-1])
    next_token = sample(next_distribution)
    tokens.append(next_token)
```

这种“用自己刚生成的 token 作为下一步条件”的方式叫：

> **自回归生成（autoregressive generation）**

需要区分两个对象：

- 条件概率分布：模型对整个词表的预测
- 生成的 token：从该分布中选择或采样出的一个具体结果

生成一个 token 后，前文改变，下一步的整个条件分布也会改变。

## 7. 串回 NLL、交叉熵与困惑度

概率链式法则：

$$
P(x_{1:T})
=
\prod_{t=1}^{T}
p_\theta(x_t\mid x_{<t})
$$

取自然对数，把乘积变成求和：

$$
\ln P(x_{1:T})
=
\sum_{t=1}^{T}
\ln p_\theta(x_t\mid x_{<t})
$$

加负号得到序列总 NLL：

$$
\boxed{
-\ln P(x_{1:T})
=
-\sum_{t=1}^{T}
\ln p_\theta(x_t\mid x_{<t})
}
$$

对有效 token 求平均：

$$
\overline{\mathcal L}
=
-\frac1N
\sum_{t\in\mathcal I_{\text{valid}}}
\ln p_\theta(x_t\mid x_{<t})
$$

其中 $\mathcal I_{\text{valid}}$ 是需要计分的位置集合，$N$ 是其中的有效 token 数。

再取指数就是困惑度：

$$
\boxed{
\operatorname{PPL}
=
\exp(\overline{\mathcal L})
}
$$

完整链路：

```text
前文 x_<t
    ↓ 模型 + Softmax
P(整个词表 | x_<t)
    ↓ 取实际 token x_t 的概率
p(x_t | x_<t)
    ↓ -ln
逐 token NLL
    ↓ 有效位置求平均
平均 NLL
    ↓ exp
PPL
```

因此 3.1–3.6 实际上描述的是同一条训练链路的不同部分。

## 8. 计数概率与神经语言模型的区别

在小例子中，可以直接使用频数：

$$
\hat P(A\mid B)
=
\frac{\operatorname{count}(A\cap B)}
{\operatorname{count}(B)}
$$

但真实前文组合几乎无穷多。一个长前文可能从未在训练数据中原样出现，纯计数法会遇到严重稀疏问题。

神经语言模型学习的是参数化近似：

$$
p_\theta(x_t\mid x_{<t})
$$

它通过 embedding、注意力和神经网络参数，在相似上下文之间共享统计规律。因此即使某个完整提示词没有原样出现过，模型通常仍能输出一个分布。

需要保持边界意识：

- 模型输出的是从训练数据中学到的**估计值**，不是真理
- 条件概率高不代表事实一定正确
- $P(A\mid B)$ 不等于 $P(B\mid A)$
- 条件相关不自动代表因果关系
- 上下文窗口有限时，模型实际只能使用窗口内可见的前文

## 9. NumPy 核心实现

### 9.1 从计数得到条件分布

假设每一行对应一个前文，每一列对应一个候选 token：

```python
counts = np.array(
    [
        [6, 3, 1],
        [1, 2, 7],
    ],
    dtype=np.float64,
)

conditional_probabilities = (
    counts / counts.sum(axis=-1, keepdims=True)
)
```

结果每一行之和都是 $1$：

```python
assert np.allclose(
    conditional_probabilities.sum(axis=-1),
    1.0,
)
```

### 9.2 从 logits 得到每个位置的条件分布

```python
def softmax(logits, axis=-1):
    shifted = logits - np.max(
        logits,
        axis=axis,
        keepdims=True,
    )
    exp_values = np.exp(shifted)
    return exp_values / exp_values.sum(
        axis=axis,
        keepdims=True,
    )
```

如果：

```text
logits.shape == (batch, seq_len, vocab_size)
```

则：

```python
probabilities = softmax(logits, axis=-1)
```

满足：

```python
assert np.allclose(
    probabilities.sum(axis=-1),
    1.0,
)
```

## 10. 常见误区

| 错误说法 | 正确理解 |
|---|---|
| $P(A\mid B)=P(B\mid A)$ | 条件方向通常不可交换 |
| 一个 token 有固定概率 | 概率取决于当前前文 |
| 对所有序列位置的概率求和等于 1 | 固定一个位置和前文，对词表维求和等于 1 |
| 整段序列概率是逐 token 概率相加 | 使用概率链式法则连乘 |
| 训练时当前位置可以看到未来 token | causal mask 必须阻止未来信息泄漏 |
| 一次前向只能训练一个位置 | causal mask 允许所有位置并行预测 |
| 生成后条件不变 | 新 token 会被追加到前文，下一步分布随之改变 |
| 条件概率高就代表事实正确 | 它只是模型学到的概率估计 |
| 概率链式法则就是求导链式法则 | 名称相同，但对象和公式不同 |

## 核心记忆卡

```text
基础定义：
    P(A | B) = P(A ∩ B) / P(B)
    P(A ∩ B) = P(B) P(A | B)

语言模型：
    p_theta(x_t | x_<t)
    = 已知前文时，实际下一个 token 的概率

条件分布：
    固定前文，对整个词表求和等于 1

序列概率：
    P(x_1:T) = product_t P(x_t | x_<t)

训练：
    输入与目标错开一个 token
    causal mask 阻止看到未来

生成：
    预测分布 → 选一个 token → 追加到前文 → 再预测

损失：
    total NLL = -sum_t ln P(x_t | x_<t)
    PPL = exp(mean NLL)
```

掌握标准只保留四项：

1. 能用计数解释并计算 $P(A\mid B)$
2. 能写出语言模型的概率链式分解并手算短序列概率
3. 能解释训练时的 one-token shift、causal mask 和 logits 词表维
4. 能把条件概率串回逐 token NLL、PPL 与自回归生成
