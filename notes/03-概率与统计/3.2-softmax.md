## 一句话总结

**Softmax 把一组任意实数分数 `logits` 转换成总和为 1 的概率分布，同时保留分数的大小顺序。**

```text
神经网络输出 logits
        ↓
对每个分数取指数
        ↓
除以所有指数值之和
        ↓
得到概率分布 probabilities
```

例如：

```text
logits:        [2.0, 1.0, 0.0]
softmax:       [0.6652, 0.2447, 0.0900]
概率总和:       1.0000
```

---

## 1. 为什么需要 Softmax

LLM 在预测下一个 token 时，神经网络先为词表中的每个 token 输出一个分数：

| token | 模型 | 学习 | 天气 |
|---|---:|---:|---:|
| logits | 2.0 | 1.0 | 0.0 |

这些分数叫 **logits**。它们：

- 可以是任意实数
- 可以为负数
- 可以大于 $1$
- 总和不必等于 $1$
- 还不能直接当作概率

但 3.1 学过，合法的离散概率分布必须满足：

$$
p_i\ge0
$$

$$
\sum_{i=1}^{K}p_i=1
$$

Softmax 的任务就是完成这个转换：

$$
\text{任意实数 logits}
\xrightarrow{\text{softmax}}
\text{合法分类概率分布}
$$

---

## 2. Softmax 公式

给定 $K$ 个 logits：

$$
\boldsymbol z
=
\begin{bmatrix}
z_1&z_2&\cdots&z_K
\end{bmatrix}
$$

第 $i$ 个类别的 softmax 概率为：

$$
\boxed{
p_i
=
\operatorname{softmax}(\boldsymbol z)_i
=
\frac{e^{z_i}}
{\sum_{j=1}^{K}e^{z_j}}
}
$$

符号含义：

| 符号 | 含义 |
|---|---|
| $\boldsymbol z$ | 整个 logits 向量 |
| $z_i$ | 第 $i$ 个类别的 logit |
| $K$ | 类别总数；LLM 输出层中通常是词表大小 |
| $e^{z_i}$ | 第 $i$ 个 logit 的指数值 |
| $\sum_j e^{z_j}$ | 所有指数值之和，即归一化分母 |
| $p_i$ | 第 $i$ 个类别转换后的概率 |

注意：softmax 是对**整个向量联合计算**，不是分别对每个标量独立计算。

## 3. 手算一个 Softmax

给定：

$$
\boldsymbol z=[2,1,0]
$$

### 第一步：取指数

$$
e^2\approx7.389,\qquad e^1\approx2.718,\qquad e^0=1
$$

### 第二步：计算总和

$$
S=e^2+e^1+e^0
\approx7.389+2.718+1
=11.107
$$

### 第三步：每项除以总和

$$
p_1=\frac{7.389}{11.107}\approx0.6652
$$

$$
p_2=\frac{2.718}{11.107}\approx0.2447
$$

$$
p_3=\frac{1}{11.107}\approx0.0900
$$

所以：

$$
\operatorname{softmax}([2,1,0])
\approx
[0.6652,0.2447,0.0900]
$$

检查：

$$
0.6652+0.2447+0.0900\approx1
$$

最大的 logit 是 `2`，转换后仍对应最大的概率。

---

## 4. 为什么公式能得到概率

### 4.1 指数保证每一项为正

对任意实数 $z_i$：

$$
e^{z_i}>0
$$

所以 softmax 的分子一定为正，最终概率也一定为正。

### 4.2 除以总和完成归一化

$$
\sum_{i=1}^{K}p_i
=
\sum_{i=1}^{K}
\frac{e^{z_i}}{\sum_{j=1}^{K}e^{z_j}}
$$

$$
=
\frac{\sum_{i=1}^{K}e^{z_i}}
{\sum_{j=1}^{K}e^{z_j}}
=1
$$

### 4.3 指数函数保留大小顺序

指数函数严格递增，因此：

$$
z_a>z_b
\quad\Longrightarrow\quad
e^{z_a}>e^{z_b}
\quad\Longrightarrow\quad
p_a>p_b
$$

所以：

$$
\boxed{
\arg\max_i z_i
=
\arg\max_i p_i
}
$$

Softmax 改变数值的表示方式，但不会改变最大项的位置。

---

## 5. Softmax 关心的是相对差距

### 5.1 所有 logits 加同一个常数，结果不变

给所有 logits 加上常数 $c$：

$$
\operatorname{softmax}(\boldsymbol z+c)_i
=
\frac{e^{z_i+c}}{\sum_j e^{z_j+c}}
$$

因为：

$$
e^{z_i+c}=e^{z_i}e^c
$$

所以：

$$
\frac{e^{z_i}e^c}{\sum_j e^{z_j}e^c}
=
\frac{e^{z_i}}{\sum_j e^{z_j}}
$$

因此：

$$
\boxed{
\operatorname{softmax}(\boldsymbol z+c)
=
\operatorname{softmax}(\boldsymbol z)
}
$$

例如：

```text
softmax([2, 1, 0])
= softmax([102, 101, 100])
= softmax([0, -1, -2])
```

三组 logits 的绝对值不同，但任意两项之间的差距相同。

### 5.2 乘同一个常数通常会改变结果

```text
softmax([2, 1, 0])      ≈ [0.6652, 0.2447, 0.0900]
softmax([4, 2, 0])      ≈ [0.8668, 0.1173, 0.0159]
```

第二组的差距被放大，因此分布更集中。

> **平移不改变 softmax；缩放通常会改变 softmax。**

## 6. 数值稳定性：为什么要减去最大值

直接实现公式：

```python
exp_values = np.exp(logits)
probabilities = exp_values / exp_values.sum()
```

在数学上正确，但计算机中可能溢出。例如：

```text
logits = [1000, 1001, 1002]
```

`exp(1000)` 已经远超常见浮点数可表示的范围，会得到 `inf`，继续计算可能产生 `nan`。

利用上一节的平移不变性，可以让所有 logits 减去最大值：

$$
m=\max_i z_i
$$

$$
\operatorname{softmax}(\boldsymbol z)
=
\operatorname{softmax}(\boldsymbol z-m)
$$

例子：

```text
原 logits：       [1000, 1001, 1002]
减去最大值 1002： [-2,   -1,    0]
```

此时最大的指数值为：

$$
e^0=1
$$

不会发生正向溢出。

稳定实现：

```python
def softmax(logits, axis=-1):
    shifted = logits - np.max(logits, axis=axis, keepdims=True)
    exp_values = np.exp(shifted)
    return exp_values / exp_values.sum(axis=axis, keepdims=True)
```

核心记忆：

$$
\boxed{
\text{stable softmax}
=
\frac{e^{z_i-\max(\boldsymbol z)}}
{\sum_j e^{z_j-\max(\boldsymbol z)}}
}
$$

---

## 7. 温度参数 Temperature

带温度 $T$ 的 softmax：

$$
\boxed{
p_i(T)
=
\frac{e^{z_i/T}}
{\sum_j e^{z_j/T}},
\qquad T>0
}
$$

对 `[2, 1, 0]`：

| 温度 $T$ | softmax 概率（约） | 效果 |
|---:|---|---|
| $0.5$ | `[0.8668, 0.1173, 0.0159]` | 更尖锐、更集中 |
| $1.0$ | `[0.6652, 0.2447, 0.0900]` | 原始分布 |
| $2.0$ | `[0.5065, 0.3072, 0.1863]` | 更平坦；采样时更随机 |

直觉：

```text
T < 1：logits 除以小数 → 差距放大 → 高分项更突出
T = 1：普通 softmax
T > 1：logits 除以大数 → 差距缩小 → 概率更平均
```

极限直觉：

$$
T\to0^+
\quad\Longrightarrow\quad
\text{概率集中到最大 logit}
$$

$$
T\to+\infty
\quad\Longrightarrow\quad
\text{趋近均匀分布}
$$

只要 $T>0$，温度不会改变 logits 的大小顺序，所以最大概率项的位置不变；它改变的是概率有多集中。

> 温度主要用于控制生成时的随机性，不是让模型“学到更多知识”。

## 8. `axis`：到底沿哪一维做 Softmax

### 8.1 单个向量

```python
logits.shape == (vocab_size,)
probabilities = softmax(logits, axis=-1)
```

沿唯一的词表维计算。

### 8.2 一批 token 位置

LLM 输出层的 logits 常见形状：

```text
(batch_size, seq_len, vocab_size)
```

需要让**每个样本的每个位置**都在整个词表上形成一个概率分布：

```python
probabilities = softmax(logits, axis=-1)
```

此时：

```python
probabilities.sum(axis=-1).shape == (batch_size, seq_len)
```

而且这些和都应接近 `1`。

为什么需要 `keepdims=True`：

```text
logits:                    (batch, seq_len, vocab_size)
max(..., keepdims=True):   (batch, seq_len, 1)
```

保留长度为 `1` 的词表轴后，NumPy 才能利用广播，让每个位置减去自己的最大 logit。

### 8.3 轴选错会发生什么

如果误写：

```python
softmax(logits, axis=0)
```

就会在不同 batch 样本之间归一化，而不是在候选 token 之间归一化。数值仍可能“看起来像概率”，但语义已经错了。

## 9. Softmax 在 LLM 中的两个核心位置

### 9.1 输出层：得到下一个 token 分布

简化流程：

```text
隐藏状态 hidden states
        ↓ 线性层
词表 logits
shape = (batch, seq_len, vocab_size)
        ↓ softmax(axis=-1)
词表概率分布
        ↓ 选择 / 采样
下一个 token
```

例如前文：

```text
我正在学习大
```

输出层可能得到：

| token | 模型 | 学 | 雨 | 。 |
|---|---:|---:|---:|---:|
| logit | 2.4 | 1.1 | -0.3 | 0.7 |
| probability | 0.657 | 0.179 | 0.044 | 0.120 |

这些概率是在当前前文条件下，对整个词表的离散分类分布。

### 9.2 注意力：得到关注权重

Scaled Dot-Product Attention：

$$
\operatorname{Attention}(Q,K,V)
=
\operatorname{softmax}
\left(
\frac{QK^\mathsf T}{\sqrt{d_k}}
\right)V
$$

其中：

1. $QK^\mathsf T$ 产生注意力分数
2. 除以 $\sqrt{d_k}$ 控制分数尺度
3. softmax 把每一行分数变成总和为 $1$ 的注意力权重
4. 权重再对 $V$ 做加权求和

注意力中的概率不是“下一个 token 的概率”，而是当前位置对其他可见位置的**相对关注权重**。

实际的因果注意力还会在不可见位置加入很大的负数（框架中也常表示为 `-inf`），使这些位置经过 softmax 后得到 `0` 或接近 `0` 的权重。本节的 NumPy 教学实现只接收有限 logits，先专注掌握普通 softmax。

## 10. Softmax 的导数直觉

这一部分先建立直觉，不要求立即默写。

Softmax 的导数为：

$$
\frac{\partial p_i}{\partial z_j}
=
p_i(\delta_{ij}-p_j)
$$

其中 $\delta_{ij}$ 是 Kronecker delta：

$$
\delta_{ij}
=
\begin{cases}
1,&i=j\\
0,&i\ne j
\end{cases}
$$

所以：

### 自己对自己

$$
\frac{\partial p_i}{\partial z_i}
=
p_i(1-p_i)\ge0
$$

当 $0<p_i<1$ 时，提高某个 logit 会提高它自己的概率。

### 自己对其他项

$$
\frac{\partial p_i}{\partial z_j}
=
-p_ip_j<0,
\qquad i\ne j
$$

提高另一个类别的 logit，会压低当前类别的概率，因为所有概率必须共同竞争总量 $1$。

这说明 softmax 的各项不是彼此独立的。

## 11. Softmax 和 Sigmoid 不一样

| 项目 | Softmax | Sigmoid |
|---|---|---|
| 输入 | 一组 logits | 每个 logit 可独立处理 |
| 输出关系 | 类别之间竞争 | 各输出可相互独立 |
| 输出总和 | 必须为 $1$ | 不要求为 $1$ |
| 常见任务 | 单标签多分类、token 预测 | 二分类、多标签分类 |

假设一张图只能属于“猫、狗、鸟”中的一个类别，适合 softmax。

假设一张图可以同时包含“人、汽车、树”，每个标签可以独立存在，更适合逐项 sigmoid。

## 12. 常见误区

### 误区 1：直接用 logits 除以 logits 总和

错。例如：

```text
logits = [-2, 1, 3]
```

直接除以总和会保留负数，不能得到合法概率。Softmax 先取指数，保证每项为正。

### 误区 2：Softmax 只取最大值

错。`argmax` 只返回最大项；softmax 会为**所有类别**分配概率。非最大项通常仍有正概率。

### 误区 3：所有 logits 加 100，模型会更确定

错。统一平移不改变 softmax。要改变分布的尖锐程度，需要改变 logits 之间的差距，例如缩放或使用温度。

### 误区 4：温度越高，模型越确定

恰好相反。温度越高，分布通常越平坦；温度越低，分布越集中。

### 误区 5：直接写 `np.exp(logits)` 总是安全

错。大 logits 会导致浮点溢出。稳定实现必须先减去该分布中的最大值。

### 误区 6：只要最后有一维，就一定对最后一维做 Softmax

不一定。必须根据语义判断哪个轴代表互相竞争的类别。LLM 输出层中通常是 `vocab_size`，所以常用 `axis=-1`。

### 误区 7：注意力 Softmax 就是 token 输出概率

错。两处使用相同的数学函数，但对象不同：

- 输出层：词表候选 token 之间竞争
- 注意力：可见位置之间竞争

## 13. NumPy 从零实现

```python
import numpy as np


def softmax(
    logits: np.ndarray,
    axis: int = -1,
    temperature: float = 1.0,
) -> np.ndarray:
    values = np.asarray(logits, dtype=np.float64)

    if values.ndim == 0:
        raise ValueError("softmax 至少需要一维输入")
    if values.size == 0:
        raise ValueError("softmax 不能处理空输入")
    if not np.all(np.isfinite(values)):
        raise ValueError("logits 不能包含 NaN 或 Inf")
    if not np.isfinite(temperature) or temperature <= 0.0:
        raise ValueError("temperature 必须是有限正数")

    maximum = np.max(values, axis=axis, keepdims=True)
    shifted = (values - maximum) / temperature
    exp_values = np.exp(shifted)
    denominator = exp_values.sum(axis=axis, keepdims=True)
    return exp_values / denominator
```

使用：

```python
logits = np.array([2.0, 1.0, 0.0])
probabilities = softmax(logits)

assert np.all(probabilities > 0.0)
assert np.isclose(probabilities.sum(), 1.0)
assert np.argmax(probabilities) == np.argmax(logits)
```

## 核心记忆卡

```text
输入：
    logits 是任意实数分数，不是概率

公式：
    p_i = exp(z_i) / Σ_j exp(z_j)

结果：
    p_i > 0
    Σ_i p_i = 1
    argmax(probabilities) = argmax(logits)

稳定实现：
    shifted = logits - max(logits)
    probabilities = exp(shifted) / sum(exp(shifted))

温度：
    softmax(logits / T)
    T 低 → 更尖锐
    T 高 → 更平坦

LLM：
    输出层沿 vocab_size 做 softmax
    注意力沿候选位置做 softmax
```
