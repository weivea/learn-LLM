# 3.5 困惑度（perplexity）

> 重要度：⭐⭐⭐（语言模型最常见的概率评估指标之一）
> 目标：能从平均 NLL 计算困惑度；能解释其直觉、正确使用 mask，并判断两个 PPL 是否可以直接比较
> 前置：3.1 概率分布 · 3.3 交叉熵 / NLL · 3.4 对数
> 配套脚本：[`examples/3_5_perplexity.py`](../../examples/3_5_perplexity.py)
> 配套自测：[`3.5-自测题.md`](3.5-自测题.md)（仅 5 道综合题）

## 一句话总结

**困惑度是平均 token NLL 的指数：平均损失越小，困惑度越低，语言模型对真实文本越不“困惑”。**

使用自然对数时：

$$
\boxed{
\operatorname{PPL}
=
e^{\overline{\mathcal L}_{\mathrm{NLL}}}
}
$$

也可以写成：

$$
\boxed{
\operatorname{PPL}
=
\exp\left(
-\frac1N\sum_{t=1}^{N}\ln p_t
\right)
}
$$

其中 $p_t$ 是模型在第 $t$ 个有效位置分给**实际出现 token** 的概率。

```text
真实 token 概率高
    → 平均 NLL 小
    → PPL 小
    → 模型在这批文本上预测得更好
```

---

## 1. 从平均 NLL 到困惑度

上一节已经学过单个 token 的 NLL：

$$
\mathcal L_t=-\ln p_t
$$

对 $N$ 个有效 token 求平均：

$$
\overline{\mathcal L}
=
-\frac1N\sum_{t=1}^{N}\ln p_t
$$

这个数在 log 尺度上，直觉不够直接。对它执行自然对数的逆运算 $e^x$：

$$
\operatorname{PPL}
=
e^{\overline{\mathcal L}}
$$

因此，PPL 并不是一种新的独立损失，而是把平均 NLL 转换回普通尺度。

由于指数函数单调递增：

$$
\overline{\mathcal L}_A
<
\overline{\mathcal L}_B
\quad\Longleftrightarrow\quad
\operatorname{PPL}_A
<
\operatorname{PPL}_B
$$

所以在相同评估条件下：

> **平均 NLL 越低，PPL 也越低；模型越好。**

训练时通常直接最小化平均 NLL，而不是先计算 PPL。两者优化方向完全相同，但 NLL 的数值和梯度更方便处理。

---

## 2. 一个可以手算的例子

三个位置的真实 token 概率分别为：

$$
[0.5,\ 0.25,\ 0.125]
$$

逐 token NLL：

$$
\begin{aligned}
-\ln0.5&\approx0.693\\
-\ln0.25&\approx1.386\\
-\ln0.125&\approx2.079
\end{aligned}
$$

平均 NLL：

$$
\overline{\mathcal L}
=
\frac{0.693+1.386+2.079}{3}
\approx1.386
$$

因此：

$$
\operatorname{PPL}
=
e^{1.386}
\approx4
$$

注意，不能先求三个概率的算术平均：

$$
\frac{0.5+0.25+0.125}{3}
\ne
\frac1{\operatorname{PPL}}
$$

PPL 对应的是**几何平均**。

---

## 3. PPL 为什么是几何平均概率的倒数

从定义开始：

$$
\operatorname{PPL}
=
\exp\left(
-\frac1N\sum_{t=1}^{N}\ln p_t
\right)
$$

利用 $\sum\ln p_t=\ln\prod p_t$：

$$
\begin{aligned}
\operatorname{PPL}
&=
\exp\left(
-\ln\left[
\left(\prod_{t=1}^{N}p_t\right)^{1/N}
\right]
\right)\\
&=
\frac1{
\left(\prod_{t=1}^{N}p_t\right)^{1/N}
}
\end{aligned}
$$

所以：

$$
\boxed{
\operatorname{PPL}
=
\frac1{\text{真实 token 概率的几何平均}}
}
$$

上例的概率乘积为：

$$
0.5\times0.25\times0.125=\frac1{64}
$$

几何平均为：

$$
\sqrt[3]{\frac1{64}}=\frac14
$$

因此：

$$
\operatorname{PPL}=\frac{1}{1/4}=4
$$

反过来也成立：

$$
\boxed{
\text{真实 token 概率的几何平均}
=
\frac1{\operatorname{PPL}}
}
$$

例如 PPL 为 $10$ 时，真实 token 概率的几何平均为 $0.1$。

## 4. 如何理解“等效候选数”

假设每个位置都有 $K$ 个候选 token，模型总是给真实 token 概率：

$$
p_t=\frac1K
$$

那么：

$$
\operatorname{PPL}
=
\exp\left(-\ln\frac1K\right)
=
K
$$

这带来常用直觉：

> PPL 可以粗略理解为模型每一步面对的“等效候选数”或“有效分支数”。

例如：

```text
PPL = 2   → 类似每一步在 2 个等可能候选中犹豫
PPL = 10  → 类似每一步在 10 个等可能候选中犹豫
```

但这只是帮助理解的**等效直觉**，不是说模型真的只在固定数量的 token 中均匀选择。真实语言模型的词表概率通常很不均匀，而且每个位置的分布都不同。

### 4.1 取值范围

因为 $0<p_t\le1$：

$$
-\ln p_t\ge0
$$

所以：

$$
\boxed{
\operatorname{PPL}\ge1
}
$$

- 理论最小值是 $1$：每个真实 token 的概率都为 $1$
- PPL 没有固定上界
- 如果某个真实 token 的概率严格为 $0$，理论 PPL 为无穷大

## 5. 语言模型中的完整公式

对于 token 序列：

$$
x_1,x_2,\ldots,x_T
$$

语言模型在第 $t$ 个位置预测：

$$
p_\theta(x_t\mid x_{<t})
$$

也就是根据前文 $x_{<t}$，给实际出现 token $x_t$ 的条件概率。

如果共有 $N$ 个需要评分的有效位置，则：

$$
\boxed{
\operatorname{PPL}
=
\exp\left(
-\frac1N
\sum_{t\in\mathcal V}
\ln p_\theta(x_t\mid x_{<t})
\right)
}
$$

其中 $\mathcal V$ 是有效位置集合，$N=|\mathcal V|$。

处理链路：

```text
前文
  ↓
模型输出词表 logits
  ↓ stable log_softmax
每个候选 token 的 log-probability
  ↓ 按 target token ID 取值
真实 token 的 log-probability
  ↓ 取负、只保留有效位置、求平均
平均 token NLL
  ↓ exp
PPL
```

---

## 6. 必须按有效 token 求平均

LLM 批量评估中常见：

```text
logits.shape  == (batch_size, seq_len, vocab_size)
targets.shape == (batch_size, seq_len)
mask.shape    == (batch_size, seq_len)
```

设 $M_{b,t}\in\{0,1\}$ 表示位置是否参与评分：

$$
\overline{\mathcal L}
=
\frac{
\sum_{b,t}M_{b,t}\mathcal L_{b,t}
}{
\sum_{b,t}M_{b,t}
}
$$

然后：

$$
\operatorname{PPL}=e^{\overline{\mathcal L}}
$$

分母必须是：

```text
有效 token 数 = mask.sum()
```

而不是：

```text
batch_size × seq_len
```

通常需要排除：

- padding token
- 不要求模型预测的位置
- 仅作为上下文、但本轮不应重复计分的位置

如果把 padding 或重复位置算进去，PPL 就不再对应真正想评估的 token。

### 6.1 为什么不能使用总 NLL

两段文本即使每个 token 的预测质量相同，更长的文本也会累积更大的总 NLL。

```text
总 NLL：会随有效 token 数增加
平均 NLL：消除长度带来的线性影响
PPL：由平均 NLL 得到，可评估平均每个 token
```

因此公式中的除数 $N$ 不能省略。

## 7. 从 logits 稳定计算 PPL

不要先把整段序列的概率相乘。长序列的概率乘积可能下溢为 `0.0`。

稳定做法是：

1. 对 logits 执行稳定的 `log_softmax`
2. 取出每个位置真实 token 的 log-probability
3. 对有效位置的 NLL 求平均
4. 最后只执行一次 `exp`

NumPy 核心实现：

```python
def log_softmax(logits, axis=-1):
    maximum = np.max(logits, axis=axis, keepdims=True)
    shifted = logits - maximum
    return shifted - np.log(
        np.exp(shifted).sum(axis=axis, keepdims=True)
    )


log_probabilities = log_softmax(logits, axis=-1)
target_log_probabilities = np.take_along_axis(
    log_probabilities,
    targets[..., np.newaxis],
    axis=-1,
)[..., 0]

mean_nll = -target_log_probabilities[mask].mean()
perplexity = np.exp(mean_nll)
```

完整输入检查和数值实验见：

```bash

```

## 8. 哪些 PPL 可以直接比较

只有评估口径足够一致时，PPL 高低才有意义。至少应保持：

- **相同测试文本或数据分布**
- **相同 tokenizer 和 token 单位**
- **相同上下文长度与切分方式**
- **相同 mask、特殊 token 和计分规则**

### 8.1 不同 tokenizer 的 PPL 通常不能直接比较

同一句话可能被 tokenizer A 切成 5 个 token，被 tokenizer B 切成 8 个 token。PPL 是“每 token”的指标，token 单位不同，平均的分母和每步预测难度也不同。

因此：

> **不同 tokenizer 得到的 token-level PPL 通常不能直接横向比较。**

### 8.2 不同数据集的 PPL 不能代表同一难度

模型在熟悉领域的文本上可能 PPL 很低，在陌生领域可能很高。只有在同一测试集上，才能较公平地比较两个模型。

### 8.3 长文本评估要统一上下文策略

模型只能看到有限长度的上下文。使用滑动窗口评估长文本时，应只给新进入窗口、且尚未评分的 token 计分，避免重复统计。不同窗口长度和步长也会改变可用上下文，必须统一。

## 9. PPL 能说明什么，不能说明什么

PPL 直接衡量：

> 模型给测试文本中真实下一个 token 分配概率的能力。

在同一评估设置下，较低 PPL 通常说明模型更好地拟合了该文本分布。

但 PPL 不能单独保证：

- 生成内容事实正确
- 推理能力更强
- 回答更有帮助
- 内容安全、无偏见
- 生成风格更符合人的偏好

模型甚至可能通过更好地记忆某类文本得到较低 PPL，却不一定在所有实际任务上表现更好。因此 PPL 是重要指标，但不是语言模型能力的完整成绩单。

## 10. 对数底数必须与逆运算匹配

如果平均 NLL 使用自然对数，使用：

$$
\operatorname{PPL}=e^{\overline{\mathcal L}_{\mathrm{nat}}}
$$

如果使用以 $2$ 为底的对数，使用：

$$
\operatorname{PPL}=2^{\overline{\mathcal L}_{\mathrm{bit}}}
$$

只要底数和逆运算匹配，得到的 PPL 相同。

```text
np.log  → np.exp
np.log2 → 2 ** x
```

不能把 `log2` 计算出的平均损失直接传给 `np.exp`。

## 11. 常见误区

| 错误说法 | 正确理解 |
|---|---|
| PPL 越大，模型越好 | 在相同评估条件下，PPL 越低越好 |
| PPL 是正确率 | 它来自真实 token 概率的几何平均，不是命中比例 |
| PPL 为 10，模型每次恰好只考虑 10 个 token | 10 是等效候选数直觉，不是字面候选数量 |
| 用总 NLL 直接取指数 | 必须先除以有效 token 数，得到平均 NLL |
| padding 也参与平均 | 应使用 mask 排除不计分位置 |
| 不同 tokenizer 的 PPL 可以直接比较 | token 单位不同，通常不能直接横比 |
| PPL 低就一定更会推理、更真实 | PPL 只衡量概率预测，不覆盖全部能力 |
| 先乘所有 token 概率最直观 | 长序列会下溢，应在 log 空间求平均 |

## 核心记忆卡

```text
单个 token：
    NLL_t = -ln(p_t)

有效 token 平均：
    mean_NLL = sum(mask * NLL) / sum(mask)

困惑度：
    PPL = exp(mean_NLL)

等价形式：
    PPL = 1 / 真实 token 概率的几何平均
    几何平均概率 = 1 / PPL

方向与范围：
    相同评估条件下，越低越好
    PPL >= 1

稳定计算：
    logits
      → stable log_softmax
      → 取 target log-probability
      → mask 后求平均 NLL
      → exp

比较前提：
    同数据、同 tokenizer、同上下文策略、同计分规则

局限：
    PPL 不是正确率，也不完整代表推理、事实性或安全性
```

掌握标准只保留四项：

1. 能从 token 概率或平均 NLL 手算 PPL
2. 能解释 PPL 是几何平均概率的倒数，以及“等效候选数”只是直觉
3. 能从 logits 使用 mask 稳定计算 PPL
4. 能判断两个 PPL 是否具备直接比较的条件
