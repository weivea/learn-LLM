## 4 · 自回归生成与完整策略链（20 分）

某个三 token 回答为：

```text
A B C
```

模型给出：

$$
P(A\mid\text{prompt})=0.7
$$

$$
P(B\mid\text{prompt},A)=0.6
$$

$$
P(C\mid\text{prompt},A,B)=0.5
$$

1. 计算在该模型下生成完整序列 `A B C` 的概率。
2. 为什么不能把三个概率理解为彼此无条件独立？
3. 如果第一步采样成了 `D`，第二步还能继续使用 $P(B\mid\text{prompt},A)=0.6$ 吗？为什么？
4. 写出从模型 logits 到实际下一个 token 的完整流程，至少包含 temperature、top-k / top-p、归一化和 categorical sampling。
5. Greedy decoding 是否保证得到整段联合概率最大的序列？它与随机采样的主要区别是什么？

✍️ 你的答案：
1. 根据概率的链式法则，生成完整序列 `A B C` 的概率为：

   $$
   P(A,B,C\mid\text{prompt})
   =0.7\times0.6\times0.5
   =0.21
   $$

2. 不能，因为每个 token 的概率都依赖已经生成的前文，它们是条件概率而不是彼此无关的无条件概率。前文发生变化时，下一个 token 的概率分布也会变化。
3. 不能。第一步采样成 `D` 后，第二步应由模型根据新前文重新计算 $P(B\mid\text{prompt},D)$。通常：

   $$
   P(B\mid\text{prompt},A)\ne P(B\mid\text{prompt},D)
   $$

4. 从 logits 到实际下一个 token 的流程为：

   ```text
   模型根据当前全部前文输出 logits
       ↓
   logits / temperature，调节分布的尖锐或平坦程度
       ↓
   通过 Softmax 归一化，得到概率和为 1 的分布
       ↓
   使用 top-k / top-p 过滤候选 token
       ↓
   对保留下来的概率重新归一化，使概率和恢复为 1
       ↓
   从 categorical distribution 中随机采样一个 token
       ↓
   将该 token 追加到前文，继续生成下一个 token
   ```

   工程实现中，也可以先把被 top-k / top-p 过滤掉的 logits 设为 $-\infty$，再统一执行 Softmax，此时被过滤 token 的概率就是 $0$。
5. Greedy decoding 不保证得到整段联合概率最大的序列。它每一步只选择当前条件概率最大的 token，因此只保证当前这一步是局部最优；当前选择可能导致后续 token 的概率很低，从而使整段序列的联合概率并非全局最大。

   Greedy decoding 每一步都选概率最大的 token，通常是确定性的；随机采样则按照 categorical distribution 抽取 token，非最高概率 token 也有机会被选中，因此生成结果更多样，但同一输入的结果可能不同。

```python
from numbers import Integral

import numpy as np


def validate_vector(values, name):
    array = np.asarray(values, dtype=np.float64)
    if array.ndim != 1 or array.size == 0:
        raise ValueError(f"{name} 必须是一维非空数组")
    if not np.all(np.isfinite(array)):
        raise ValueError(f"{name} 不能包含 NaN 或 Inf")
    return array


def validate_probabilities(probabilities):
    values = validate_vector(probabilities, "probabilities")
    if np.any(values < 0.0):
        raise ValueError("概率不能为负数")
    if not np.isclose(values.sum(), 1.0):
        raise ValueError("概率之和必须为 1")
    return values


def temperature_distribution(logits, temperature=1.0):
    scores = validate_vector(logits, "logits")
    if not np.isfinite(temperature) or temperature <= 0.0:
        raise ValueError("temperature 必须是大于 0 的有限数")

    scaled = scores / temperature
    shifted = scaled - np.max(scaled)
    exp_values = np.exp(shifted)
    return exp_values / exp_values.sum()


def top_k_distribution(probabilities, k):
    values = validate_probabilities(probabilities)
    if isinstance(k, bool) or not isinstance(k, Integral):
        raise ValueError("k 必须是整数")
    if not 1 <= k <= values.size:
        raise ValueError("k 必须满足 1 <= k <= 词表大小")

    top_indices = np.argsort(values)[-k:]
    filtered = np.zeros_like(values)
    filtered[top_indices] = values[top_indices]
    return filtered / filtered.sum()


def top_p_distribution(probabilities, top_p):
    values = validate_probabilities(probabilities)
    if not np.isfinite(top_p) or not 0.0 < top_p <= 1.0:
        raise ValueError("top_p 必须满足 0 < top_p <= 1")

    order = np.argsort(values)[::-1]
    sorted_probabilities = values[order]
    cumulative = np.cumsum(sorted_probabilities)
    crossing_index = int(np.searchsorted(cumulative, top_p, side="left"))
    keep_indices = order[:crossing_index + 1]

    filtered = np.zeros_like(values)
    filtered[keep_indices] = values[keep_indices]
    return filtered / filtered.sum()


def sample_index(probabilities, rng):
    values = validate_probabilities(probabilities)
    return int(rng.choice(values.size, p=values))
```

在 temperature Softmax 中先减去最大值，可以避免指数运算发生数值溢出。Top-k 固定保留概率最大的 $k$ 项；top-p 按概率降序排列后，保留累计概率首次达到或超过阈值的最小集合。过滤后都要除以剩余概率之和，重新归一化。

`sample_index` 使用传入的随机数生成器进行 categorical sampling，因此由相同 seed 创建的生成器会产生相同的伪随机序列。

## 第 1 题

累积区间：

| token | 累积区间 |
|---|---|
| `数学` | $[0.00,0.50)$ |
| `代码` | $[0.50,0.80)$ |
| `模型` | $[0.80,0.95)$ |
| `天气` | $[0.95,1.00)$ |

所以：

```text
U = 0.12 → 数学
U = 0.52 → 代码
U = 0.83 → 模型
U = 0.99 → 天气
```

令 $C$ 表示 `数学` 在 $1000$ 次独立采样中的出现次数，则：

$$
C\sim\operatorname{Binomial}(1000,0.5)
$$

期望：

$$
\mathbb E[C]
=
1000(0.5)
=
500
$$

方差：

$$
\operatorname{Var}(C)
=
1000(0.5)(1-0.5)
=
250
$$

标准差：

$$
\operatorname{Std}(C)
=
\sqrt{250}
\approx
15.811
$$

出现 $481$ 次距离期望 $500$ 约为：

$$
\frac{500-481}{15.811}
\approx
1.20
$$

个标准差，是正常的随机波动，不能仅凭这一次实验就判断 $0.50$ 错误。概率描述大量重复实验中的长期频率，不保证有限样本计数恰好等于期望。

## 第 2 题

温度公式：

$$
P_T(i)
=
\frac{\exp(z_i/T)}
{\sum_j\exp(z_j/T)}
$$

当 $T=1$：

$$
\operatorname{softmax}([2,1,0])
\approx
[0.6652,\ 0.2447,\ 0.0900]
$$

当 $T=0.5$，调整后的 logits 为 `[4, 2, 0]`：

$$
\operatorname{softmax}([4,2,0])
\approx
[0.8668,\ 0.1173,\ 0.0159]
$$

当 $T=2$，调整后的 logits 为 `[1, 0.5, 0]`：

$$
\operatorname{softmax}([1,0.5,0])
\approx
[0.5065,\ 0.3072,\ 0.1863]
$$

$T=0.5$ 最尖锐，$T=2$ 最平坦。对 $T>0$，除以同一个正数不改变 logits 的大小顺序，Softmax 也保持顺序，因此概率排名不变。

下面写法错误：

```python
softmax(logits) / temperature
```

它没有按正确方式改变概率的相对比例，而且结果的概率和通常不再为 $1$。正确写法是：

```python
softmax(logits / temperature)
```

$T=0$ 不能直接代入公式，因为除以 $0$ 没有定义。许多生成 API 将 `temperature=0` 特殊处理为 greedy decoding，即选择最大 logit，而不是执行除法。

[0.4706,\ 0.2941,\ 0.2353,\ 0,\ 0]
$$

Top-p $=0.70$ 时：

```text
前 1 项累计：0.40
前 2 项累计：0.65 < 0.70
前 3 项累计：0.85 >= 0.70
```

因此也保留前三项，结果同样为：

$$
[0.4706,\ 0.2941,\ 0.2353,\ 0,\ 0]
$$

必须保留第三项，因为 top-p 要保留“累积概率首次达到或超过阈值”的最小集合；只保留前两项会让保留的原始概率质量只有 $0.65$，尚未达到 $0.70$。

Top-p $=0.60$ 时，前两项累计概率为 $0.65$，所以保留前两项：

$$
\left[
\frac{0.40}{0.65},
\frac{0.25}{0.65},
0,
0,
0
\right]
\approx
[0.6154,\ 0.3846,\ 0,\ 0,\ 0]
$$

Top-k 固定保留 $k$ 个候选；top-p 保留累计概率达到阈值的最小集合，因此候选数量会随当前分布动态变化。

## 第 4 题

使用概率链式法则：

$$
\begin{aligned}
P(A,B,C\mid\text{prompt})
&=
P(A\mid\text{prompt})\\
&\quad\cdot
P(B\mid\text{prompt},A)\\
&\quad\cdot
P(C\mid\text{prompt},A,B)\\
&=
0.7\times0.6\times0.5\\
&=
0.21
\end{aligned}
$$

三个概率都是依赖已有前文的条件概率，不是三个互不相关事件的无条件概率。

若第一步采样成 `D`，第二步必须重新计算：

$$
P(\cdot\mid\text{prompt},D)
$$

不能继续使用基于前文 `prompt, A` 的旧概率。

完整流程：

```text
模型根据当前全部前文输出 logits
    ↓
logits / temperature
    ↓
Softmax 得到概率
    ↓
按 top-k / top-p 过滤候选
    ↓
将剩余概率重新归一化为和 1
    ↓
从 categorical distribution 随机采样一个 token
    ↓
追加到前文并进入下一步
```

工程实现也可以先在 logits 上把过滤项设为 $-\infty$，再统一 Softmax。

Greedy 每一步只选择当前概率最大的 token，只保证局部选择最大，不保证整段序列的联合概率全局最大。Greedy 是确定性选择；随机采样则按概率抽取，允许非最大概率 token 出现，从而产生更多样的路径。

## 第 5 题

```python
from numbers import Integral

import numpy as np


def validate_vector(values, name):
    array = np.asarray(values, dtype=np.float64)
    if array.ndim != 1 or array.size == 0:
        raise ValueError(f"{name} 必须是一维非空数组")
    if not np.all(np.isfinite(array)):
        raise ValueError(f"{name} 不能包含 NaN 或 Inf")
    return array


def validate_probabilities(probabilities):
    values = validate_vector(
        probabilities,
        "probabilities",
    )
    if np.any(values < 0.0):
        raise ValueError("概率不能为负数")
    if not np.isclose(values.sum(), 1.0):
        raise ValueError("概率之和必须为 1")
    return values


def temperature_distribution(logits, temperature=1.0):
    scores = validate_vector(logits, "logits")
    if (
        not np.isfinite(temperature)
        or temperature <= 0.0
    ):
        raise ValueError(
            "temperature 必须是大于 0 的有限数"
        )

    scaled = scores / temperature
    shifted = scaled - np.max(scaled)
    exp_values = np.exp(shifted)
    return exp_values / exp_values.sum()


def top_k_distribution(probabilities, k):
    values = validate_probabilities(probabilities)
    if isinstance(k, bool) or not isinstance(k, Integral):
        raise ValueError("k 必须是整数")
    if not 1 <= k <= values.size:
        raise ValueError(
            "k 必须满足 1 <= k <= 词表大小"
        )

    top_indices = np.argsort(values)[-k:]
    filtered = np.zeros_like(values)
    filtered[top_indices] = values[top_indices]
    return filtered / filtered.sum()


def top_p_distribution(probabilities, top_p):
    values = validate_probabilities(probabilities)
    if (
        not np.isfinite(top_p)
        or not 0.0 < top_p <= 1.0
    ):
        raise ValueError(
            "top_p 必须满足 0 < top_p <= 1"
        )

    order = np.argsort(values)[::-1]
    sorted_probabilities = values[order]
    cumulative = np.cumsum(sorted_probabilities)
    crossing_index = int(
        np.searchsorted(
            cumulative,
            top_p,
            side="left",
        )
    )
    keep_count = min(
        crossing_index + 1,
        values.size,
    )
    keep_indices = order[:keep_count]

    filtered = np.zeros_like(values)
    filtered[keep_indices] = values[keep_indices]
    return filtered / filtered.sum()


def sample_index(probabilities, rng):
    values = validate_probabilities(probabilities)
    return int(rng.choice(values.size, p=values))
```

相同 seed 创建的两个生成器会产生相同伪随机序列，因此题目中的两个采样列表相等。固定 seed 用于可复现调试，不会把随机采样变成 greedy。
