## 1 · 交叉熵与 NLL（20 分）

正确类别是第 `1` 项：

$$
\boldsymbol q=[0,1,0]
$$

模型预测：

$$
\boldsymbol p=[0.1,0.7,0.2]
$$

1. 写出交叉熵公式，并手算该样本的损失。使用 $\ln0.7\approx-0.357$。
2. 为什么这个 one-hot 分类问题中的交叉熵等于 NLL？
3. 当正确类别概率 $p_y$ 增大时，损失如何变化？损失的最小值是多少？

✍️ 你的答案：
1. 公式为：
$$
H(\boldsymbol q,\boldsymbol p)
=
-\sum_{i=1}^{K}q_i\ln p_i
$$

损失
$$
\mathcal L
=
-\sum_i q_i\ln p_i
=
-0*\ln 0.1-1*\ln 0.7-0*\ln 0.2
=
-1*\ln 0.7
=
0.357
$$

2. 因为one-hot 分类里只有一个$q_i=1$,其它分类的目标概率都是0，
$$
H(\boldsymbol q,\boldsymbol p)
=
-\sum_{i=1}^{K}q_i\ln p_i
=-q_i\ln p_i
=
-\ln p_i=NLL
$$
3. 当正确类别概率 $p_y$ 增大时，损失渐小, 损失的最小值是0

> ✅ **批改第 1 题（19 / 20）**：公式、计算和变化趋势正确。第 2 问最后一步的下标应写得更准确：
>
> $$
> -\sum_iq_i\ln p_i=-q_y\ln p_y=-\ln p_y
> $$
>
> 这里的 $y$ 明确表示唯一正确类别；原答案继续使用 $i$，容易误解为任意类别。

## 2 · 从 logits 稳定计算（20 分）

给定：

$$
\boldsymbol z=[2,1,0]
$$

正确类别索引为 `0`，并已知：

$$
\operatorname{softmax}(\boldsymbol z)_0\approx0.6652
$$

1. 计算该样本的 NLL。使用 $\ln0.6652\approx-0.4076$。
2. 令 $m=\max_jz_j$，写出从 logits 计算损失的稳定公式。
3. 为什么训练框架通常让交叉熵函数直接接收 logits，而不是让用户先做 Softmax？

✍️ 你的答案：
1. $NLL= -ln(\operatorname{softmax}(\boldsymbol z)_0)\approx0.4076$
2. 公式为
$$
\mathcal L
=
-z_y+m
+\ln\sum_j e^{z_j-m}
$$
3. 因为训练框架里的交叉熵函数会对 softmax和 nll做融合计算，避免重复计算softmax

> ⚠️ **批改第 2 题（15 / 20）**：
>
> 1. **NLL 的符号写反了**。因为 $\ln0.6652\approx-0.4076$，所以：
>
>    $$
>    \mathcal L=-\ln0.6652=-(-0.4076)=\boxed{0.4076}
>    $$
>
>    NLL 应为非负数，不是 `-0.4076`。
> 2. 稳定公式正确。
> 3. 融合计算不仅是避免重复 Softmax，更重要的是直接使用稳定的 `log_softmax + NLL`，避免指数溢出、小概率下溢和 `log(0)`。如果用户先做 Softmax，框架还会把概率误当作 logits 再做一次 `log_softmax`，结果也会改变。

## 4 · 梯度方向（15 分）

模型概率和 one-hot 目标为：

$$
\boldsymbol p=[0.1,0.7,0.2]
$$

$$
\boldsymbol q=[0,1,0]
$$

利用：

$$
\frac{\partial\mathcal L}{\partial\boldsymbol z}
=
\boldsymbol p-\boldsymbol q
$$

1. 算出三个 logits 的梯度。
2. 梯度下降后，正确类别和错误类别的 logits 分别怎样变化？这为什么能让模型学得更准？

✍️ 你的答案：
1. 梯度
$$
\frac{\partial\mathcal L}{\partial\boldsymbol z}
=
[0.1,-0.3,0.2]
$$
2. 正确类别的logits变大，错误类别的logits变小。这让他们更靠近目标概率分布

> ⚠️ **批改第 4 题（13 / 15）**：梯度和更新方向正确，但“让 logits 更靠近概率分布”不准确；logits 本身不是概率分布。准确解释是：
>
> - 正确类别梯度为负，梯度下降会提高它的 logit
> - 错误类别梯度为正，梯度下降会降低它们的 logits
> - 再经过 Softmax，正确类别概率升高、错误类别概率降低，因此预测更接近 one-hot 目标

```python
import numpy as np

def log_softmax(logits: np.ndarray, axis: int = -1) -> np.ndarray:
    values = np.asarray(logits, dtype=np.float64)

    if values.ndim == 0:
        raise ValueError("log_softmax 至少需要一维输入")
    if values.size == 0:
        raise ValueError("log_softmax 不能处理空输入")
    if not np.all(np.isfinite(values)):
        raise ValueError("logits 不能包含 NaN 或 Inf")

    maximum = np.max(values, axis=axis, keepdims=True)
    shifted = values - maximum
    log_denominator = np.log(
        np.exp(shifted).sum(axis=axis, keepdims=True)
    )
    return shifted - log_denominator

def cross_entropy_from_logits(logits, targets, reduction="mean"):
    values = np.asarray(logits, dtype=np.float64)
    labels = np.asarray(targets)

    if values.ndim == 0:
        raise ValueError("logits 至少需要一维输入")
    if values.size == 0 or values.shape[-1] == 0:
        raise ValueError("logits 不能包含空的词表维")
    if not np.all(np.isfinite(values)):
        raise ValueError("logits 不能包含 NaN 或 Inf")
    if not np.issubdtype(labels.dtype, np.integer):
        raise TypeError("targets 必须是整数类别索引")
    if labels.shape != values.shape[:-1]:
        raise ValueError(
            "targets.shape 必须等于 logits.shape[:-1]"
        )
    if labels.size == 0:
        raise ValueError("targets 不能为空")
    if labels.min() < 0 or labels.max() >= values.shape[-1]:
        raise ValueError("targets 包含超出词表范围的类别索引")
    log_probabilities = log_softmax(values, axis=-1)
    selected = np.take_along_axis(
        log_probabilities,
        labels[..., np.newaxis],
        axis=-1,
    )[..., 0]
    losses = -selected

    if reduction == "none":
        return losses
    if reduction == "sum":
        return losses.sum()
    if reduction == "mean":
        return losses.mean()
    raise ValueError("reduction 必须是 none、sum 或 mean")
```
为什么不能把 `-np.log(np.max(probabilities, axis=-1))` 当作正确实现？\
回答： 应该取真实标签对应概率 $p_y$ ，即使它不是最大概率
