| 问题 | PMF | PDF |
|---|---|---|
| 用于哪类随机变量 | | |
| 函数值能否直接表示单点概率 | | |

| 函数值能否大于 $1$ | | |

✍️ 你的答案：

|---|---|---|

>

> - “PDF 高度 × 区间宽度”只在区间内密度恒定时直接成立；一般情况必须计算 $\int_a^b f_X(x)\,dx$。

### C3 · 采样与经验频率

解释下面代码中每个关键参数的作用：

```python
rng = np.random.default_rng(2026)
samples = rng.choice(
    len(probabilities),
    size=20_000,
    p=probabilities,
)
frequencies = np.bincount(
    samples,
    minlength=len(probabilities),
) / len(samples)
```

为什么 `frequencies` 通常接近、但不完全等于 `probabilities`？

✍️ 你的答案：

- `np.random.default_rng(2026)`：创建一个随机数生成器；`2026` 是随机种子，使每次运行得到相同的采样结果，便于复现实验。
- `len(probabilities)`：表示候选类别的数量。这里是 `4`，因此采样结果是 `0` 到 `3` 的整数下标。
- `size=20_000`：表示从概率分布中采样 `20_000` 次。
- `p=probabilities`：指定每个下标被采到的理论概率。
- `samples`：保存 `20_000` 次采样得到的类别下标。
- `np.bincount(samples, ...)`：统计每个类别下标出现的次数。
- `minlength=len(probabilities)`：保证计数结果包含所有类别。即使某个类别一次都没有出现，也会为它保留一个值为 `0` 的位置。
- `/ len(samples)`：用每个类别的出现次数除以总采样次数，得到经验频率。

`frequencies` 来自有限次随机采样，因此会有随机波动，通常不会与理论概率 `probabilities` 完全相等。根据大数定律，采样次数越多，经验频率通常越接近理论概率。固定随机种子只会让这次随机结果可以重复，并不会消除随机性。

### C6 · 连续区间的数值积分

解释为什么下面代码计算的是区间概率，而不是单点概率：

```python
x = np.linspace(-1.0, 1.0, 10_001)
density = np.exp(-(x**2) / 2.0) / np.sqrt(2.0 * np.pi)
dx = np.diff(x)
probability = np.sum(
    (density[:-1] + density[1:]) * dx / 2.0
)
```

✍️ 你的答案：

- `np.linspace(-1.0, 1.0, 10_001)`：在区间 $[-1,1]$ 上取 `10_001` 个点，把整个区间分成 `10_000` 个很小的子区间。
- `density`：计算标准正态分布的 PDF 在这些点上的概率密度。密度值本身不是概率。
- `np.diff(x)`：计算每个相邻点之间的距离，即每个小区间的宽度 `dx`。
- `(density[:-1] + density[1:]) / 2.0`：计算每个小区间左右端点密度的平均值。
- 平均密度乘以 `dx`：得到一个小梯形的面积，用它近似该小区间的概率。
- `np.sum(...)`：把 `10_000` 个小区间的面积相加，近似计算：

$$
P(-1\le X\le1)
=
\int_{-1}^{1}f(x)\,dx
$$

因此，这段代码计算的是 PDF 在整个 $[-1,1]$ 区间下的面积，也就是区间概率，而不是某个点的概率。对于连续随机变量，任意单点的概率都是 $0$；`density[i]` 表示该点的概率密度，也不能直接当作单点概率。
