## 一句话总结

**对数回答“底数的多少次方等于这个数”，并把乘法变成加法。**

$$
\boxed{
\log_b x=y
\quad\Longleftrightarrow\quad
b^y=x
}
$$

在深度学习中通常使用自然对数 $\ln$：

$$
\boxed{
\ln x=\log_e x
}
$$

对语言模型最重要的作用是：

```text
很多 token 概率相乘
        ↓ 取 log
很多 log-probability 相加
        ↓ 加负号
序列总 NLL
```

也就是：

$$
\boxed{
-\ln\left(\prod_{t=1}^{T}p_t\right)
=
-\sum_{t=1}^{T}\ln p_t
}
$$

## 1. 对数是指数的逆运算

### 1.1 从指数反过来看

已知：

$$
2^3=8
$$

如果问题变成：

> 2 的多少次方等于 8？

答案仍然是 $3$，用对数写成：

$$
\log_2 8=3
$$

两种写法描述的是同一件事：

| 指数写法 | 对数写法 |
|---|---|
| $2^3=8$ | $\log_2 8=3$ |
| $10^{-2}=0.01$ | $\log_{10}0.01=-2$ |
| $e^0=1$ | $\ln1=0$ |
| $e^1=e$ | $\ln e=1$ |

指数与对数互为逆运算：

$$
\boxed{
\ln(e^x)=x
}
$$

$$
\boxed{
e^{\ln x}=x,\qquad x>0
}
$$

---

## 2. 底数、真数与定义域

在 $\log_bx=y$ 中，$b$ 是底数，$x$ 是真数。实数范围内必须满足：

$$
\boxed{
b>0,\qquad b\ne1,\qquad x>0
}
$$

因为 $e^y$ 对任何实数 $y$ 都大于 $0$，所以 $\ln0$ 没有有限值，并且：

$$
\ln x\to-\infty
$$

负数也不可能写成 $e^y$，所以 $\ln(-1)$ 在实数范围内没有定义。NumPy 中：

```python
np.log(0.0)   # -inf，并产生警告
np.log(-1.0)  # nan，并产生警告
```

`-inf` 和 `nan` 不是可以继续当作普通损失使用的正常数值。

常见的三种底数：

| 写法 | 底数 | 常见场景 |
|---|---:|---|
| $\ln x$ | $e\approx2.718$ | 深度学习、微积分、概率统计 |
| $\log_2x$ | $2$ | 信息论，单位常写作 bit |
| $\log_{10}x$ | $10$ | 数量级、科学计数法 |

在 NumPy 中：

```python
np.log(x)    # 自然对数 ln(x)
np.log2(x)   # log_2(x)
np.log10(x)  # log_10(x)
```

> 深度学习公式或代码只写 `log` 时，通常默认指自然对数。

## 3. 自然对数的形状与概率区间

对机器学习最常用的底数 $e>1$，$\ln x$ 是单调递增函数：

```text
x 增大 → ln(x) 增大
```

几个关键位置：

| $x$ | $\ln x$（约） |
|---:|---:|
| $0.01$ | $-4.605$ |
| $0.1$ | $-2.303$ |
| $0.5$ | $-0.693$ |
| $1$ | $0$ |
| $e$ | $1$ |
| $10$ | $2.303$ |

概率满足：

$$
0<p\le1
$$

因此概率的自然对数满足：

$$
\boxed{
\ln p\le0
}
$$

再取负：

$$
\boxed{
-\ln p\ge0
}
$$

这正是上一节 NLL 非负的原因。

特别注意负号：

$$
\ln0.1\approx-2.303
$$

所以：

$$
-\ln0.1=-(-2.303)=2.303
$$

不是 $-2.303$。

## 4. 三条核心运算律

对 $a>0$、$c>0$：

$$
\boxed{
\begin{aligned}
\ln(ac)&=\ln a+\ln c\\
\ln\left(\frac ac\right)&=\ln a-\ln c\\
\ln(a^r)&=r\ln a
\end{aligned}
}
$$

最重要的是第一条：**乘积经过 log 后变成加法**。

但加法不能拆：

$$
\boxed{
\ln(a+c)\ne\ln a+\ln c
}
$$

例如 $\ln(2+3)=\ln5$，而 $\ln2+\ln3=\ln6$。

## 5. 为什么语言模型喜欢 log

### 5.1 一个 token：把高概率变成低损失

单个正确 token 的 NLL：

$$
\mathcal L_t=-\ln p_t
$$

| 正确 token 概率 $p_t$ | $\ln p_t$（约） | NLL：$-\ln p_t$（约） |
|---:|---:|---:|
| $1.00$ | $0.000$ | $0.000$ |
| $0.50$ | $-0.693$ | $0.693$ |
| $0.10$ | $-2.303$ | $2.303$ |
| $0.01$ | $-4.605$ | $4.605$ |

于是：

```text
正确 token 概率高 → log-probability 接近 0 → NLL 小
正确 token 概率低 → log-probability 很负   → NLL 大
```

### 5.2 多个 token：乘积变求和

假设三个位置正确 token 的概率是：

$$
p_1=0.8,\qquad p_2=0.5,\qquad p_3=0.25
$$

整段序列的概率乘积为：

$$
P=0.8\times0.5\times0.25=0.1
$$

取对数：

$$
\begin{aligned}
\ln P
&=
\ln(0.8\times0.5\times0.25)\\
&=
\ln0.8+\ln0.5+\ln0.25\\
&\approx-2.303
\end{aligned}
$$

序列总 NLL：

$$
\boxed{
\mathcal L_{\mathrm{total}}
=
-\ln P
=
-\sum_{t=1}^{T}\ln p_t
}
$$

本例：

$$
\mathcal L_{\mathrm{total}}\approx2.303
$$

平均 token NLL：

$$
\boxed{
\overline{\mathcal L}
=
-\frac1T\sum_{t=1}^{T}\ln p_t
}
$$

本例：

$$
\overline{\mathcal L}
\approx
\frac{2.303}{3}
\approx0.768
$$

这解释了为什么 LLM 可以先计算每个有效 token 的 NLL，再求和或求平均。

## 6. 数值稳定性：不要先乘再取 log

### 6.1 长序列的概率乘积会下溢

如果连续 $200$ 个位置的正确 token 概率都是 $0.01$：

$$
P=0.01^{200}=10^{-400}
$$

这是一个极小的正数。`float64` 无法表示这么小的普通结果，直接计算可能得到：

```python
probability = np.prod(np.full(200, 0.01))
# probability == 0.0
```

随后：

```python
np.log(probability)
# -inf
```

数学上的真实概率并不是 $0$，只是浮点数发生了**下溢**。

### 6.2 在 log 空间求和仍然有限

不要先算乘积，而是直接求 log-probability 之和：

$$
\begin{aligned}
\ln P
&=
\sum_{t=1}^{200}\ln0.01\\
&=
200\times(-4.605\ldots)\\
&\approx-921.034
\end{aligned}
$$

代码：

```python
log_probability = np.log(
    np.full(200, 0.01)
).sum()
```

得到的是有限数值，序列总 NLL 为：

$$
-\ln P\approx921.034
$$

核心原则：

```text
错误路径：很多小概率相乘 → 下溢成 0 → log(0) = -inf
稳定路径：每个概率先取 log → 有限负数求和
```

## 7. `logits` 不等于 `log-probabilities`

LLM 中常见三个不同对象：

| 名称 | 含义 | 数值约束 |
|---|---|---|
| logits | 模型输出的原始分数 | 可正、可负，不要求和为 1 |
| probabilities | Softmax 后的概率 | $0<p_i<1$，且 $\sum_i p_i=1$ |
| log-probabilities | 概率的自然对数 $\ln p_i$ | 每项 $\le0$，且 $\sum_i e^{\ln p_i}=1$ |

处理链路：

```text
logits
  ↓ softmax
probabilities
  ↓ log
log-probabilities
```

但实现时通常不会真的先算 Softmax 再取 log，而是直接使用稳定的 `log_softmax`：

$$
\operatorname{log\_softmax}(\boldsymbol z)_i
=
z_i-\ln\sum_j e^{z_j}
$$

这里需要计算：

$$
\operatorname{logsumexp}(\boldsymbol z)
=
\ln\sum_j e^{z_j}
$$

稳定形式是先减最大 logit：

$$
\boxed{
\operatorname{logsumexp}(\boldsymbol z)
=
m+\ln\sum_j e^{z_j-m},
\qquad
m=\max_jz_j
}
$$

因此：

$$
\boxed{
\operatorname{log\_softmax}(\boldsymbol z)_i
=
z_i-
\left(
m+\ln\sum_j e^{z_j-m}
\right)
}
$$

> 不要直接计算 `np.log(logits)`。logits 不是概率，而且可以为负数。

## 8. 对数关注比例变化

正确答案概率提高 $10$ 倍时：

$$
\begin{aligned}
-\ln(10p)
&=
-\ln10-\ln p\\
&=
-\ln p-\ln10
\end{aligned}
$$

所以只要概率合法，提高 $10$ 倍都会让 NLL 减少：

$$
\ln10\approx2.303
$$

```text
0.01 → 0.10：NLL 从 4.605 降到 2.303，减少 2.303
0.10 → 1.00：NLL 从 2.303 降到 0.000，减少 2.303
```

对数关注的是**倍数或比例变化**，而不只是绝对差值。

## 9. 换底与困惑度预告

任意合法底数可以换成自然对数：

$$
\boxed{
\log_bx=\frac{\ln x}{\ln b}
}
$$

自然对数对应单位 **nat**，以 2 为底对应 **bit**。换底只改变数值尺度，不改变模型优劣排序。

下一节会学习困惑度。平均 token NLL 使用自然对数时：

$$
\boxed{
\operatorname{PPL}
=
e^{\overline{\mathcal L}}
}
$$

这是用逆运算 $e^x$ 把平均 log 损失变回普通尺度；具体含义留到 3.5。

## 10. NumPy 核心实现

稳定的 `logsumexp`：

```python
def logsumexp(values, axis=-1, keepdims=False):
    maximum = np.max(values, axis=axis, keepdims=True)
    shifted = values - maximum
    result = maximum + np.log(
        np.exp(shifted).sum(axis=axis, keepdims=True)
    )
    if keepdims:
        return result
    return np.squeeze(result, axis=axis)
```

稳定的 `log_softmax`：

```python
def log_softmax(logits, axis=-1):
    return logits - logsumexp(
        logits,
        axis=axis,
        keepdims=True,
    )
```

关键性质：

```python
log_probabilities = log_softmax(logits, axis=-1)
probabilities = np.exp(log_probabilities)

assert np.allclose(
    probabilities.sum(axis=-1),
    1.0,
)
```

LLM 常见形状：

```text
logits.shape            == (batch, seq_len, vocab_size)
log_probabilities.shape == (batch, seq_len, vocab_size)
```

沿最后的 `vocab_size` 维执行 `log_softmax`，shape 不变。

## 11. 常见误区

| 错误说法 | 正确理解 |
|---|---|
| $\ln0=0$ | $\ln1=0$；$\ln0$ 没有有限值 |
| 概率取 log 后仍为正 | $0<p<1$ 时 $\ln p<0$ |
| $\ln(a+c)=\ln a+\ln c$ | 只有乘积能拆成 log 之和 |
| 先乘完序列概率再取 log | 应直接对 log-probabilities 求和，避免下溢 |
| `np.log` 以 10 为底 | `np.log` 是自然对数 |
| logits 就是 log-probabilities | `log_softmax(logits)` 才是 log-probabilities |
| 可以直接 `np.log(logits)` | logits 可为负，应使用整组 `log_softmax` |

## 核心记忆卡

```text
定义：
    log_b(x) = y  ⇔  b^y = x

自然对数：
    ln(x) = log_e(x)
    np.log(x) 就是 ln(x)

定义域：
    x > 0

逆运算：
    ln(e^x) = x
    e^(ln x) = x

运算律：
    ln(ab)  = ln(a) + ln(b)
    ln(a/b) = ln(a) - ln(b)
    ln(a^r) = r ln(a)
    ln(a+b) 不能拆

概率：
    0 < p <= 1
    ln(p) <= 0
    -ln(p) >= 0

语言模型：
    sequence log-prob = Σ_t ln(p_t)
    total NLL         = -Σ_t ln(p_t)
    mean NLL          = total NLL / 有效 token 数

稳定计算：
    不要先乘很多小概率
    直接对 log-probabilities 求和

对象区别：
    logits → log_softmax → log-probabilities

下一节：
    perplexity = exp(mean NLL)
```

掌握标准只保留四项：

1. 能在指数形式与对数形式之间互换，并说清定义域
2. 能正确使用三条运算律，知道 $\ln(a+b)$ 不能拆
3. 能解释 $-\ln p$ 的符号、大小和概率变化之间的关系
4. 能说明序列为什么在 log 空间求和，并写出稳定 `logsumexp`
