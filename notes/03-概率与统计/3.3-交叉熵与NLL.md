> 重要度：⭐⭐⭐（语言模型最核心的训练损失）

> 前置：3.1 概率分布 · 3.2 Softmax

## 一句话总结

**对于只有一个正确 token 的任务，交叉熵和 NLL 都是在计算正确 token 概率的负对数：**

$$
\boxed{
\mathcal L=-\ln p_y
}
$$

其中 $p_y$ 是模型分给正确 token $y$ 的概率。

```text
正确 token 概率越大 → 损失越小
正确 token 概率越小 → 损失越大
```

LLM 训练的主线可以先记成：

```text
前文
  ↓
模型输出词表 logits
  ↓
Softmax 得到词表概率
  ↓
取出正确的下一个 token 的概率 p_y
  ↓
-ln(p_y) 得到这个位置的损失
  ↓
对有效 token 的损失求平均
```

---

## 1. 为什么需要损失函数

假设正确的下一个 token 是“模型”，两个模型给出的概率分别是：

| 模型 | $P(\text{模型})$ | 是否预测正确 |
|---|---:|---|
| A | $0.51$ | 是 |
| B | $0.99$ | 是 |

如果只看 `argmax` 或准确率，两者都预测正确，得分没有区别。

但模型 B 对正确答案更有把握，显然应该得到更小的损失：

$$
-\ln(0.51)\approx0.673
$$

$$
-\ln(0.99)\approx0.010
$$

所以交叉熵不只判断“答案对不对”，还会衡量：

> **模型给正确答案分配了多少概率。**

这比只返回 `0` 或 `1` 的准确率提供了更细腻的训练信号。

---

## 2. 本节够用的对数知识

对数会在 3.4 单独学习。本节先记住自然对数 $\ln$ 的四个性质：

1. $\ln(1)=0$
2. 当 $0<p<1$ 时，$\ln(p)<0$
3. $p$ 越接近 $0$，$-\ln(p)$ 越大
4. $\ln(ab)=\ln(a)+\ln(b)$

常见概率对应的负对数：

| 正确答案概率 $p_y$ | NLL：$-\ln p_y$（约） | 含义 |
|---:|---:|---|
| $1.00$ | $0.000$ | 完全确信且正确 |
| $0.80$ | $0.223$ | 比较确信 |
| $0.50$ | $0.693$ | 不太确定 |
| $0.10$ | $2.303$ | 给正确答案的概率很低 |
| $0.01$ | $4.605$ | 非常自信地忽略了正确答案 |

因此：

$$
\boxed{
0<p_y\le1
\quad\Longrightarrow\quad
-\ln p_y\ge0
}
$$

损失不是概率，所以它可以大于 $1$，并且理论上没有上界。

深度学习中默认使用自然对数，损失单位也可称为 **nat**。现阶段不需要纠结单位，只要统一使用 `np.log` 即可。

## 3. 负对数似然（NLL）

### 3.1 什么是似然

已经观察到正确答案是类别 $y$ 时，模型给这个答案的概率为：

$$
p_y=P_\theta(y\mid x)
$$

这里：

- $x$ 是输入，例如当前前文
- $y$ 是实际出现的下一个 token
- $\theta$ 表示模型参数
- $p_y$ 是模型认为这个观测结果有多可能

从当前模型的角度看，$p_y$ 就是这个观测结果的**似然**。

训练希望正确答案的似然尽可能大：

$$
\max_\theta p_y
$$

这等价于最小化它的负对数：

$$
\boxed{
\mathcal L_{\mathrm{NLL}}
=
-\ln p_y
}
$$

### 3.2 为什么加上负号

概率越大，对数越大：

```text
p_y 增大
    ↓
ln(p_y) 增大
    ↓
-ln(p_y) 减小
```

因此，“最大化正确答案的概率”就变成了机器学习中更常见的“最小化损失”。

### 3.3 多个 token 为什么要用对数

假设三个位置的正确 token 概率分别是：

$$
0.8,\qquad0.5,\qquad0.25
$$

整段正确序列的联合似然可先直观理解为这些概率的乘积：

$$
0.8\times0.5\times0.25=0.1
$$

取负对数后，乘积变成加法：

$$
-\ln(0.1)
=
-\ln(0.8)-\ln(0.5)-\ln(0.25)
\approx2.303
$$

平均到三个 token：

$$
\frac{2.303}{3}\approx0.768
$$

这带来两个好处：

- 很多小概率连乘容易下溢，对数求和更稳定
- 每个 token 的损失可以直接相加或求平均

## 4. 交叉熵（Cross-Entropy）

### 4.1 一般公式

设真实目标分布为：

$$
\boldsymbol q=[q_1,q_2,\ldots,q_K]
$$

模型预测分布为：

$$
\boldsymbol p=[p_1,p_2,\ldots,p_K]
$$

交叉熵定义为：

$$
\boxed{
H(\boldsymbol q,\boldsymbol p)
=
-\sum_{i=1}^{K}q_i\ln p_i
}
$$

它衡量模型分布 $\boldsymbol p$ 对目标分布 $\boldsymbol q$ 的匹配程度。

### 4.2 one-hot 标签下，交叉熵就是 NLL

假设三个候选 token 是：

```text
[模型, 学习, 天气]
```

正确答案是“学习”，目标分布可写成 one-hot：

$$
\boldsymbol q=[0,1,0]
$$

模型预测：

$$
\boldsymbol p=[0.1,0.7,0.2]
$$

交叉熵：

$$
\begin{aligned}
H(\boldsymbol q,\boldsymbol p)
&=
-\left(
0\ln0.1+1\ln0.7+0\ln0.2
\right)\\
&=
-\ln0.7\\
&\approx0.357
\end{aligned}
$$

只有正确类别对应的项保留下来：

$$
\boxed{
H(\boldsymbol q,\boldsymbol p)
=
-\ln p_y
=
\mathcal L_{\mathrm{NLL}}
}
$$

因此，在 LLM 的普通 next-token 训练中，经常把 **cross-entropy loss** 和 **NLL loss** 当作同一个数值目标来讨论。

### 4.3 两个名称并非在所有场景都完全同义

- **NLL** 是更一般的统计学概念：对模型赋予观测数据的似然取负对数
- **交叉熵** 比较目标分布 $\boldsymbol q$ 与预测分布 $\boldsymbol p$
- 当目标是 one-hot 的分类标签时，交叉熵恰好化简为正确类别的 NLL
- 如果使用软标签，多个 $q_i$ 都可能非零，交叉熵就不再只是挑出一个正确类别

本阶段最需要掌握的是：

> **LLM 的普通 token 标签是类别索引；在这种场景下，交叉熵 = 正确 token 的 NLL。**

## 5. 从 logits 直接计算交叉熵

上一节学过：

$$
p_y
=
\frac{e^{z_y}}{\sum_j e^{z_j}}
$$

代入 NLL：

$$
\begin{aligned}
\mathcal L
&=
-\ln p_y\\
&=
-\ln
\left(
\frac{e^{z_y}}{\sum_j e^{z_j}}
\right)\\
&=
-z_y+\ln\sum_j e^{z_j}
\end{aligned}
$$

所以：

$$
\boxed{
\mathcal L
=
-z_y+\operatorname{logsumexp}(\boldsymbol z)
}
$$

### 5.1 数值稳定形式

直接计算 $e^{1000}$ 会溢出。和稳定 Softmax 一样，先减去最大 logit：

$$
m=\max_j z_j
$$

因为对每个 $z_j$ 都有 $z_j=(z_j-m)+m$，所以：

$$
\begin{aligned}
\mathcal L
&=
-z_y+\ln\sum_j e^{z_j}\\
&=
-z_y+\ln\sum_j e^{(z_j-m)+m}\\
&=
-z_y+\ln\left(
e^m\sum_j e^{z_j-m}
\right)\\
&=
-z_y+\ln e^m
+\ln\sum_j e^{z_j-m}\\
&=
-z_y+m
+\ln\sum_j e^{z_j-m}
\end{aligned}
$$

因此：

$$
\boxed{
\mathcal L
=
-z_y+m
+\ln\sum_j e^{z_j-m}
}
$$

因为所有 logits 减去同一个常数不会改变 Softmax，这个变形不会改变数学结果。

### 5.2 一个 logits 例子

给定：

$$
\boldsymbol z=[2,1,0]
$$

Softmax 概率约为：

$$
\boldsymbol p=[0.6652,0.2447,0.0900]
$$

如果正确类别是第 `0` 项：

$$
\mathcal L=-\ln0.6652\approx0.4076
$$

如果正确类别是第 `2` 项：

$$
\mathcal L=-\ln0.0900\approx2.4076
$$

同一组预测中，模型给正确类别的概率越低，损失越大。

### 5.3 为什么框架通常接收 logits

PyTorch 的 `CrossEntropyLoss` 等函数通常接收：

```text
原始 logits + 类别索引 targets
```

而不是：

```text
已经 softmax 的 probabilities + targets
```

框架会把 `log_softmax` 和 NLL 融合起来稳定计算。手动先做 Softmax 再传入一个期待 logits 的函数，会造成**重复 Softmax**，既改变结果又降低稳定性。

可以用伪代码理解框架内部的过程：

```text
输入：logits、正确类别 target

log_probabilities = stable_log_softmax(logits)
loss = -log_probabilities[target]

返回 loss
```

因此，正确调用方式是：

```text
loss = CrossEntropyLoss(logits, target)
```

不要写成：

```text
probabilities = softmax(logits)
loss = CrossEntropyLoss(probabilities, target)
```

后一种写法中，`CrossEntropyLoss` 会把 `probabilities` 误当作 logits，并在内部再次执行 `log_softmax`。实际计算路径变成：

```text
logits
    → softmax
    → probabilities（被误当作新的 logits）
    → log_softmax
    → NLL
```

## 6. LLM 中如何对 token 损失求平均

LLM 输出 logits 的常见形状：

```text
(batch_size, seq_len, vocab_size)
```

目标 token ID 的形状：

```text
(batch_size, seq_len)
```

沿最后的 `vocab_size` 维计算每个位置的交叉熵后：

```text
token_losses.shape == (batch_size, seq_len)
```

### 6.1 有效 token 平均

如果 $M_{b,t}$ 表示位置 $(b,t)$ 是否有效：

$$
M_{b,t}
=
\begin{cases}
1,&\text{有效 token}\\
0,&\text{padding 或需要忽略的位置}
\end{cases}
$$

则平均损失为：

$$
\boxed{
\mathcal L_{\mathrm{mean}}
=
\frac{
\sum_{b,t}M_{b,t}\mathcal L_{b,t}
}{
\sum_{b,t}M_{b,t}
}
}
$$

分母是**有效 token 数量**，不是固定写成 `batch_size × seq_len`。

### 6.2 `none`、`sum`、`mean`

常见 reduction：

| reduction | 结果 | 用途 |
|---|---|---|
| `none` | 保留每个 token 的损失 | 做 mask、分析难例 |
| `sum` | 所有损失相加 | 统计整批总 NLL |
| `mean` | 损失求平均 | 常用训练目标 |

训练通常使用平均值，使损失和梯度尺度不至于仅因为 batch 或序列更长就线性变大。

## 7. Softmax + 交叉熵的梯度直觉

对单个样本：

$$
\mathcal L
=
-\sum_i q_i\ln p_i
$$

为避免求和下标和求导下标混淆，下面把交叉熵中的求和下标写成 $j$。将 Softmax

$$
p_j
=
\frac{e^{z_j}}{\sum_k e^{z_k}}
$$

代入交叉熵：

$$
\begin{aligned}
\mathcal L
&=
-\sum_j q_j
\ln\left(
\frac{e^{z_j}}{\sum_k e^{z_k}}
\right)\\
&=
-\sum_j q_j
\left(
z_j-\ln\sum_k e^{z_k}
\right)\\
&=
-\sum_j q_jz_j
+
\left(\sum_jq_j\right)
\ln\sum_k e^{z_k}\\
&=
-\sum_j q_jz_j
+
\ln\sum_k e^{z_k}
\end{aligned}
$$

最后一步使用了目标分布满足 $\sum_jq_j=1$。现在对第 $i$ 个 logit $z_i$ 求偏导：

$$
\begin{aligned}
\frac{\partial\mathcal L}{\partial z_i}
&=
-\frac{\partial}{\partial z_i}
\sum_jq_jz_j
+
\frac{\partial}{\partial z_i}
\ln\sum_k e^{z_k}\\
&=
-q_i
+
\frac{1}{\sum_k e^{z_k}}
\frac{\partial}{\partial z_i}
\sum_k e^{z_k}\\
&=
-q_i
+
\frac{e^{z_i}}{\sum_k e^{z_k}}\\
&=
-q_i+p_i
\end{aligned}
$$

Softmax 与交叉熵组合后，对第 $i$ 个 logit 的梯度会简化为：

$$
\boxed{
\frac{\partial\mathcal L}{\partial z_i}
=
p_i-q_i
}
$$

如果目标是 one-hot：

### 正确类别 $i=y$

$$
\frac{\partial\mathcal L}{\partial z_y}
=
p_y-1<0
$$

梯度下降会减去这个负数，因此**提高正确类别的 logit**。

### 错误类别 $i\ne y$

$$
\frac{\partial\mathcal L}{\partial z_i}
=
p_i>0
$$

梯度下降会减去正数，因此**降低错误类别的 logit**。

直觉总结：

```text
模型多给错误类别的概率 → 把它们的 logits 往下压
模型少给正确类别的概率 → 把正确 logit 往上推
```

这就是交叉熵如何给梯度下降提供学习方向。若损失还对多个 token 取平均，梯度会多一个平均系数，但方向不变。

## 8. NumPy 从零实现

对第 $i$ 个 logit，`log_softmax` 的普通形式为：

$$
\begin{aligned}
\operatorname{log\_softmax}(\boldsymbol z)_i
&=
\ln\left(
\frac{e^{z_i}}{\sum_j e^{z_j}}
\right)\\
&=
z_i-\ln\sum_j e^{z_j}
\end{aligned}
$$

为避免直接计算较大的 $e^{z_j}$，令 $m=\max_jz_j$，得到数值稳定形式：

$$
\boxed{
\operatorname{log\_softmax}(\boldsymbol z)_i
=
(z_i-m)
-
\ln\sum_j e^{z_j-m}
}
$$

代码中的 `maximum`、`shifted` 和 `log_denominator` 分别对应 $m$、$z_i-m$ 和 $\ln\sum_j e^{z_j-m}$。

稳定的 `log_softmax`：

```python
def log_softmax(logits, axis=-1):
    maximum = np.max(logits, axis=axis, keepdims=True)
    shifted = logits - maximum
    log_denominator = np.log(
        np.exp(shifted).sum(axis=axis, keepdims=True)
    )
    return shifted - log_denominator
```

设 $u$ 表示任意一个样本或 token 位置，$y_u$ 是该位置的目标类别索引。先从 `log_softmax` 的结果中选出目标类别的对数概率：

$$
s_u
=
\operatorname{log\_softmax}(\boldsymbol z_u)_{y_u}
=
\ln p_{u,y_u}
$$

取负后得到该位置的交叉熵，也就是正确类别的 NLL：

$$
\boxed{
\ell_u
=
-s_u
=
-\ln p_{u,y_u}
}
$$

代码中的三种 `reduction` 分别对应：

$$
\begin{aligned}
\mathrm{none}:\quad
&\boldsymbol\ell
=
[\ell_1,\ell_2,\ldots,\ell_N]\\
\mathrm{sum}:\quad
&\mathcal L_{\mathrm{sum}}
=
\sum_{u=1}^{N}\ell_u\\
\mathrm{mean}:\quad
&\mathcal L_{\mathrm{mean}}
=
\frac{1}{N}\sum_{u=1}^{N}\ell_u
\end{aligned}
$$

从 logits 和类别索引计算交叉熵：

```python
def cross_entropy_from_logits(logits, targets, reduction="mean"):
    # 沿最后的类别维计算每个类别的 log-probability。
    log_probabilities = log_softmax(logits, axis=-1)

    # 按 targets 选出每个位置正确类别的 log-probability。
    selected = np.take_along_axis(
        log_probabilities,
        targets[..., np.newaxis],
        axis=-1,
    )[..., 0]

    # 取负得到每个位置的交叉熵（NLL）。
    losses = -selected

    # 保留逐位置损失，或对全部损失求和、求平均。
    if reduction == "none":
        return losses
    if reduction == "sum":
        return losses.sum()
    if reduction == "mean":
        return losses.mean()
    raise ValueError("reduction 必须是 none、sum 或 mean")
```

形状关系：

```text
logits:             (..., vocab_size)
targets:            (...)
log_probabilities:  (..., vocab_size)
losses:             (...)
```

```bash

```

## 9. 常见误区

### 误区 1：总是对最大概率取负对数

错。应该取**真实标签对应概率** $p_y$，即使它不是最大概率。

### 误区 2：交叉熵损失必须在 $[0,1]$

错。损失不是概率。它最小为 $0$，但没有有限上界。

### 误区 3：先 Softmax，再传给期待 logits 的交叉熵函数

错。这会重复 Softmax。应先确认 API 接收的是 logits、log-probabilities 还是 probabilities。

### 误区 4：直接计算 `-np.log(softmax(logits))` 总是安全

错。Softmax 的小概率可能下溢为 `0`，随后 `log(0)` 得到负无穷。应使用稳定的 `log_softmax` 或框架融合实现。

### 误区 5：padding token 也应该参与平均

通常不应该。padding 只是为了对齐形状，应使用 mask 或 `ignore_index` 排除。

### 误区 6：交叉熵只看预测是否正确

错。即使两个模型的 `argmax` 相同，给正确类别的概率不同，损失也不同。

### 误区 7：交叉熵和 NLL 在所有数学问题中都完全相同

不严谨。对 one-hot 分类目标，两者数值相同；一般软目标交叉熵会综合多个类别。

## 核心记忆卡

```text
单个正确 token：
    loss = -ln(p_correct)

one-hot 分类：
    cross_entropy(q, p)
    = -Σ_i q_i ln(p_i)
    = -ln(p_correct)
    = NLL

从 logits 稳定计算：
    log_probs = stable_log_softmax(logits)
    loss = -log_probs[target]

LLM 形状：
    logits       (batch, seq_len, vocab_size)
    targets      (batch, seq_len)
    token_losses (batch, seq_len)

有效 token 平均：
    sum(mask * token_losses) / sum(mask)

梯度：
    dL/dlogits = probabilities - targets_one_hot

训练方向：
    提高正确 token 的 logit
    降低错误 token 的 logits
```

掌握标准只保留三项：

1. 能解释并手算 $-\ln p_y$
2. 能说明 one-hot 分类中交叉熵为什么等于 NLL
3. 能从 logits 写出稳定 NumPy 实现，并正确处理 token 维与 mask
