## 2 · 最小熵、最大熵与零概率（20 分）

有三个四分类分布：

$$
\begin{aligned}
\boldsymbol p&=[1,0,0,0]\\
\boldsymbol q&=[0.5,0.5,0,0]\\
\boldsymbol r&=[0.25,0.25,0.25,0.25]
\end{aligned}
$$

本题使用以 `2` 为底的对数。

1. 分别计算 $H(\boldsymbol p)$、$H(\boldsymbol q)$ 和 $H(\boldsymbol r)$。
2. 对四分类分布，熵的理论取值范围是什么？
3. 哪个分布最容易预测，哪个最难预测？说明原因。
4. 公式中的 $0\log0=0$ 是否表示 $\log0=0$？正确解释是什么？
5. 不精确计算，判断 $[0.70,0.20,0.10]$ 和 $[0.34,0.33,0.33]$ 哪个熵更大，并说明原因。

✍️ 你的答案：

1.
$$
\begin{aligned}
H(\boldsymbol p)
&= -\sum_{i=1}^{K}p_i\log p_i  \\
&= 0+0+0+0 \\
&= 0
\end{aligned}
\\

\begin{aligned}
H(\boldsymbol q)
&= -\sum_{i=1}^{K}p_i\log p_i  \\
&=0.5+0.5+0+0 \\
&= 1
\end{aligned}
\\

\begin{aligned}
H(\boldsymbol q)
&= -\sum_{i=1}^{K}r_i\log r_i  \\
&= 0.5+0.5+0.5+0.5 \\
&= 2
\end{aligned}
$$
2. 对4分类分布，伤的取值范围是：
   $$
   0\le H(\boldsymbol p)\le\log 4 \\
            得到\\
   0\le H(\boldsymbol p)\le2
   $$

3. p最容易预测，因为它的熵为0，表示整体难预测程度为0，是完全确定的，r最难预测，它的熵为2=log(4),表示最大的难预测程度
4. 不是表示 $\log0=0$，而是使用极限：

$$
\lim_{p\to0^+}p\log p=0
$$

5. `[0.34,0.33,0.33]`  的熵更大，直觉上 分布越平均，不确定性越大，熵越大

> ✅ **批改第 2 题（19 / 20）**：三个熵、理论范围、最大/最小熵、$0\log0$ 的极限解释以及分布均匀程度的判断均正确。符号上需要纠正：第三个计算应写成 $H(\boldsymbol r)$，不是 $H(\boldsymbol q)$；各分布的求和项最好分别使用 $p_i$、$q_i$、$r_i$，并明确本题是 $\log_2$。文字中的“伤的取值范围”应为“熵的取值范围”。

## 3 · LLM 的熵、NLL 与预测正确性（20 分）

模型在某个位置给三个 token 的概率为：

$$
\boldsymbol p=[0.7,0.2,0.1]
$$

实际出现的是第三个 token。本题使用自然对数。

1. 计算预测分布的熵 $H(\boldsymbol p)$，保留四位小数。
2. 计算实际 token 的 NLL，保留四位小数。
3. 为什么两个数不同？分别回答了什么问题？
4. 另一个模型给出 $\boldsymbol q=[0.98,0.01,0.01]$，实际 token 仍是第三个。它的熵约为 `0.1119` nats。比较两个模型的熵与 NLL，并说明为什么“熵更低”不代表“预测更正确”。
5. 判断并纠正：“某个位置的 token 熵很高，说明模型一定很差。”

✍️ 你的答案：

1. $H(\boldsymbol p)=0.8018$
2. $NLL=-\ln(0.1)=2.3026$
3. 两个数衡量的对象不同：熵考虑整个预测分布，是所有候选 token 自信息的概率加权平均，回答“模型在这个位置总体有多不确定”；NLL 只考虑实际出现的第三个 token，回答“模型对这次真实结果有多意外”。
4. 对于 $\boldsymbol q$：

   $$
   H(\boldsymbol q)\approx0.1119,\qquad
   NLL_q=-\ln(0.01)\approx4.6052
   $$

   因此 $H(\boldsymbol q)<H(\boldsymbol p)$，但 $NLL_q>NLL_p$。$\boldsymbol q$ 的概率更加集中、熵更低，却把概率集中在错误的第一个 token 上，只给真实 token 分配了 $0.01$ 的概率。熵低只表示模型更加确定，不表示它确定的是正确答案。
5. 这句话不正确。高 token 熵只说明当前预测分布比较分散，可能是模型不确定，也可能是当前上下文本来就允许多种合理续写。判断模型好坏还需要结合真实 token 的 NLL、任务指标和生成质量。

## 4 · Temperature、采样与有效候选数（20 分）

同一组 logits 在三种 temperature 下得到近似分布：

```text
T = 0.5：[0.8668, 0.1173, 0.0159]
T = 1.0：[0.6652, 0.2447, 0.0900]
T = 2.0：[0.5065, 0.3072, 0.1863]
```

1. 不计算，先按熵从小到大排序，并解释依据。
2. 使用自然对数计算三个熵，保留四位小数。
3. 分别计算有效候选数 $e^H$，保留两位小数。
4. Temperature 改变的是模型参数、原始 logits，还是用于采样的概率分布？
5. 若改成 greedy decoding，每次都选概率最大的 token，是否表示模型的原始预测分布熵变成了 `0`？说明“分布的熵”和“解码规则”之间的区别。

✍️ 你的答案：

1. 熵从小到大的排序为：

   $$
   H_{T=0.5}<H_{T=1.0}<H_{T=2.0}
   $$

   温度越小，分布越尖锐，确定性越强，熵越小；温度越高，分布越平坦，熵越大。
2. 根据 $H(\boldsymbol p)=-\sum_i p_i\ln p_i$，三个自然熵约为：

   $$
   \begin{aligned}
   H_{T=0.5}&\approx0.4411\text{ nats}\\
   H_{T=1.0}&\approx0.8324\text{ nats}\\
   H_{T=2.0}&\approx1.0202\text{ nats}
   \end{aligned}
   $$

3. 三个分布的有效候选数为：

   $$
   \begin{aligned}
   e^{H_{T=0.5}}&\approx1.55\\
   e^{H_{T=1.0}}&\approx2.30\\
   e^{H_{T=2.0}}&\approx2.77
   \end{aligned}
   $$

   它们表示三个分布的不确定性分别约等于 `1.55`、`2.30` 和 `2.77` 个等概率候选。
4. Temperature 不改变训练好的模型参数，也不改变模型最初输出的原始 logits。它通过

   $$
   p_i(T)=\frac{e^{z_i/T}}{\sum_j e^{z_j/T}}
   $$

   缩放 logits，从而改变用于采样或解码的概率分布。$T<1$ 时分布更尖锐，$T>1$ 时分布更平坦。
5. 不表示模型原始预测分布的熵变成了 `0`。Greedy decoding 只是按照 $\arg\max_i p_i$ 确定性地选择概率最大的 token，并不会修改模型原本输出的概率分布及其熵。“分布的熵”衡量模型对候选 token 有多不确定；“解码规则”决定如何从该分布中选择 token。若只把 greedy decoding 的最终选择看作一个确定分布，其熵可以视为 `0`，但它与模型原始 token 分布的熵是两个不同的对象。

> ✅ **批改第 4 题（20 / 20）**：三个熵及有效候选数均正确；temperature 的作用对象、尖锐程度变化，以及 greedy decoding 与模型原始分布熵的区别解释完整。

## 5 · 公式与 NumPy 代码纠错（20 分）

下面代码试图计算一维离散概率分布的自然熵：

```python
import numpy as np


def entropy(probabilities):
    p = np.asarray(probabilities, dtype=np.float64)
    return -np.mean(np.log(p))
```

1. 指出核心数学错误。
2. 输入包含 `0` 时，这段代码会出现什么问题？如何正确实现 $0\log0=0$？
3. 补充必要输入校验并改写函数。至少拒绝：空数组、非一维数组、NaN/Inf、负概率、概率和不为 `1`。
4. 你的函数对 `[1,0,0,0]` 和 `[0.25,0.25,0.25,0.25]` 应分别返回多少 nats？
5. 判断并纠正：“对任意模型和任意测试集，`exp(entropy(model_probs))` 都等于测试集 perplexity。”

✍️ 你的答案：

1. 核心数学错误是 `-np.mean(np.log(p))` 对所有候选 token 的自信息进行了等权平均：

   $$
   -\frac{1}{K}\sum_i\ln p_i
   $$

   但熵必须使用每个结果自身的概率作为权重：

   $$
   H(\boldsymbol p)=-\sum_i p_i\ln p_i
   $$

2. 当输入包含 `0` 时，`np.log(0)` 会产生 `-inf` 和运行时警告，原函数可能返回 `inf`。$0\log0=0$ 并不是说 $\log0=0$，而是采用极限约定：

   $$
   \lim_{p\to0^+}p\ln p=0
   $$

   实现时只对大于 `0` 的概率计算 $p_i\ln p_i$。
3. 一种完整实现为：

   ```python
   import numpy as np


   def entropy(probabilities):
      p = np.asarray(probabilities, dtype=np.float64)
      if p.ndim != 1 or p.size == 0:
         raise ValueError("probabilities 必须是一维非空数组")
      if not np.all(np.isfinite(p)):
         raise ValueError("probabilities 不能包含 NaN 或 Inf")
      if np.any(p < 0.0):
         raise ValueError("概率不能为负")
      if not np.isclose(p.sum(), 1.0):
         raise ValueError("概率之和必须为 1")

      positive = p > 0.0
      return float(-np.sum(p[positive] * np.log(p[positive])))
   ```

   其中 `positive` 掩码可以避免直接计算 `np.log(0)`。
4. 两个结果分别为：

   $$
   H([1,0,0,0])=0\text{ nats}
   $$

   $$
   \begin{aligned}
   H([0.25,0.25,0.25,0.25])
   &=-4(0.25\ln0.25)\\
   &=\ln4\\
   &\approx1.3863\text{ nats}
   \end{aligned}
   $$

5. 这句话不正确。单个预测分布的有效候选数为：

   $$
   N_{\mathrm{eff}}
   =\exp\left(-\sum_i p_i\ln p_i\right)
   $$

   它描述模型在某个位置的预测概率有多分散。测试集困惑度则是真实 token 的平均 NLL 的指数：

   $$
   \operatorname{PPL}
   =\exp\left(
   \frac{1}{N}\sum_{t=1}^{N}
   -\ln p_\theta(y_t\mid y_{<t})
   \right)
   $$

   它只查看每个位置上模型分配给真实 token 的概率。因此，`exp(entropy(model_probs))` 通常不等于测试集 PPL。只有真实 token 确实按照同一个模型分布产生，并进行相应的期望或充分多次采样等特殊条件下，两者才可能一致或近似一致。

5. 不表示。Greedy decoding 是从原始预测分布中确定性选择 `argmax` 的规则，不会反过来修改原始预测分布及其熵。若把“解码器最终总选择 argmax”单独看成一个确定分布，其熵可视为 `0`；但这和模型原始 token 分布的熵是两个对象。

## 5 · 参考答案

1. `-np.mean(np.log(p))` 对各类别的自信息做了等权普通平均。熵必须用类别自身概率加权：

   $$
   H(\boldsymbol p)
   =
   -\sum_i p_i\ln p_i
   $$

2. `np.log(0)` 会得到负无穷并产生警告。应只对正概率项计算乘积；零概率项按极限记为 `0`。

3. 一种实现：

   ```python
   import numpy as np


   def entropy(probabilities):
       p = np.asarray(probabilities, dtype=np.float64)
       if p.ndim != 1 or p.size == 0:
           raise ValueError("probabilities 必须是一维非空数组")
       if not np.all(np.isfinite(p)):
           raise ValueError("probabilities 不能包含 NaN 或 Inf")
       if np.any(p < 0.0):
           raise ValueError("概率不能为负")
       if not np.isclose(p.sum(), 1.0):
           raise ValueError("概率之和必须为 1")

       positive = p > 0.0
       return float(-np.sum(p[positive] * np.log(p[positive])))
   ```

4. 结果为：

   $$
   H([1,0,0,0])=\boxed{0\text{ nats}}
   $$

   $$
   H([0.25,0.25,0.25,0.25])
   =
   \ln4
   \approx
   \boxed{1.3863\text{ nats}}
   $$

5. 错误。`exp(entropy(model_probs))` 描述某个模型预测分布自身的有效候选数；测试集 PPL 是测试数据上平均 NLL 的指数，通常对应真实数据分布与模型分布之间的交叉熵。只有数据确实按同一个模型分布产生等特定条件下，两者才一致。
