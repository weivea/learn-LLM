## 1 · KL 的公式、逐项计算与交叉熵分解（20 分）

给定两个离散概率分布：

$$
q=[0.5,0.25,0.25],
\qquad
p=[0.4,0.4,0.2]
$$

本题使用自然对数。

1. 写出 $D_{\mathrm{KL}}(q\|p)$ 的公式，并说明为什么由 $q$ 提供求和权重。
2. 分别计算三个 $q_i\ln(q_i/p_i)$，再计算总 KL，保留四位小数。
3. 第二项为什么是负数？这是否违反 KL 非负性？
4. 计算 $H(q)$ 与 $H(q,p)$，验证：

   $$
   H(q,p)=H(q)+D_{\mathrm{KL}}(q\|p)
   $$

5. $D_{\mathrm{KL}}(q\|p)=0$ 的条件是什么？KL 很小说明什么，又不能说明什么？

✍️ 你的答案：

1.
   $$
   D_{\mathrm{KL}}(q\|p)
   =
   \sum_{i=1}^{K}
   q_i\ln\frac{q_i}{p_i}
   $$
   KL 散度是在结果按真实分布 $q$ 出现时，用模型分布 $p$ 代替 $q$ 所增加的平均对数代价。

2. [0.5 * Math.log(0.5/0.4), 0.25*Math.log(0.25/0.4), 0.25*Math.log(0.25/0.2)] = [0.1116, -0.1175, 0.0558] \
   $D_{\mathrm{KL}}(q\|p)\approx 0.0498$

3. 第二项表示模型p的概率高古了实际概率q, 所以 $\ln\frac{q_i}{p_i}$ 为负数，那$q_i\ln\frac{q_i}{p_i}$也为负数，并不违反，KL的非负性是要求，所有的$q_i\ln\frac{q_i}{p_i}$ 总和为非负。单个或者一部分可以为负
4.
$$
H(q) = -\sum_{i=1}^{K}q_i\ln q_i \approx 1.0397 \\
H(q,p)=  -\sum_{i=1}^{K}q_i\ln p_i \approx 1.0896 \\
1.0896 \approx 1.0397 + 0.0498
$$

5. $D_{\mathrm{KL}}(q\|p)=0$ 说明 p = q;
   KL很小说明 p跟q很接近，不能说明p正确

## 2 · 方向、非对称性与零概率（20 分）

给定：

$$
q=[0.5,0.5],
\qquad
p=[0.9,0.1]
$$

本题使用自然对数。

1. 分别计算 $D_{\mathrm{KL}}(q\|p)$ 和 $D_{\mathrm{KL}}(p\|q)$，保留四位小数。
2. 为什么交换方向后数值不同？两个方向分别由哪个分布决定结果出现频率？
3. 分别判断并计算：

   $$
   D_{\mathrm{KL}}([1,0]\|[0.5,0.5])
   $$

   $$
   D_{\mathrm{KL}}([0.5,0.5]\|[1,0])
   $$

4. 完整说明 $q_i=0$ 与 $q_i>0,p_i=0$ 两种边界怎样处理。
5. 判断并纠正：“KL 非负，而且两个分布相同才为 $0$，所以它就是一种严格的距离。”

✍️ 你的答案：
1.
$$
D_{\mathrm{KL}}(q\|p) = 0.5108 \\
D_{\mathrm{KL}}(p\|q) = 0.3681
$$

2. 因为KL散度不是严格的距离，q，p顺序交换后代表的含义不一样，它表达的是“结果按照真实分布 (前一项) 出现时，模型分布 (后一项) 对这些结果的概率判断比 真实分布(前一项) 差多少”。\
   $D_{\mathrm{KL}}(q\|p)$由q决定结果出现频率 \
   $D_{\mathrm{KL}}(p\|q)$由p决定结果出现频率

3.
$$
D_{\mathrm{KL}}([1,0]\|[0.5,0.5]) \approx 0.693147 \\
D_{\mathrm{KL}}([0.5,0.5]\|[1,0]) = inf
$$
4. 当 $q_i=0$ 时，该结果在真实分布 $q$ 下不会出现，该项按极限记为：

   $$
   0\ln\frac{0}{p_i}=0
   $$

   因此它对 KL 散度没有贡献，包括 $q_i=p_i=0$ 的情况。若 $q_i>0,p_i=0$，真实分布认为该结果可能出现，但模型认为它绝不可能，此时：

   $$
   q_i\ln\frac{q_i}{0}=+\infty
   $$

   因此整个 $D_{\mathrm{KL}}(q\|p)=+\infty$。

5. 这个说法错误。KL 散度虽然非负，并且当且仅当两个分布相同时为 $0$，但它通常不满足对称性：

   $$
   D_{\mathrm{KL}}(q\|p)
   \ne
   D_{\mathrm{KL}}(p\|q)
   $$

   KL 散度也不满足一般的三角不等式，因此不是严格的数学距离，而是一个有方向的分布差异度量。本题中 $0.5108\ne0.3681$ 就说明了它不满足对称性。

## 5 · NumPy 代码纠错与边界实现（20 分）

下面代码声称计算 $D_{\mathrm{KL}}(q\|p)$：

```python
import numpy as np


def kl_divergence(q, p):
    q = np.asarray(q, dtype=np.float64)
   p = np.asarray(p, dtype=np.float64)
   return np.sum(p * np.log(p / q))
```

1. 指出它在方向上的错误；它实际计算的是什么？
2. 输入含 $0$ 时，这段代码会出现哪些数学或数值问题？
3. 补充必要输入校验并正确实现 $D_{\mathrm{KL}}(q\|p)$。至少拒绝：空数组、非一维数组、shape 不同、NaN / Inf、负概率、概率和不为 $1$。
4. 实现必须正确区分：

   ```text
   KL([1, 0] || [0.5, 0.5])
   KL([0.5, 0.5] || [1, 0])
   ```

   两者应分别返回什么？
5. 判断并纠正：“为了避免 $\log 0$，给所有概率加 $10^{-12}$ 就行，这不会改变 KL 的含义。”

✍️ 你的答案：

1. 原代码把 $p_i$ 作为权重，并计算 $\ln(p_i/q_i)$，因此实际计算的是：

   $$
   D_{\mathrm{KL}}(p\|q)
   =
   \sum_i p_i\ln\frac{p_i}{q_i}
   $$

   而不是声称的 $D_{\mathrm{KL}}(q\|p)$。另外，原代码中 `p` 和 `return` 两行也存在缩进错误。

2. 输入含 $0$ 时，直接计算可能出现以下问题：

   - $0/0$ 产生 `NaN`；
   - 正数除以 $0$ 产生 `Inf`；
   - $\ln 0$ 产生 `-Inf`；
   - NumPy 中 `0 * Inf` 或 `0 * NaN` 会产生 `NaN`。

   数学上应按 KL 的边界定义处理：

   $$
   q_i=0
   \quad\Longrightarrow\quad
   q_i\ln\frac{q_i}{p_i}=0
   $$

   $$
   q_i>0,\ p_i=0
   \quad\Longrightarrow\quad
   q_i\ln\frac{q_i}{p_i}=+\infty
   $$

3. 正确实现如下：

   ```python
   import numpy as np


   def validate_distribution(probabilities, name):
      values = np.asarray(probabilities, dtype=np.float64)

      if values.ndim != 1:
         raise ValueError(f"{name} 必须是一维数组")
      if values.size == 0:
         raise ValueError(f"{name} 不能为空")
      if not np.all(np.isfinite(values)):
         raise ValueError(f"{name} 不能包含 NaN 或 Inf")
      if np.any(values < 0.0):
         raise ValueError(f"{name} 不能包含负概率")
      if not np.isclose(values.sum(), 1.0):
         raise ValueError(f"{name} 的概率之和必须为 1")

      return values


   def kl_divergence(q, p):
      q_values = validate_distribution(q, "q")
      p_values = validate_distribution(p, "p")

      if q_values.shape != p_values.shape:
         raise ValueError("q 和 p 必须具有相同 shape")

      positive_q = q_values > 0.0

      if np.any(p_values[positive_q] == 0.0):
         return np.inf

      return float(
         np.sum(
            q_values[positive_q]
            * (
               np.log(q_values[positive_q])
               - np.log(p_values[positive_q])
            )
         )
      )
   ```

   只在 $q_i>0$ 的位置计算对数，可以跳过按定义贡献为 $0$ 的项；计算前单独检测 $q_i>0,p_i=0$，保证应为正无穷的情况不会被忽略。

4. 两个结果分别是：

   $$
   \begin{aligned}
   D_{\mathrm{KL}}([1,0]\|[0.5,0.5])
   &=1\ln\frac{1}{0.5}\\
   &=\ln2\\
   &\approx\boxed{0.6931}
   \end{aligned}
   $$

   第二个方向存在 $q_2=0.5>0,p_2=0$，所以：

   $$
   D_{\mathrm{KL}}([0.5,0.5]\|[1,0])
   =
   \boxed{+\infty}
   $$

5. 这个说法错误。给所有概率加 $\varepsilon=10^{-12}$ 会改变原概率分布，而且会使概率和不再等于 $1$。即使随后重新归一化，计算的也是平滑后分布之间的 KL，而不是原分布的 KL。

   更重要的是，平滑会把原本应为 $+\infty$ 的支撑集不匹配变成有限值，从而改变问题的数学含义。正确做法是依据 KL 的边界定义分别处理零概率。如果业务确实需要平滑，应明确平滑规则、重新归一化，并说明计算目标已经改变。

## 1 · 参考答案

1. 公式为：

   $$
   D_{\mathrm{KL}}(q\|p)
   =
   \sum_i q_i\ln\frac{q_i}{p_i}
   $$

   这是 $X\sim q$ 时 log-ratio 的期望。结果按真实分布 $q$ 出现，所以每项必须由 $q_i$ 加权；模型分布 $p$ 被用来近似 $q$。

2. 三项分别为：

   $$
   \begin{aligned}
   0.5\ln\frac{0.5}{0.4}
   &\approx0.1116\\
   0.25\ln\frac{0.25}{0.4}
   &\approx-0.1175\\
   0.25\ln\frac{0.25}{0.2}
   &\approx0.0558
   \end{aligned}
   $$

   所以：

   $$
   \begin{aligned}
   D_{\mathrm{KL}}(q\|p)
   &\approx
   0.1116-0.1175+0.0558\\
   &\approx
   \boxed{0.0499\text{ nats}}
   \end{aligned}
   $$

3. 第二项中 $q_2<p_2$，所以 $q_2/p_2<1$，其对数为负。这不违反非负性，因为 KL 保证的是所有项之和非负，不是每一项都非负。

4. 熵为：

   $$
   \begin{aligned}
   H(q)
   &=
   -0.5\ln0.5
   -0.25\ln0.25
   -0.25\ln0.25\\
   &\approx
   \boxed{1.0397\text{ nats}}
   \end{aligned}
   $$

   交叉熵为：

   $$
   \begin{aligned}
   H(q,p)
   &=
   -0.5\ln0.4
   -0.25\ln0.4
   -0.25\ln0.2\\
   &\approx
   \boxed{1.0896\text{ nats}}
   \end{aligned}
   $$

   验证：

   $$
   1.0397+0.0499
   \approx
   1.0896
   $$

5. 在离散分布的对应概率完全相同时，KL 为 $0$。KL 很小说明在指定方向和对数尺度下模型分布 $p$ 接近真实分布 $q$，但它不能说明两个分布在反方向的 KL 同样小，也不能单独说明模型任务效果、生成质量或奖励一定更好。

## 5 · 参考答案

1. 原代码的权重是 $p_i$，log-ratio 是 $p_i/q_i$，所以实际计算：

   $$
   D_{\mathrm{KL}}(p\|q)
   $$

   而不是函数声称的 $D_{\mathrm{KL}}(q\|p)$。

2. $0/0$ 会产生 `NaN`，$\log 0$ 会产生负无穷，随后与零相乘也可能得到 `NaN`。数学上必须分别处理 $q_i=0$ 和 $q_i>0,p_i=0$，不能让 NumPy 的默认浮点传播代替定义。

3. 一种完整实现为：

   ```python
   import numpy as np


   def validate_distribution(probabilities, name):
       values = np.asarray(probabilities, dtype=np.float64)
       if values.ndim != 1 or values.size == 0:
           raise ValueError(f"{name} 必须是一维非空数组")
       if not np.all(np.isfinite(values)):
           raise ValueError(f"{name} 不能包含 NaN 或 Inf")
       if np.any(values < 0.0):
           raise ValueError(f"{name} 不能包含负概率")
       if not np.isclose(values.sum(), 1.0):
           raise ValueError(f"{name} 的概率之和必须为 1")
       return values


      def kl_divergence(q, p):
       q_values = validate_distribution(q, "q")
         p_values = validate_distribution(p, "p")
         if q_values.shape != p_values.shape:
            raise ValueError("q 和 p 必须具有相同 shape")

         positive_q = q_values > 0.0
         if np.any(p_values[positive_q] == 0.0):
           return np.inf

       return float(
           np.sum(
               q_values[positive_q]
               * (
                  np.log(q_values[positive_q])
                  - np.log(p_values[positive_q])
               )
           )
       )
   ```

4. 两个结果分别为：

   $$
   \boxed{
   D_{\mathrm{KL}}([1,0]\|[0.5,0.5])
   =
   \ln2
   \approx0.6931
   }
   $$

   $$
   \boxed{
   D_{\mathrm{KL}}([0.5,0.5]\|[1,0])
   =
   +\infty
   }
   $$

5. 错误。无条件加 $\varepsilon$ 会改变两个概率分布，通常还会让概率和不再为 $1$；若再归一化，则计算的是平滑后分布的 KL，而不是原分布的 KL。它还会把数学上应为 $+\infty$ 的支撑集错误隐藏成有限值。若业务需要平滑，应明确规则、重新归一化，并说明目标已经改变。
